属性绑定

示例 1​

任务​

已定义国家（Country）、地区（Region）和城市（City）的概念。地区和城市总是位于某个国家。城市可能有指定的地区，但并非必须。

CLASS Country 'Country';
name 'Name' = DATA ISTRING[100] (Country) IN id;

CLASS Region 'Region';
name 'Name' = DATA ISTRING[100] (Region) IN id;

country 'Country' = DATA Country (Region) NONULL DELETE;
nameCountry 'Country' (Region r)= name(country(r)) IN id;

CLASS City 'City';
name 'Name' = DATA ISTRING[100] (City);

country 'Country' = DATA Country (City) NONULL DELETE;
nameCountry (City c) = name(country(c));

region 'Region' = DATA Region (City);
nameRegion (City c) = name(region(c));

FORM cities 'Cities'
    OBJECTS c = City
    PROPERTIES(c) name, nameCountry, nameRegion, NEW, DELETE
;

NAVIGATOR {
    NEW cities;
}

我们需要定义逻辑，使得城市只能链接到属于其所在国家的地区。

解决方案​

CONSTRAINT country(City c) != country(region(c)) CHECKED BY region[City] MESSAGE '城市地区的国家必须与城市的国家匹配';

CONSTRAINT语句定义了一个必须始终为NULL的条件。在这种情况下，如果为地区定义了国家，为城市定义了地区，并且地区的国家与城市的国家不同，则基于此条件的约束将被违反。CHECKED BY块表示在对话框中选择地区时，默认会过滤地区，以免违反此约束。需要注意的是，如果尚未为给定城市设置国家，则条件将始终为NULL，对话框中将显示所有现有地区。

示例 2​

任务​

类似于示例1。

我们需要在选择地区时自动填充国家。

解决方案​

WHEN LOCAL CHANGED(region(City c)) AND NOT CHANGED(country(c)) DO {
    country(c) <- country(region(c));
}

需要注意的是，在用户选择地区并设置国家后，再次调用对话框时，将只显示所选国家的地区。如果用户想再次查看所有地区，则需要先清除国家。检查国家未更改的表达式是为了提供以下行为：如果城市被外部操作更改，同时更改了国家和地区，则国家不应随此事件而更改。

示例 3​

任务​

类似于示例1。

我们需要创建地区和城市之间的相同类型的关联，但方式是在首先设置国家之前不能选择地区。

解决方案​

CONSTRAINT region(City c) AND NOT country(c) = country(region(c)) CHECKED BY region[City] MESSAGE '城市地区的国家必须与城市的国家匹配';

与第一个示例的区别在于，如果设置了地区但未设置国家，则新条件也将为真。因此，在这种情况下，对话框将不显示任何地区。

示例 4​

任务​

类似于示例1。

如果用户先选择一个地区，然后选择一个与原始地区不对应的国家，则用户在尝试保存时将收到错误消息。

我们需要确保在选择新国家时，如果地区与该国家不对应，则地区将被删除。

解决方案​

WHEN LOCAL CHANGED(country(City c)) AND country(c) != country(region(c)) DO {
    region(c) <- NULL;
}

示例 5​

任务​

类似于示例1。

第一个示例的解决方案有一个缺点。如果城市链接到特定地区，系统将不允许您更改该地区的国家。约束将被违反。

我们需要确保当地区的国家更改时，该地区所有城市的国家也会自动更改。

解决方案​

// 选项 1
WHEN SETCHANGED(country(region(City c))) DO
    country(c) <- country(region(c));

// 选项 2
WHEN SETCHANGED(country(Region r)) DO
    country(City c) <- country(r) WHERE region(c) = r;
