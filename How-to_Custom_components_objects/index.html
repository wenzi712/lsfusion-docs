自定义组件（对象）

默认情况下，表单上具有 GRID 视图的每个对象都在表单上显示为带有列的平面表格。 但是，您可以在平台中创建自己的组件来可视化对象列表。

作为一个示例，让我们考虑以"卡片"形式显示带有图像的产品列表的任务。

领域逻辑​

首先，让我们创建项目的类和属性，以及编辑表单：

CLASS Item '项目';

name '名称' = DATA STRING (Item) NONULL;
price '价格' = DATA NUMERIC[12,2] (Item) NONULL;
image '' = DATA IMAGEFILE (Item);

FORM item '项目'
    OBJECTS i = Item PANEL
    PROPERTIES(i) name, price, image
    
    EDIT Item OBJECT i
;

DESIGN item {
    OBJECTS {
        MOVE PROPERTY(image(i)) {
            fill = 1;
        }
    }
}

每个项目都必须指定名称、价格和图像。

界面​

让我们创建一个带有项目列表的表单。为此，让我们在表单中添加对象 Item、其属性以及添加、编辑和删除的操作：

FORM items '项目列表'
    OBJECTS i = Item CUSTOM 'itemCards'
    PROPERTIES(i) READONLY image, price, name
    PROPERTIES(i) NEWSESSION new = NEW, edit = EDIT GRID, DELETE GRID
;

NAVIGATOR {
    NEW items;
}

关键字 CUSTOM 指定不应使用标准表格界面来绘制项目列表， 而是使用由函数 itemCards 创建的组件。让我们在文件 itemcards.js 中声明此函数，我们将其放在文件夹 resources/web 中。 它将返回一个包含两个函数的对象：render 和 update。

函数 render 接受控制器和要在其中创建显示数据所需的新元素的元素作为输入：

render: (element, controller) => {
    let cards = document.createElement("div")
    cards.classList.add("item-cards");

    element.cards = cards;
    element.appendChild(cards);
},

在此示例中，我们创建一个新的 div cards，记住它，并将其附加到 element。

为了更新显示的值，平台每次都会调用 update 函数，并向其传递与 render 函数相同的 element， 以及对象的 list：

update: (element, controller, list) => {
    while (element.cards.lastElementChild) {
        element.cards.removeChild(element.cards.lastElementChild);
    }

    for (let item of list) {
        let card = document.createElement("div")
        card.classList.add("item-card");

        if (controller.isCurrent(item))
            card.classList.add("item-card-current");

        let cardImage = document.createElement("img")
        cardImage.classList.add("item-card-image");
        cardImage.src = item.image;
        card.appendChild(cardImage);

        let cardPrice = document.createElement("div")
        cardPrice.classList.add("item-card-price");
        cardPrice.innerHTML = item.price;
        card.appendChild(cardPrice);

        let cardName = document.createElement("div")
        cardName.classList.add("item-card-name");
        cardName.innerHTML = item.name;
        card.appendChild(cardName);

        element.cards.appendChild(card);

        card.onclick = function(event) {
            if (!controller.isCurrent(item)) controller.changeObject(item);
        }
        card.ondblclick = function(event) {
            controller.changeProperty('edit', item);
        }
    }
}

因为每当数据更改时都会调用 update 函数，所以首先会删除所有先前创建的元素（即项目卡片）。

此示例使用最简单的更新方案，但如有必要，可以通过仅更新已更改值的 DOM 来对其进行优化。 为此，controller 具有 getDiff 方法，您可以将新的对象 list 作为参数传递给该方法。 此方法将返回一个带有数组 add、update、remove 的对象，分别存储添加、更改和删除的对象。 示例：

let diff = controller.getDiff(list);
for (let object of diff.add) { ... }
for (let object of diff.update) { ... }
for (let object of diff.remove) { ... }

删除旧元素后，为 list 数组中的每个对象创建一个 div card，其中放置每个属性的所需显示元素。 对象字段的名称对应于表单上属性的名称。isCurrent 方法确定列表中的哪个对象是当前对象。

在函数的最后，向项目卡片添加鼠标点击处理程序。

单击时，会在控制器上调用 changeObject 方法，该方法更改当前对象。 未指定第二个参数（rendered）（即假定为 false），这意味着服务器最终必须使用新的对象列表（可能是同一个）调用 update 函数。 由于 isCurrent 方法的值会更改，重新创建项目卡片将更改界面中当前选择的对象。

双击时，会调用 changeProperty 方法，该方法更改在第二个参数中传递的对象的 edit 属性的当前值。 由于 edit 是一个操作，因此未传递第三个参数（应该更改为的属性当前值的值），而是调用该操作。 在这种情况下，将打开项目编辑表单。

为了将 render 和 update 函数组合成一个，创建了 itemCards 函数，该函数在同一对象中返回它们：

function itemCards() {
    return {
        render: function (element, controller) => {
            ...
        },
        update: function (element, controller, list) {
            ...
        }
    }
}

要完成设计设置，请创建文件 itemcards.css，我们也将其放在 resources/web 文件夹中：

.item-cards {
    display: grid;
    grid-template-columns: repeat(auto-fit, minmax(150px, 1fr));
    grid-auto-rows: 200px;
    grid-gap: 10px;
}

.item-card {
    cursor: pointer;
    display: flex;
    flex-direction: column;
    overflow: hidden;
    align-items: center;
    padding: 8px;
}
.item-card-current {
    background-color: lightblue;
}

.item-card-image {
    flex: 1;
    min-height: 100px;
}

.item-card-price {
    font-weight: bold;
}

.item-card-name {
    color: gray;
}

为了在浏览器中打开页面时加载创建的 js 和 css 文件，必须将它们的初始化添加到操作 onWebClientInit 中，方法是将文件名添加到 onWebClientInit(STRING) 属性。需要一个数值来指定加载顺序：

onWebClientInit() + {
    onWebClientInit('itemcards.js') <- 1;
    onWebClientInit('itemcards.css') <- 2;
}
