启动参数

应用服务器（Server）​

Java​

Java应用服务器启动参数在启动命令中设置（例如，对于手动或自动安装）：

名称类型描述默认值
系统参数（以X开头）标准标准Java参数。最重要的是要注意：
  • cp - 类路径，Java在其中查找类文件和其他资源（包括lsFusion模块）的路径。默认值为. - 当前文件夹（对于自动安装则不同）。
  • Xmx - 最大内存大小。默认值取决于运行应用服务器的计算机配置。对于复杂逻辑，建议至少分配4GB。
-XX:CMSInitiatingOccupancyFractionint通常，这是负责CMS垃圾收集器启动阈值的标准参数。同时，平台使用此参数通过LRU缓存来控制内存使用量（如果超出目标，则设置更积极的清理参数，反之则设置较不积极的参数）。对于高负载服务器，建议将其设置在40到60之间。70
自定义参数（以D开头）-Dlsfusion.server.lightstartboolean"轻量"启动模式（通常用于开发过程中）。在此模式下，服务器不执行元数据同步操作或创建安全策略设置表单等，因此减少了启动时间和启动时消耗的内存量。
在IDE中，它通过lsFusion服务器配置中的复选框设置（默认启用）。		false
-Dlsfusion.server.devmodeboolean开发模式。在此模式下：
  • 不启动系统任务（以免干扰调试器）
  • 可以在交互式打印视图中编辑报表设计
  • 启用对API和UI的匿名访问（系统参数 enableAPI、enableUI）。此外，在此模式下，匿名访问是以管理员身份而不是匿名用户身份进行的
  • 客户端在连接丢失时自动重新连接
  • 关闭从资源读取报表的缓存
在IDE中，运行lsFusion服务器配置时自动启用。		false
-Dlsfusion.server.testmodeboolean启用一些实验性功能
如果启用了断言（-ea选项），则自动启用		false

lsFusion​

服务器应用程序的lsFusion启动参数可以通过以下方式之一设置（按优先级顺序，优先级较低的在底部）：

  • 在资源文件lsfusion.xml中使用这些参数的位置（适用于平台分支版本）
  • 在lsfusion.properties中（通常是项目的一部分，这意味着它默认为所有安装生效）
  • 在conf/settings.properties中（对于特定安装）
  • 在Java启动选项中（以D开头，例如-Dlogics.topModule=FFF）
名称类型描述默认值
db.server, db.name, db.user, db.password, db.connectTimeoutstring, string, string, string, int数据库服务器连接参数：
  • db.server - 数据库服务器地址（必要时，在冒号后加上端口，例如localhost:6532）
  • db.name - 数据库名称
  • db.user - 连接到数据库服务器的用户名
  • db.password - 连接到数据库服务器的用户密码
  • db.connectTimeout - DBMS连接超时
localhost, lsfusion, postgres, , 1000
rmi.port, rmi.exportName, http.portint, string, int应用服务器的访问设置：
  • rmi.port - 应用服务器端口（RMI注册/导出对象）
  • rmi.exportName - 应用服务器名称（由其导出的根RMI对象）。如果需要在一个端口上导出多个逻辑，使用此参数是有意义的
  • http.port - 嵌入在应用服务器中的Web服务器端口（用于外部系统访问）
7652, default, 7651
logics.includePaths, logics.excludePaths, logics.topModule, logics.orderDependenciesstring, string, string, string项目参数（加载哪些模块以及加载顺序，详细说明在此）logics.includePaths等于*，其他为空
user.country, user.language, user.timezone, user.twoDigitYearStart（user.setCountry, user.setLanguage, user.setTimezone）string, string, string, int定义区域设置参数（区域设置 - 语言、国家等，详细说明在此）的标准Java参数
由于Java Spring的特性（即即使在启动命令中未明确指定，Java Spring也认为区域设置参数已设置，也就是说，在.properties文件中设置这些参数会被忽略），平台支持这些参数的"克隆"，以set开头：如果指定了它们（无论是在.properties文件中还是在启动字符串中），它们会"重载"原生参数。也就是说，优先级是操作系统、-Duser.*、.properties文件中的User.set*和-Duser.set*（以上都不适用于user.twoDigitYearStart，因为它不是标准Java参数）		前三个由操作系统设置决定，当前年份减去80
db.namingPolicy, db.maxIdLengthstring, int表和字段的命名策略参数：db.namingPolicy - 属性的Java类名称（全名，包含包）；在构造函数中，它必须接受一个类型为int的参数 - 名称的最大大小。内置策略类名称：
  • 带签名的完整名称 - lsfusion.server.physics.dev.id.name.FullDBNamingPolicy
  • 不带签名的完整名称 - lsfusion.server.physics.dev.id.name.NamespaceDBNamingPolicy
  • 简短名称 - lsfusion.server.physics.dev.id.name.ShortDBNamingPolicy
db.maxIdLength - 表或字段名称的最大大小。作为表和字段命名策略Java类构造函数的第一个参数传递。		带签名的完整名称，63
db.denyDropModules, db.denyDropTablesboolean, boolean启动时禁止删除：
  • db.denyDropModules - 模块
  • db.denyDropTables - 表
false, false
logics.initialAdminPasswordstring默认管理员密码

示例conf/settings.properties文件（第3节）：​

$FUSION_DIR$/conf/settings.properties

db.server=localhost
db.name=lsfusion
db.user=postgres
db.password=pswrd

rmi.port=7652
信息

默认情况下，假设启动参数文件conf/settings.properties和lsfusion.properties位于应用服务器的启动文件夹中。但是，在GNU Linux下进行自动安装时，这些文件的符号链接（以及日志文件夹的符号链接）会自动创建到其他文件，这些文件的布局更符合Linux理念。

Web服务器（Client）​

Java​

Java Web服务器启动参数在Tomcat启动命令中设置，Tomcat又会启动该Web服务器（例如，对于自动安装）。

名称类型描述
系统参数（以X开头）标准标准Java参数。最重要的是要注意：
  • Xmx - 最大内存大小。对于复杂逻辑，建议至少分配2GB。

lsFusion​

Web服务器的lsFusion启动参数可以通过以下方式之一设置（按优先级顺序，优先级较低的在底部）：

  • 在Web应用程序的上下文参数中：
    • 在Web应用程序的文件/WEB-INF/web.xml中，context-param标签（适用于平台分支版本）
    • 在Web应用程序的文件/META-INF/context.xml中，Context标签，Parameter标签（适用于平台分支版本）
    • 在Tomcat的文件$CATALINA_BASE/conf/[enginename]/[hostname]/[contextpath].xml中，标签Context，标签Parameter，其中：
      • $CATALINA_BASE$是Tomcat安装的文件夹（例如，对于自动安装，此文件夹为$INSTALL_DIR/Client）
      • [contextpath] - Web应用程序的上下文路径（例如，对于自动安装，此名称默认为空，在Tomcat中等同于名称ROOT；对于手动安装，它取决于war文件的名称），
      • [enginename]和[hostname]是tomcat实现机制和Web服务器计算机的名称（例如，对于自动安装，这些名称分别为catalina和localhost）
    • 在Tomcat的文件$CATALINA_BASE/conf/server.xml中，Context标签，Parameter标签（不推荐）
  • 在URL参数中（例如http://tryonline.lsfusion.org?host=3.3.3.3&port=4444）
名称类型描述默认值
host, port, exportNamestring, int, string应用服务器的连接设置。必须与应用服务器的访问参数匹配。
  • host - 应用服务器地址
  • port - 应用服务器端口。必须与参数rmi.port匹配
  • exportName - 应用服务器名称。必须与参数rmi.exportName匹配
localhost, 7652, default

示例Tomcat配置文件（第3节中的上下文参数）：​

$CATALINA_BASE/conf/[enginename]/[hostname]/ROOT.xml

<?xml version='1.0' encoding='utf-8'?>
<Context>
    <Parameter name="host" value="localhost" override="false"/>
    <Parameter name="port" value="7652" override="false"/>
</Context>
信息

除了启动参数外，平台还具有系统参数，这些参数的设置方式略有不同，主要与平台各组件的进程相关（即它们启动后发生的进程）。
