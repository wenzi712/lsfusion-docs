IMPORT 操作符

IMPORT操作符创建一个动作，将数据从指定文件导入到指定属性（参数）中，或者一般地，导入到指定表单中。

语法​

IMPORT [importFormat] FROM fileExpr importDestination [DO actionOperator [ELSE elseActionOperator]]
IMPORT formName [importFormat] [FROM (fileExpr | (groupId1 = fileExpr1 [, ..., groupIdM = fileExprM])]

importFormat可以通过以下选项之一指定：

JSON [CHARSET charsetStr]
XML [ATTR] [CHARSET charsetStr]
CSV [separator] [HEADER | NOHEADER] [ESCAPE | NOESCAPE] [CHARSET charsetStr]
XLS [HEADER | NOHEADER] [SHEET (sheetExpr | ALL)]
DBF [CHARSET charsetStr]
TABLE

importDestination可以通过以下选项之一指定：

TO [(objClassId1, objClassId2, ..., objClassIdK)] propertyId1 [= columnId1], ..., propertyIdN [= columnIdN] [WHERE whereId]
FIELDS [(objClassId1 objAlias1, objClassId2 objAlias1, ..., objClassIdK objAliasK)] propClassId1 [propAlias1 =] columnId1 [NULL], ..., propClassIdN [propAliasN =] columnIdN [NULL]

描述​

IMPORT操作符创建一个动作，将数据从指定文件导入到指定属性的值中或指定表单中。

如果未指定导入文件的格式，则会根据导入文件的类（或者如果类是FILE，则根据扩展名）自动确定，如下所示：

格式扩展名类
JSONjsonJSONFILE
XMLxmlXMLFILE
CSVcsvCSVFILE
XLSxls 或 xlsxEXCELFILE
DBFdbfDBFFILE
TABLEtableTABLEFILE
信息

第一个传入的文件用于通过其扩展名自动确定平面文件格式

参数​

导入源​

  • fileExpr

    表达式，其值为导入的文件。表达式的值必须是文件类（FILE、RAWFILE、JSONFILE等）的对象。如果在导入表单时未指定此表达式，则默认表达式为System.importFile()。

  • groupId1, ..., groupIdM

    要导入数据的导入表单的对象组名称。简单ID。仅用于从平面格式导入表单。

  • fileExpr1, ..., fileExprM

    表达式，其值是需要为指定对象组导入的文件。每个表达式的值必须是文件类（FILE、RAWFILE、JSONFILE等）的对象。仅用于从平面格式导入表单。对于空对象组，使用名称root。

导入格式​

  • ATTR

    关键字，指定应从元素的属性中读取值。如果未指定，则从子元素中读取。仅适用于从XML导入。

  • separator

    CSV文件中的分隔符。字符串字面量。如果未指定，则默认分隔符为;。

  • HEADER | NOHEADER

    关键字，指定CSV或XLS文件中是否存在标题行（HEADER）或不存在（NOHEADER）。默认为NOHEADER。

    使用NOHEADER选项时：

    • 列名被认为是：A、B、...、Z、AA、...、AE、...
    • 如果找不到列/与目标属性的类型不匹配，则该列的值被视为NULL（在其他导入格式中，平台在这些情况下会抛出错误）。

  • ESCAPE | NOESCAPE

    关键字，指定CSV文件中特殊字符（\r、\n、"（双引号））和指定分隔符separator是否需要转义（ESCAPE）或不需要转义（NOESCAPE）。仅在可以保证指定分隔符不会出现在数据中的情况下使用NOESCAPE才有意义。默认为ESCAPE。

  • SHEET (sheetExpr | ALL)

    指定从Excel文件导入特定工作表的选项。如果未指定该选项，则采用工作表编号1。

    • sheetExpr

      表达式，其值确定从Excel文件导入的工作表编号。表达式的值必须是INTEGER或LONG类。编号从1开始。

    • ALL

      关键字，意味着将从Excel文件的所有工作表导入。

  • CHARSET charsetStr

    指定导入使用的编码的选项。

    • charsetStr

      定义编码的字符串字面量。JSON、CSV的默认值为UTF-8，DBF的默认值为CP1251。对于XML，默认使用标题中指定的编码。如果不存在，则默认使用UTF-8。

  • actionOperator

    上下文相关动作操作符，描述对每个导入记录执行的动作。

  • elseActionOperator

    上下文相关动作操作符，描述如果没有导入记录时执行的动作。导入数据的参数不能用作此动作的参数。

导入目标​

  • formName

    要导入数据的表单名称。复合ID。

  • objClassId1, ..., objClassIdK

    导入对象的类。由类ID指定。K <= 1。默认情况下，假设导入的是一个INTEGER类的对象。

  • objAlias1, ..., objAliasK

    导入对象写入的局部参数的名称。简单ID。K <= 1。默认情况下，假设导入的是一个名为row的对象。

  • propertyId1, ..., propertyIdN

    数据列（字段）导入到的属性ID列表。属性参数及其类必须与导入的对象及其类匹配。

  • columnId1, ..., columnIdN

    源文件中数据将导入到相应属性的列ID列表。列表中的每个元素由简单ID或字符串字面量指定。当指定不存在的列的ID时，或者在没有ID的情况下，与属性对应的列是列表中为前一个属性指定的列之后的列，如果指定了第一个属性，则为第一列。对于DBF文件，列ID不区分大小写。

  • whereId

    属性ID，对于每个导入的对象，将为此属性值的类的默认值写入该属性。属性参数和类必须与导入的对象及其类匹配。如果未指定属性且导入对象的数量大于0，则使用名称为imported且具有导入对象类的属性（例如System.imported[INTEGER]）。

  • propClassId1, ..., propClassIdN

    导入列的内置类名称列表。

  • propAlias1, ..., propAliasN

    数据列（字段）导入到的局部参数的名称。简单ID。如果未指定名称，则源文件中列（字段）的名称将用作参数名称。

  • NULL

    关键字。指定导入期间的NULL值（如果导入格式支持它们）不会被替换为默认值（例如，数字为0，字符串为空字符串等）。

示例​

import()  {

    LOCAL xlsFile = EXCELFILE ();

    LOCAL field1 = BPSTRING[50] (INTEGER);
    LOCAL field2 = BPSTRING[50] (INTEGER);
    LOCAL field3 = BPSTRING[50] (INTEGER);
    LOCAL field4 = BPSTRING[50] (INTEGER);

    LOCAL headField1 = BPSTRING[50] ();
    LOCAL headField2 = BPSTRING[50] ();

    INPUT f = EXCELFILE DO {
        IMPORT XLS SHEET 2 FROM f TO field1 = C, field2, field3 = F, field4 = A;
        IMPORT XLS SHEET ALL FROM f TO field1 = C, field2, field3 = F, field4 = A;

        FOR imported(INTEGER i) DO { // imported属性 - 用于迭代数据的系统属性
            MESSAGE 'field1 value = ' + field1(i);
            MESSAGE 'field2 value = ' + field2(i);
            MESSAGE 'field3 value = ' + field3(i);
            MESSAGE 'field4 value = ' + field4(i);
       }
    }

    LOCAL t = FILE ();
    EXTERNAL SQL 'jdbc:postgresql://localhost/test?user=postgres&password=12345' EXEC 'SELECT x.a,x.b,x.c,x.d FROM orders x WHERE x.id = $1;' PARAMS '4553' TO t;
    // 使用FIELDS选项导入
    IMPORT FROM t() FIELDS INTEGER a, DATE b, BPSTRING[50] c, BPSTRING[50] d DO        
        NEW o = Order {
            number(o) <- a;
            date(o) <- b;
            customer(o) <- c;
            // 查找具有此名称的货币
            currency(o) <- GROUP MAX Currency currency IF name(currency) = d; 
        }


    INPUT f = FILE DO
        IMPORT CSV '*' HEADER CHARSET 'utf-8' FROM f TO field1 = C, field2, field3 = F, field4 = A;
    INPUT f = FILE DO
        IMPORT XML ATTR FROM f TO field1, field2;
    INPUT f = FILE DO
        IMPORT XML ROOT 'element' ATTR FROM f TO field1, field2;
    INPUT f = FILE DO
        IMPORT XML ATTR FROM f TO() headField1, headField2;

    INPUT f = FILE DO
        INPUT memo = FILE DO
            IMPORT DBF MEMO memo FROM f TO field1 = 'DBFField1', field2 = 'DBFField2';
}

date = DATA DATE (INTEGER);
sku = DATA BPSTRING[50] (INTEGER);
price = DATA NUMERIC[14,2] (INTEGER);
order = DATA INTEGER (INTEGER);
FORM import
    OBJECTS o = INTEGER // 订单
    OBJECTS od = INTEGER // 订单项
    PROPERTIES (o) dateOrder = date // 从dateOrder字段导入日期
    PROPERTIES (od) sku = sku, price = price // 从sku和price字段导入产品数量
    FILTERS order(od) = o // 将顶级订单写入order

;

importForm()  {
    INPUT f = FILE DO {
        IMPORT import JSON FROM f;
        SHOW import; // 显示导入的内容

        // 在数据库中创建对象
        FOR DATE date = date(INTEGER io) NEW o = Order DO {
            date(o) <- date;
            FOR order(INTEGER iod) = io NEW od = OrderDetail DO {
                price(od) <- price(iod);
                sku(od) <- GROUP MAX Sku sku IF name(sku) = sku(iod); // 查找具有此名称的sku
            }
        }
    }
}
