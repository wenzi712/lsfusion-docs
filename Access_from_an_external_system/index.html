从外部系统访问

Action API​

该平台允许外部系统使用各种网络协议访问基于 lsFusion 的系统。此类交互的接口是调用具有指定参数的操作，以及在必要时返回某些属性值（无参数）作为结果。假设所有参数和结果对象都是 内置类 的对象。

定义操作​

被调用的操作可以通过以下三种方式之一定义：

  • EXEC – 指定操作的名称。
  • EVAL – 指定 lsFusion 语言的代码。假设此代码包含一个名为 run 的操作声明。这将是被调用的操作。
  • EVAL ACTION – 指定 lsFusion 语言的操作代码。要访问参数，使用特殊字符 $ 和参数编号（从 1 开始）。

协议​

该平台目前支持以下网络协议：

HTTP​

此协议的通信同时支持应用服务器（端口 7651）和同一端口上安装了 Web 客户端的 Web 服务器（如果有）。

URL 格式根据 操作定义 的方法如下：

  • EXEC - http://服务器地址:端口/exec?action=<操作名称>。action 参数必须始终指定。
  • EVAL - http://服务器地址:端口/eval?script=<代码>. 如果未指定 script 参数，则假定代码在第一个 BODY 参数中传递。
  • EVAL ACTION – http://服务器地址:端口/eval/action?script=<操作代码>. 如果未指定 script 参数，则假定代码在第一个 BODY 参数中传递。
参数​

参数可以在请求字符串中传递（通过在字符串末尾附加 &p=<参数值> 之类的构造），也可以在请求体（BODY）中传递。假定 URL 参数（按其在请求中出现的顺序）在 BODY 参数之前被替换为执行的操作。

处理 BODY 参数时，具有来自以下 表 的内容类型的参数被视为文件，并作为文件类（FILE、PDFFILE 等）的对象传递给操作参数。在此过程中，相应的文件扩展名从上述表中获取。如果在表中找不到特定的内容类型，但它以 application 开头，则该参数仍被视为文件，并且文件扩展名从内容类型的右侧部分获取（例如，对于 application/abc 内容类型，它将是 abc）。具有 application/null 内容类型的参数被视为等于 NULL。

内容类型不同于上述类型的 BODY 参数被视为字符串，并在调用时自动转换为被调用操作的参数类。空字符串被转换为 NULL。

已执行请求的 标头 自动保存到 System.headers[TEXT] 属性。标头的名称写入此属性的唯一参数，标头的值写入属性的值。

结果​

其值必须作为结果返回的属性通过在请求字符串末尾添加 &return=<属性名称> 之类的字符串传递。假定指定属性的值按其在请求字符串中出现的顺序返回。默认情况下，如果未指定结果属性，则结果属性是以下 列表 中第一个具有非 NULL 值的属性。

如果请求的结果是文件（FILE、PDFFILE 等），则响应的 内容类型 根据文件扩展名按照以下 表 确定。如果在此表中找不到文件扩展名，则内容类型设置为 application/<文件扩展名>。

在这种情况下，文件扩展名自动确定，类似于 WRITE 操作符。

在上述三种情况下，如果结果值为 NULL，则在内容类型中用 null 字符串（例如，application/null）替换文件扩展名，并返回空字符串作为响应本身。

不同于文件的请求结果被转换为字符串，并作为 text/plain 内容类型传递。NULL 值返回为空字符串。

System.headersTo[TEXT] 属性的值自动写入请求结果的 标头。因此，标头名称从此属性的唯一参数读取，标头值从属性值读取。

BODY 中的多个结果/参数​

如果请求 BODY 的类型为 multipart/* 或 application/x-www-form-urlencoded，它将被拆分为多个部分，每个部分将被视为单独的请求参数。在这种情况下，这些参数的顺序等于请求 BODY 相应部分中的顺序。

同时，如果返回的结果数量多于一个，则会发生以下情况：

  • 如果请求具有 returnmultitype=bodyurl 参数，则传输时的响应内容类型设置为 application/x-www-form-urlencoded，结果被编码，就像它们在请求字符串中传递一样。
  • 否则，传输过程中的响应内容类型设置为 multipart/mixed，结果作为此响应的内部部分传递。
信息

请注意，参数和请求结果的处理与通过 HTTP 协议 访问外部系统 期间的处理大致相似（在这种情况下，参数被处理为结果，反之亦然，结果被处理为参数）。

有状态 API​

上述 API 是 REST API。因此，变更会话 在调用启动时创建，并在调用结束后立即关闭。但是，在某些情况下，这种行为是不可取的，您需要在一段时间内累积更改（例如，当用户输入数据时），这意味着会话必须保存并在会话之间传递。为此，您可以在查询字符串末尾添加以下格式的字符串：&session=<会话 ID>，其中 <会话 ID> 是任何非空字符串。在这种情况下，会话在调用后不会关闭，而是与会话前传递的 ID 相关联，以便所有使用此 ID 的后续调用都将在此会话中执行。为了关闭会话（调用结束后），您需要在请求字符串中的 ID 添加 _close 后缀（例如，&session=0_close）。

信息

由于 cookie 文件隐式用于处理 HTTP 会话，因此重要的是不要忘记在有状态的 http 调用之间保存/传递 cookie（但是，这通常由浏览器、Java 中的 HttpClient 等自动完成）

信息

平台的当前实现假设，如果使用会话，则在当前调用中创建的系统元素（例如，本地属性）会被删除 — 即它们在后续调用中不可见。

认证​

执行 http 请求时，通常需要标识代表其执行指定操作的用户。目前，平台支持两种类型的认证：

  • 基本标识 - 用户名和密码以编码形式在 Authorization: Basic <凭证> 标题中传递。
  • 基于令牌的认证包括两个阶段：
    • 在第一阶段，您需要使用基本认证执行 Authentication.getAuthToken[] 操作。此操作的结果将是具有固定生命周期的认证令牌（默认情况下为一天）。请求示例：http://localhost/exec?action=getAuthToken。
    • 您收到的令牌可以通过在 Authorization: Bearer <token> 标头中传递来在其生命周期内用于认证（类似于平台当前实现中用于生成认证令牌的 JWT）。

Form API​

除了执行操作外，平台还支持用于处理 表单 的 API（类似于 JSON API），或者更具体地说，处理它们的 交互式视图。由于它是专为异步模式设计的有状态 API（这意味着 HTTPS 接口本身具有许多系统参数，例如请求索引、最新接收的响应的索引等），因此使用针对您要集成的特定语言/平台的特殊库来使用此 API 会更容易：

协议​

JavaScript​

JavaScript 库可在中央 npm 存储库中以 @lsfusion/core 名称获取。

此 API 中的关键概念是状态概念。状态是一个 JS 对象，其结构与表单元素的对应关系如下：

  • 对象组 对应于存储在状态对象的 js 字段中的 JS 对象。字段名称与对象组名称匹配。对象组中的每个 JS 对象又在 list 字段中存储 JS 对象数组（考虑了 过滤器 和 顺序）。对象组的 JS 对象对应于 当前 对象集合。此外，数组中的每个 JS 对象（包括对象组的 JS 对象）在 value 字段中存储对象的值 — 如果对象组中只有一个对象，则仅存储值；如果有多个对象，则存储一个 JS 对象，其字段名称等于对象名称，值等于对象值。
  • 属性 对应于存储在 JS 对象字段（字段名称等于属性名称）中的值，该值根据参数的存在和 其视图 以以下方式确定：
    • 属性有参数：
      • 属性视图等于该属性的 显示组 的 JS 对象的 list 数组中的每个 JS 对象的 GRID。
      • 属性的视图等于该属性显示组的 JS 对象的 PANEL、TOOLBAR
    • 属性没有参数 - 属于 JS 状态对象。

库的任务是在表单创建期间和后续修改期间自动保持上述状态的更新（这种行为通常称为反应性）。

该库导出以下函数：

  • create - 创建一个新表单。参数：
    • setState - 状态更改请求函数。此函数应该接受单个参数 — 状态更改函数（该函数又只有一个参数，即先前的状态，并输出下一个状态作为结果），并作为执行的结果将此函数添加到状态更改队列中（或者，例如，根据视图逻辑的实现立即应用它）。这种状态管理逻辑与 React 中的状态管理逻辑完全相同，并且通常，如果在 React 组件内部使用，setState 参数作为 updateState => this.setState(updateState) 传递。
    • baseUrl - lsFusion Web 服务器的 URL - 字符串，例如 'https://demo.lsfusion.org/hockeystats'。
    • formData - 描述表单的对象。必须包含带有表单名称的 name 字段（例如 {name: "MainForm"}）或带有表单代码的 script 字段（例如，script:"OBJECTS i = Invoice PROPERTIES (i) date, stock"）
  • change - 更改表单数据。参数：
    • setState - 状态更改请求函数。
    • changes - 包含需要具体更改内容的 JS 对象。JS 更改对象的结构与 JS 状态对象的结构相同，除了对象组的 JS 对象没有/不应该有 list 字段 — 即，所有更改都应该为当前对象集合进行。但是，如果需要，可以将 value 设置为单元素数组，这意味着不需要更改当前对象，但应该为指定的对象（而不是当前对象）更改属性值。例如，change(setState, {game:{value:[30], hostGoals:40, guestGoals:30}}) 会将进球数更改为 40 和 30，但不是针对当前游戏，而是针对对象 id 为 30 的游戏。您还可以在 JS 更改对象中指定操作（JS 状态对象中没有操作）。在这种情况下，相应字段的值可以是任意的。例如，change(setState, {game: {doSmthWithGame : true}})
    • currentState - 当前状态的 JS 对象。可选参数。为了确保异步模式下的最佳用户体验，建议用户仅更改他在更改期间看到的对象的值，而不是在处理更改时（此时状态可能会更改，当前对象也可能会更改）。因此，在调用此函数时，建议将用于渲染用户在其中发起此更改的视图的状态作为 currentState 参数传递。
  • close - 关闭表单。参数：
    • setState - 状态更改请求函数。
  • formCreated - 检查表单是否已初始化（以及相应地，状态是否已填充）。返回布尔值。参数：
    • state - JS 状态对象
  • numberOfPendingRequests - 显示当前排队的更改请求数量。返回 long 类型的值。参数：
    • state - JS 状态对象

作为对象组和属性的名称，使用的不是表单上的名称，而是 导出/导入 名称（但是，如果未明确定义，则它们与表单上的名称匹配）。通过 Form API 处理表单时，使用 对象操作 运算符 NEW 和 DELETE 创建的操作自动分别获得导出/导入名称 NEW 和 DELETE（例如，您可以调用 change(setState, {game : {NEW:true}}) 来添加对象）。此外，当使用 Form API 时，它会自动向表单添加一个名为 logMessage 的属性，所有对话框消息都写入该属性（包括在违反 约束 时生成的消息）。

信息

认证、有状态和表单 API 仅在 Web 服务器上执行 http 请求时受支持。当应用服务器（或更具体地说，内置 Web 服务器）执行 HTTP 请求时，认证标头以及带有会话 ID 的参数会被忽略（用户被视为匿名）。内置 Web 服务器完全不支持 Form API。

示例​

Action API（Python）​

import json
import requests
from requests_toolbelt.multipart import decoder

lsfCode = ("run(INTEGER no, DATE date, FILE detail) {\n"
           "    NEW o = FOrder {\n"
           "        no(o) <- no;\n"
           "        date(o) <- date;\n"
           "        LOCAL detailId = INTEGER (INTEGER);\n"
           "        LOCAL detailQuantity = INTEGER (INTEGER);\n"
           "        IMPORT JSON FROM detail TO detailId, detailQuantity;\n"
           "        FOR imported(INTEGER i) DO {\n"
           "            NEW od = FOrderDetail {\n"
           "                id(od) <- detailId(i);\n"
           "                quantity(od) <- detailQuantity(i);\n"
           "                price(od) <- 5;\n"
           "                order(od) <- o;\n"
           "            }\n"
           "        }\n"
           "        APPLY;\n"
           "        EXPORT JSON FROM price = price(FOrderDetail od), id = id(od) WHERE order(od) == o;\n"
           "        EXPORT FROM orderPrice(o), exportFile();\n"
           "    }\n"
           "}")

order_no = 354
order_date = '10.10.2017'
order_details = [dict(id=1, quantity=10),
                 dict(id=2, quantity=15),
                 dict(id=5, quantity=4),
                 dict(id=10, quantity=18),
                 dict(id=11, quantity=1),
                 dict(id=12, quantity=3)]

order_json = json.dumps(order_details)

url = 'http://localhost:7651/eval'
payload = {'script': lsfCode, 'no': str(order_no), 'date': order_date,
           'detail': ('order.json', order_json, 'text/json')}

response = requests.post(url, files=payload)
multipart_data = decoder.MultipartDecoder.from_response(response)

sum_part, json_part = multipart_data.parts
sum = int(sum_part.text)
data = json.loads(json_part.text)

##############################################################

print(sum)
for item in data:
    print('{0:3}: price {1}'.format(int(item['id']), int(item['price'])))

##############################################################
# 205
#   4: price 5
#  18: price 5
#   3: price 5
#   1: price 5
#  10: price 5
#  15: price 5
