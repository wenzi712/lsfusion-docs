动作选项

在ACTION语句中声明动作时，可以在声明末尾指定一组动作选项。

语法​

选项按任意顺序一个接一个列出，以空格或换行符分隔：

propertyOption1 ... propertyOptionN

支持以下选项集（每个选项的语法单独列出）：

IN groupName
viewType
ON eventType { actionOperator }
CHANGEKEY key [SHOW | HIDE]
CHANGEMOUSE key [SHOW | HIDE]
STICKY | NOSTICKY
syncType
ASON eventType [ACTION] propertyId
imageSetting
annotationSetting
CONFIRM
EXTID extID

描述和参数​

  • IN groupName

    指定动作所属的属性和动作组。如果未指定此选项，则动作默认属于System.private组。

    • groupName

      组名称。复合ID。

交互式视图块​

  • viewType

    指定动作添加到表单时的动作视图类型。

    • GRID - 表格列
    • TOOLBAR - 工具栏
    • PANEL - 面板

    这类似于在FORM语句的属性块中指定viewType选项。因此，如果此选项既未在动作选项中指定，也未在表单上的属性块中直接指定，则使用表单上动作显示组的默认视图。

  • ON eventType { actionOperator }

    指定一个动作，该动作将作为此动作所有交互视图的特定事件的默认处理程序。可以在FORM语句的属性块中覆盖。

    • eventType

      表单事件类型。由以下关键字之一指定：

      • CHANGE - 当用户尝试更改指定属性的值时发生。
      • CHANGEWYS - 当用户尝试使用特殊输入机制更改指定属性的值时发生。您可以在表单事件的描述中阅读更多内容。
      • GROUPCHANGE - 当用户尝试更改表格中所有对象的属性值（组编辑）时发生。
      • EDIT - 当用户尝试编辑作为指定属性值的对象时发生。
      • CONTEXTMENU [caption] - 用户在表单上的属性上下文菜单中选择了指定项。如有必要，您还可以定义此菜单项的标题（字符串字面量）。如果未指定，则默认与动作标题相同。
      • KEYPRESS key - 用户按下了键盘上的指定key（字符串字面量）。

    • actionOperator

      上下文相关动作运算符。定义在事件上执行的动作的运算符。您可以使用属性本身的参数作为运算符参数。

  • ASON eventType [ACTION] propertyId

    指定此动作将作为指定属性或动作的所有交互视图的特定事件的默认处理程序。可以在FORM语句的属性和动作块中覆盖。

    • eventType

      类似于ON块。

    • propertyId

      属性或动作的ID，当指定的表单事件发生时，将执行为此创建的动作。

    • ACTION

      关键字。如果指定，则认为在propertyId中设置了动作。如果未指定，首先认为在propertyId中定义了属性；否则，如果未找到属性，则认为在propertyId中指定了动作。

  • imageSetting

    动作的图标设置。此选项允许您手动配置图标。它可以有以下形式之一：

    • IMAGE [imageLiteral]

      动作的手动图标指定。如果未提供imageLiteral，则启用自动分配模式。

      • imageLiteral

        字符串字面量，其值定义图标。

    • NOIMAGE

      关键字，表示动作不应有图标。

  • annotationSetting

动作注释。以@@开头。支持以下注释：

  • @@noauth

    禁用对此动作的外部请求的授权检查。

  • @@api

    当API被禁用时，允许对此动作的外部请求。

  • @@deprecated

  • @@deprecated(since)

  • @@deprecated(since, message)

    将动作标记为已弃用，不建议使用。 插件将此类属性显示为删除线。

    • since 字符串字面量，指示动作被视为弃用的平台版本。

    • message 字符串字面量，提供为什么动作被标记为弃用的解释。

  • EXTID extID

    指定用于从外部系统访问的名称。

DESIGN语句默认值块​

  • CHANGEKEY key [SHOW | HIDE]

    指定触发此动作的键盘快捷键。设置默认设计的值，可以在DESIGN语句中覆盖。

    • key

    字符串字面量，定义键盘快捷键。语法：

    keyStroke [;(modeKey=modeValue;)*]

    • keyStroke 键组合的字符串表示。定义原则类似于Java类方法KeyStroke.getKeyStroke(String)中参数的指定方式。(http://docs.oracle.com/javase/7/docs/api/javax/swing/KeyStroke.html#getKeyStroke(java.lang.String))。

    • (modeKey=modeValue;)* 指定keyStroke执行条件的选项。支持以下选项：

      • priority = priorityValue 优先级，整数值。如果多个动作满足CHANGEKEY条件，则执行优先级较高的动作。 如果未设置优先级，则等于表单中属性的序号。此外，在任何情况下，如果对象组匹配，则优先级值会增加1000。

      • preview = previewValue 所有事件都检查执行两次：首先使用isPreview = true，然后使用isPreview = false。支持的previewValue值：

        • AUTO, ONLY -> isPreview
        • NO -> !isPreview
        • ALL -> true

      • dialog = dialogValue 检查CHANGEKEY是否应在对话框窗口中执行。支持的dialogValue值：

        • AUTO, ALL -> true
        • ONLY -> isDialog
        • NO -> !isDialog

      • window = windowValue 检查CHANGEKEY是否应在模态窗口中执行。支持的windowValue值：

        • AUTO, ALL -> true
        • ONLY -> isWindow
        • NO -> !isWindow

      • group = groupValue 检查对象组是否匹配。支持的groupValue值：

        • AUTO, ALL -> true
        • ONLY -> equalGroup
        • NO -> !equalGroup

      • editing = editingValue 检查CHANGEKEY是否应在属性编辑模式下执行。支持的editingValue值：

        • AUTO -> !(isEditing() && getEditElement().isOrHasChild(Element.as(event.getEventTarget())))
        • ALL -> true
        • ONLY -> isEditing
        • NO -> !isEditing

      • showing = showingValue 检查动作当前是否在表单上可见。支持的showingValue值：

        • AUTO, ONLY -> isShowing
        • ALL -> true
        • NO -> !isShowing

      • panel = panelValue 检查动作是否位于面板中。支持的panelValue值：

        • AUTO -> !isMouse || !isPanel
        • ALL -> true
        • ONLY -> isPanel
        • NO -> !isPanel

      • cell = cellValue 检查动作是否位于表格单元格中。支持的cellValue值：

        • AUTO -> !isMouse || isCell
        • ALL -> true
        • ONLY -> isCell
        • NO -> !isCell

      对于priority以外的所有选项，默认值为AUTO。

    • SHOW

      关键字。指定时，键组合将显示在动作标题中。默认使用。

    • HIDE

      关键字。指定时，键组合将不会显示在动作标题中。

  • CHANGEMOUSE key [SHOW | HIDE]

    指定触发此动作的鼠标键组合。设置设计的默认值，可以在DESIGN指令中覆盖。

    • key

    描述鼠标键组合的字符串字面量。语法：

    keyStroke [;(modeKey=modeValue;)*]

    • keyStroke 鼠标键组合的字符串表示。目前，唯一支持的值是DBLCLK——双击。

    • (modeKey=modeValue;)* 语法与CHANGEKEY相同。

    • SHOW

      关键字，表示鼠标键组合应显示在动作标题中。这是默认行为。

    • HIDE

      关键字，表示鼠标键组合不应显示在动作标题中。

  • STICKY | NOSTICKY

    关键字。STICKY表示表格中的动作将固定在左侧，并在向右滚动时保持可见。NOSTICKY删除此固定。默认情况下，STICKY或NOSTICKY是通过启发式方法确定的。

  • syncType

    定义动作是同步执行还是异步执行：

    • WAIT — 同步。

    • NOWAIT — 异步。这是默认行为。

  • CONFIRM

    关键字。如果指定，则在执行动作时会要求用户确认。设置默认设计的值，可以在DESIGN语句中覆盖。
