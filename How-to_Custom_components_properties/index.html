自定义组件（属性）

对于每种属性类型，默认情况下会使用预定义的可视化组件来显示和编辑数据。 但是，您可以使用自己用JavaScript创建的组件来覆盖这些组件。 此功能仅在Web客户端中受支持。

考虑创建一个聊天室用于用户之间通信的任务来演示此功能。

领域逻辑​

首先，让我们创建一个领域逻辑，其中定义了_Message_实体。 每条消息包含纯文本，以及关于作者和发送时间的信息。

CLASS Message '消息';

dateTime '时间' = DATA DATETIME (Message);
text '文本' = DATA TEXT (Message);

author = DATA CustomUser (Message);
nameAuthor '作者' (Message m) = name(author(m));

replyTo = DATA Message (Message);
nameAuthorReplyTo (Message m) = nameAuthor(replyTo(m));
textReplyTo (Message m) = text(replyTo(m));

消息列表显示​

表单上的聊天消息列表将以表格中的行显示，表格只有一列。 对于此列，将更改值显示组件为将用JavaScript编写的组件。 表示值的最简单方法是使用JSON格式字符串，该字符串将存储消息的所有参数。 为了构建此字符串，我们将使用JSON运算符：

json (Message m) = 
    JSON FROM
         author = nameAuthor(m), 
         time = dateTime(m), 
         text = text(m), 
         own = IF author(m) = currentUser() THEN 1 ELSE 0, 
         replyAuthor = nameAuthorReplyTo(m), 
         replyText = textReplyTo(m), 
         replyMessage = replyTo(m);

示例值：

{
    "author":"John Doe",
    "time":"2021-10-05T15:28:05",
    "text":"Hello, Jack!",
    "own":1,
    "replyAuthor":"Jack Smith",
    "replyText":"Hello, John",
    "replyMessage":31302
}

接下来，使用JavaScript和CSS创建一个组件，该组件将在浏览器中显示消息。 该组件将在chat.js文件中创建，该文件将位于_resources/web_文件夹中。

在chat.js文件中，创建_chatMessageRender_函数。它将返回一个由两个函数组成的对象：render_和_update。

_render_函数将一个元素作为输入，需要在该元素内创建显示数据所需的新元素：

render: function (element) { 
    let message = document.createElement("div")
    message.classList.add("chat-message");

    let header = document.createElement("div");
    header.classList.add("chat-header");

    let author = document.createElement("div");
    author.classList.add("chat-author");

    element.author = author;
    header.appendChild(author);

    let replyAction = document.createElement("a");
    replyAction.classList.add("chat-reply-action");

    let replyCaption = document.createTextNode("Reply");
    replyAction.appendChild(replyCaption);

    element.replyAction = replyAction;
    header.appendChild(replyAction);

    message.appendChild(header);

    let replyContent = document.createElement("div");
    replyContent.classList.add("chat-reply-content");

    let replyAuthor = document.createElement("div");
    replyAuthor.classList.add("chat-reply-author");

    element.replyAuthor = replyAuthor;
    replyContent.appendChild(replyAuthor);

    let replyText = document.createElement("div");
    replyText.classList.add("chat-reply-text");

    element.replyText = replyText;
    replyContent.appendChild(replyText);

    element.replyContent = replyContent;
    message.appendChild(replyContent);

    let text = document.createElement("div");
    text.classList.add("chat-text");

    element.text = text;
    message.appendChild(text);

    let time = document.createElement("div");
    time.classList.add("chat-time");

    element.time = time;
    message.appendChild(time);

    element.message = message;
    element.appendChild(message);
}

此代码在_element_内创建并存储每个消息的新_div_，然后将用于绘制消息的各个部分。 结果将是以下组件结构：

为了更新显示的值，平台每次都会调用_update_函数，其中将传递与_render_函数中相同的_element_以及值本身：

update: function (element, controller, value) {
    element.author.innerHTML = value.author || '';

    element.replyAuthor.innerHTML = value.replyAuthor || '';
    element.replyText.innerHTML = value.replyText || '';

    element.time.innerHTML = value.time;
    element.text.innerHTML = value.text || '';

    if (value.own)
        element.message.classList.add('chat-message-own');
    else
        element.message.classList.remove('chat-message-own');
}

此函数将之前描述的_json_属性计算出的JavaScript对象作为_value_参数。 所有字段的值都被写入之前在_render_函数中构建的元素中。

为了将这两个函数组合成一个，创建了一个新的函数_chatMessageRender_，它在同一个对象中返回它们：

function chatMessageRender() {
    return {
        render: function (element) {
            ...
        },
        update: function (element, controller, value) {
            ...
        }
    }
}

为了在浏览器中打开页面时加载创建的js和css文件，必须通过将文件名添加到_onWebClientInit(STRING)_属性来将它们的初始化添加到_onWebClientInit_动作中。需要一个数值来指定加载顺序：

onWebClientInit() + {
    onWebClientInit('chat.js') <- 1;
    onWebClientInit('chat.css') <- 2;
}

处理用户操作​

在这个例子中，我们将处理用户对任何消息的两个操作：点击引用的消息和点击Reply按钮。 在第一种情况下，将转到原始消息，在第二种情况下 - 将消息存储在本地属性中并设置新消息的输入字段的焦点。

传递给_update_函数的_controller_参数用于通知服务器用户做出的事件：

element.replyAction.onclick = function(event) {
    controller.change({ action : 'reply' });
    $(this).closest("div[lsfusion-container='chat']").find(".chat-textarea").focus();
}

element.replyContent.onmousedown = function(event) {
    controller.change({ action : 'goToReply' });
}

点击引用的消息还使用jQuery搜索消息字段并将当前焦点设置到它上面。 具有chat-message-input-area类的DOM元素将在后面创建。

根据用户执行的操作，在控制器上调用_change_方法，该方法将事件信息作为JSON对象发送。 平台将自动将该值传递给定义的动作changeMessage：

replyTo = DATA LOCAL Message ();

changeMessage (Message m) {
    INPUT f = JSON DO
        IMPORT JSON FROM f FIELDS() STRING action DO { // 将文件作为json导入到本地属性中
            IF action = 'goToReply' THEN
                seek(replyTo(m)); // 转到引用的消息
    
            IF action = 'reply' THEN
                replyTo() <- m; // 将当前消息存储在本地属性中
        }
}

此动作读取从JavaScript传递的对象，解析JSON，然后执行相应的操作。

最后，创建一个聊天表单并向其中添加一个包含消息列表的表格。该表格将只有一列，其值将是之前构建的JSON。 关键字CUSTOM指定值将使用之前创建的_chatMessageRender_函数显示。 关键字ON CHANGE后面指定的动作在为选定消息执行_controller.change_方法时调用。

FORM chat '聊天'
    OBJECTS msg = Message LAST
    PROPERTIES(msg) json CUSTOM 'chatMessageRender' ON CHANGE changeMessage(msg)
;

接下来，自定义表单设计，将包含消息列表的表格放在带有标识符_chat_的新容器中，并删除自动创建的不必要组件：

DESIGN chat {
    OBJECTS {
        NEW chat {
            fill = 1; 
            MOVE GRID(msg) {
                headerHeight = 0;
                PROPERTY(json(msg)) {
                    autoSize = TRUE;
                }
            }
            REMOVE BOX(msg);
        }
    }
    REMOVE TOOLBARBOX;       
}

将表单添加到导航器：

NAVIGATOR {
    NEW chat;
}

发送新消息​

还需要向表单添加用户创建新消息的功能。

首先，让我们创建一个_send_动作，该动作将基于本地_message_属性和之前定义的_replyTo_属性在单独的会话中创建新消息：

message = DATA LOCAL TEXT ();

send '发送' () { 
    NEWSESSION NESTED LOCAL {
        NEW m = Message {
            dateTime(m) <- currentDateTime();
            author(m) <- currentUser();
            replyTo(m) <- replyTo();
            text(m) <- message();
            seek(m);
            APPLY;
        }
    }
}

类似于之前描述的_json_属性，我们创建一个新的_jsonInputMessage_属性，该属性将被组件用于输入新消息：

jsonInputMessage () = JSON FROM
            replyAuthor = nameAuthor(replyTo()),
            replyText = text(replyTo()),
            text = message();

接下来，创建一个函数，该函数将生成一个用于显示和输入新消息的组件。 为此，我们将使用带有_contentEditable_属性的_div_元素：

function chatMessageInputRender() {
    return {
        render: function (element) {
        let input = document.createElement("div");
        input.classList.add("chat-message-input");

        let reply = document.createElement("div");
        reply.classList.add("chat-reply");

        let replyContent = document.createElement("div");
        replyContent.classList.add("chat-reply-content");

        let replyAuthor = document.createElement("div");
        replyAuthor.classList.add("chat-reply-author");

        element.replyAuthor = replyAuthor;
        replyContent.appendChild(replyAuthor);

        let replyText = document.createElement("div");
        replyText.classList.add("chat-reply-text");

        element.replyText = replyText;
        replyContent.appendChild(replyText);

        element.replyContent = replyContent;
        reply.appendChild(replyContent);

        let replyRemove = document.createElement("div");
        replyRemove.classList.add("chat-reply-remove");

        element.replyRemove = replyRemove;
        reply.appendChild(replyRemove);

        input.appendChild(reply);

        let text = document.createElement("div");
        text.classList.add("chat-message-input-area");
        text.contentEditable = "true";

        element.text = text;
        input.appendChild(text);

        element.appendChild(input);
    },
    update: function (element, controller, value) {
        if (value !== null) {
            element.replyAuthor.innerHTML = value.replyAuthor || '';
            element.replyText.innerHTML = value.replyText || '';

            element.replyRemove.innerHTML = value.replyAuthor ? '❌' : '';

            element.text.innerHTML = value.text || '';
        }
    }
}

然后我们添加事件处理程序，这些处理程序将删除引用的消息，在按CTRL+ENTER时发送消息， 并在组件失去焦点时将输入的消息写入本地属性。

浏览器端将具有以下JavaScript代码：

element.replyRemove.onclick = function(event) {
    controller.change({ action : 'replyRemove' });
}

element.text.onkeydown = function(event) {
    if (event.keyCode == 10 || event.keyCode == 13)
        if (event.ctrlKey)
            controller.change({ action : 'send', value : element.text.innerHTML })
        else
            event.stopPropagation(); // 按ENTER后停止进一步处理
}

element.text.onblur = function (event) {
    controller.change({ action : 'change', value : element.text.innerHTML });
}

服务器上将接收这些事件的_changeInputMessage_动作：

changeInputMessage () {
    INPUT f = JSON DO
        IMPORT JSON FROM f FIELDS() STRING action, TEXT value DO {
            IF action = 'replyRemove' THEN
                replyTo() <- NULL;
    
            IF action = 'send' THEN {
                message() <- value;
                send();
            }
            
            IF action = 'change' THEN
                message() <- value;
        }
}

根据之前声明的函数和动作向表单添加输入字段，以及一个_Send_按钮：

EXTEND FORM chat
    PROPERTIES jsonInputMessage() CUSTOM 'chatMessageInputRender' ON CHANGE changeInputMessage(), 
               send()
;

更改表单的设计，使输入消息的字段和_Send_按钮位于消息列表下方：

DESIGN chat {
    chat {
        NEW chatMessage {
            horizontal = TRUE;
            alignment = STRETCH;
            MOVE PROPERTY(jsonInputMessage()) {
                fill = 1;
                autoSize = TRUE;
                width = 0;
                caption = '';
            }
            MOVE PROPERTY(send()) { fontSize = 32; alignment = STRETCH; }
        }
    }  
}

通过设置_autoSize_和_width_属性，输入组件将随着消息大小的增长而拉伸。
