搜索

系统中的每个元素都包含某些信息，使其能够在系统的其他元素中被唯一识别。然而，在每个使用元素的地方都提供关于该元素的所有信息相当繁琐；因此，平台有一个特殊的搜索算法，它允许仅使用部分信息（例如，仅使用名称和使用您正在搜索的项目的模块）来找到所需元素。

搜索步骤​

通常，搜索算法由一个或多个搜索步骤组成。这些步骤一个接一个地执行，以便如果在第一步中未找到所需元素，则执行第二步，然后是下一步，依此类推，直到找到所需元素或完成所有步骤（在后一种情况下，平台将抛出元素未找到的错误）。

每个搜索步骤又由几个子步骤组成：

  • 步骤条件 - 一组条件，每个条件都必须满足，该步骤才有意义。此检查在检查系统的任何元素之前完成，并且完全取决于调用的上下文。
  • 选择条件 - 一组条件，每个条件都必须满足才能使元素成为潜在的所需元素。在这一步中，元素彼此独立，此步骤的结果是一个选定的候选集，作为所需元素。
  • 选择操作 - 一系列操作，每个操作都根据候选集中哪个候选元素可能「更好」（即概率更高）适合担任所需元素的角色来过滤候选集。

执行最后一个选择操作后，如果剩余：

  • 恰好一个元素 - 它被视为所需元素。
  • 多个元素 - 平台将抛出无法明确确定所需元素的错误
  • 零个元素 - 平台将转到下一个搜索步骤（如果存在），或者抛出元素未找到的错误。

搜索算法的步骤、条件和操作说明：

名称匹配​

候选元素的名称与搜索名称匹配（搜索时必须始终明确指定）。比较区分大小写（即 aA != AA）。

位于依赖模块中​

候选元素模块依赖于执行搜索的模块。

位于指定命名空间中（如果明确指定）​

候选元素的命名空间等于搜索的命名空间（如果明确指定了该命名空间）。

选择优先命名空间（如果未明确指定命名空间）​

如果当前集合包含来自执行搜索的命名空间中的候选元素，则从候选集合中删除所有其他候选元素，并且选择操作完成。如果没有，则对执行搜索的模块的优先命名空间重复类似操作（按照设置这些优先命名空间的顺序）。如果在这些命名空间中也没有找到，则候选集合保持不变。

参数数量匹配​

候选元素的参数数量等于调用的参数数量。

在操作内搜索属性​

此步骤仅在搜索属性时执行，且仅当此搜索在某些操作内部（而非例如在表单声明中）时执行。

未明确指定命名空间​

仅当未指定搜索命名空间时，才执行此步骤。

位于栈的上层​

候选元素是局部属性，并且：

  • 在执行引用所需元素的操作的序列的操作符内部声明。
  • 在直接引用所需元素的操作之前声明。

参数类匹配​

调用参数的类匹配候选元素参数的类，或者引用参数的类未知。

参数类相交​

引用参数的类相交于候选元素参数的类。

选择更具体的参数类​

如果候选集合包含属性 A 和 B，且 A 的参数类匹配 B 的参数类，则从候选集合中删除属性 B。

不搜索抽象属性​

如果在属性（操作）的扩展操作符中搜索属性（操作），且搜索的属性是可扩展（抽象）属性（在搜索扩展操作符实现的右侧部分中的属性时，使用通用搜索机制），则不执行此步骤。

抽象属性（如果正在搜索抽象属性）​

候选元素是抽象属性（操作）。

选择不匹配的参数类（如果正在搜索抽象属性）​

如果：

  • 搜索的是抽象属性（操作）
  • 未明确指定引用参数类
  • 候选集合包括参数类等于调用参数类的属性（操作）
  • 候选集合包括参数类不等于调用参数类的属性（操作）

则：从候选集合中删除第三部分的属性（操作）（即那些参数类等于引用参数类的属性（操作））

搜索属性/操作的附加操作​

参数类匹配​

如果满足以下条件，则参数类 (A1, ..., An) 匹配参数类 (B1, ..., Bm)：

  • n = m，即参数数量相同
  • 对于每个参数 i，以下情况之一为真：
    • Ai 未知（等于 ?）
    • Bi 未知（等于 ?）
    • Ai 继承自 Bi（更具体地说，Bi 的子类集合包括 Ai 的子类集合）

参数类相交​

如果满足以下条件，则参数类 (A1, ..., An) 相交于参数类 (B1, ..., Bm)：

  • n = m，即参数数量相同
  • 对于每个参数 i，以下情况之一为真：
    • Ai 未知（等于 ?）
    • Bi 未知（等于 ?）
    • Ai 和 Bi 有共同的后代（更具体地说，Bi 的子类集合与 Ai 的子类集合相交）

定义引用参数的类​

当调用属性（操作）时，如果未明确设置参数类，平台会尝试从引用上下文中自动确定它们。以下是可能的上下文的（不完整）列表，以及平台在这些情况下如何确定引用参数类：

上下文调用参数类
组合/调用参数属性值的类
在表单上使用所需属性/操作获取的对象的类
将操作添加到导航器空列表
输入结果，外部访问空列表
简单分区组属性的值的类（BY 块）
会话操作符中的嵌套局部属性未知
数据导入如果导入值列表（LIST），则为空列表，否则为单个 INTEGER 元素的列表

确定属性（操作）的参数类​

如果未明确指定属性（操作）的参数类，平台会尝试从属性（操作）实现中自动确定它们。以下是可能的实现的（不完整）列表，以及平台在这些情况下如何确定属性（操作）参数类：

上下文调用参数类
表达式参数的类（按照使用顺序）
主要、抽象类在操作符本身中明确指定
公式、内部调用类可以在操作符本身中明确指定；如果未指定，所有参数类都被视为未知（?），并且数量由特定操作符的特性决定
分组组属性的值的类（BY 块）
对象组操作属于所用对象组的对象的类

示例​

MODULE ResolveA;

CLASS A;
CLASS B : A;
CLASS C : B;

f = DATA INTEGER (A);
f = DATA INTEGER (C);

META defineSmth(prm1)
    x = DATA INTEGER (prm1);
END

META defineSmth(prm1, prm2)
    x = DATA INTEGER (prm1, prm2);
END
MODULE ResolveB;

REQUIRE ResolveA;

f = DATA INTEGER (B);

h(C c) = f(c); // 将找到上层声明 - ResolveB.f[B]
j(C c) = ResolveA.f(c); // 将找到 ResolveA 中的声明 - ResolveA.f[C]
z(C c) = f[A](c); // 将找到 ResolveA 中的声明 - ResolveA.f[A]

test(C c, A a) {
    LOCAL f = INTEGER (B);

    f(c) <- 1; // 将找到上层声明 - f[B]
    MESSAGE f(a); // 将找到上层声明 - f[B]
    ResolveB.f(c) <- 1; // 将找到 ResolveB 中的上层声明 - ResolveB.f[B]
}
MODULE ResolveC;

REQUIRE ResolveB, ResolveA;

NAMESPACE ResolveA;

x(B b) = f(b); // 将找到 ResolveA 中的声明 - ResolveA.f[A]
y(B b) = ResolveB.f(b); // 将找到 ResolveA 中的声明 - ResolveB.f[B]

@defineSmth(A, B); // 将找到 ResolveA 中的声明 - ResolveA.defineSmth(prm1, prm2)
