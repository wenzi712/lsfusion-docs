表

为了存储和计算属性的值，lsFusion平台使用关系型数据库。所有数据属性以及所有标记为物化的计算属性都存储在数据库中表的字段中。对于每个表，都有一组键字段，名称为key0、key1、...、keyN，用于存储对象ID。所有其他字段存储属性值，使得在每行的相应字段中都有键字段中ID对应的对象的属性值。内置类的对象也可以用作表键。

创建表时，必须指定将作为此表中键的对象类列表。

属性表确定​

对于每个属性，可以指定它应该存储在哪个表中。在这种情况下，表键的数量必须等于属性参数的数量，并且参数类必须与表键类匹配。如果未明确设置应存储属性的表，则属性将自动放置到系统中"最近"的现有表中（即键数量与属性参数数量匹配且键类与参数类最接近的表）。如有必要，还可以使用特殊选项（NODEFAULT）指定在自动确定属性表时应忽略此表（即属性只能通过相应的选项（TABLE）显式放入此类表中）。

表命名​

对于平台中创建的每个表，都会在数据库中创建相应的表，其名称根据所选的命名策略定义如下：

命名策略字段名
完整带签名（默认）NameSpace_TableName
完整不带签名NameSpace_TableName
简短TableName

命名策略使用db.namingPolicy启动参数定义。

字段命名​

每个属性的值始终精确存储在一个字段中，其名称根据所选的命名策略定义如下：

命名策略字段名
完整带签名（默认）NameSpace_PropertyName_ClassName1_ClassName2_..,ClassNameN
完整不带签名NameSpace_PropertyName
简短PropertyName

如有必要，开发人员可以为每个属性明确指定存储该属性的字段名称。此外，如果上述方式由于某种原因不适合，还可以创建自定义的属性字段命名策略。

信息

使用过短的属性命名策略（在物化属性数量足够大的情况下）可能会显著复杂化这些属性的命名（保持其唯一性），或者相应地导致需要频繁显式命名存储这些属性的字段的情况。

命名策略使用db.namingPolicy启动参数定义。

默认表​

如果系统无法确定应放置属性的表，则会自动创建一个名称为auto_<属性签名中的类ID 1>_<属性签名中的类ID 2>_...<属性签名中的类ID n>的表。例如，对于带有类参数DATE、Item.Item、Country.Country、INTEGER的属性，将创建表auto_DATE_Item_Item_Country_Country_INTEGER。但是，建议避免使用默认表的情况，并明确指定将存储属性的表。

此外，如果基本策略由于某种原因不适合，还可以创建自定义的平台表命名策略。

默认索引​

默认情况下，为每个表的键字段key0、key1、...、keyN构建一个唯一索引，命名为pk_<表ID>，其中N是表中键字段数量减1。此外，还会为从1到N的所有K自动添加键字段keyK、...、keyN上的索引，命名为<表ID>_keyK_..._keyN_idx。

完整表​

如果一个表对于其每个键都包含系统中存在的该键类的所有对象，则称该表为完整的。通常，表的完整性是通过特殊选项（FULL）明确指定的，并通过隐式创建物化的分类属性（我们称之为完整性属性）来实现。但是，在某些情况下，如果平台确定表已经有保证包含所有必要对象的属性（例如，属于类的属性），则可能不会创建此属性。

默认情况下，完整性属性命名为_FULL_TableName。此外，在定义此属性的字段名称时，使用简短命名策略（因为表中只能有一个这样的字段，没有必要创建冗长的名称）。

值得注意的是，表的完整性仅从执行优化的角度来看很重要（例如，服务器知道某个表具有所需类的所有对象，并且使用INNER JOIN而不是LEFT JOIN，这在某些情况下可能至关重要），并且与表机制本身一样，它不影响系统的逻辑。

存储对类的归属​

对类的归属是类似于数据属性的基本数据。因此，与数据属性一样，此归属存储在某个表的字段中。同时，此表的确定方式类似于其他物化属性的表确定方式（假设对类的归属是一个具有此类一个参数的属性），唯一的区别是首先只处理完整表（即它们具有更高的优先级），只有在没有找到完整表的情况下，才处理其余（不完整）的表。

如果某个表对于其每个键都存储了该键类的所有后代对该类的归属，那么首先它会自动标记为完整的（即使没有明确指定），其次不会为此创建完整性属性（假设该属性的角色由对类的归属属性本身履行）。

默认情况下，对类的归属属性命名为_CLASS_TableName。此外，在确定此属性的字段名称时，使用简短命名策略（因为表中只能有一个这样的字段，没有必要创建冗长的名称）。

语言​

要创建表，请使用TABLE语句。要指定应存储属性的表，请在属性选项中使用TABLE选项。

示例​

TABLE book (Book);

in = DATA BOOLEAN (Sku, Stock);
TABLE skuStock (Sku, Stock); // 它将存储in属性

price = DATA NUMERIC[10,2] (Sku, DATE);
TABLE skuDate (Sku, DATE); // 它将存储Sku属性

TABLE sku (Sku) FULL;
