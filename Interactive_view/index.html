交互式视图

以交互式模式打开的表单是一个具有特定设计的图形组件，用户可以在其中触发各种事件，从而导航系统对象，查看和更改属性值，执行动作等。开发人员还可以使用此视图的附加运算符集，从而管理打开的表单。

对象视图​

交互式视图，对象组可以在表格中显示。表格中的行是对象集合，列是属性。表格中显示的记录及其顺序由当前的过滤器和排序确定。

对象的当前值可以通过使用特殊搜索运算符(SEEK)创建的动作来更改，或者如果对象组显示在表格中，则可以通过更改当前行来更改。

当对象组显示在表格中时，显示的行数（对象集合）可以根据表格可见部分的高度自动确定，或者由开发人员在创建表单时明确指定。

对象树​

平台还允许在一个表格中同时显示多个对象组。这类似于静态视图中的对象组层次结构，即如果我们有两个组A和B，那么在"合并"表格中，首先显示来自A的第一个对象集合，然后显示来自B的所有对象集合（经过过滤），然后是来自A的第二个对象集合，然后再次显示来自B的所有对象集合，依此类推。在这种情况下，B的过滤器最好使用来自A的所有对象，否则将这些组合并到单个树中没有意义。最初，当表单在表格中打开时，只显示最顶层对象组的对象，但同时，在表格左侧会创建一个特殊列，用户可以使用它自行打开节点，从而只查看较低对象组中感兴趣的对象。此创建列的另一个功能是通过在列内对元素进行制表来展示节点的嵌套关系（这使用户可以更好地理解他当前处于层次结构的哪个级别）。

对象树还可用于显示层次数据（如分类器）。在这种情况下，树中组的对象集合的后代不仅可以是较低组的对象集合，还可以是同一组的对象集合（这种对象组称为层次化对象组）。要确定层次化对象组中的这些子对象集合，需要为其定义一个额外的过滤器——与常规过滤器不同，它不仅可以引用被过滤对象集合的值，还可以引用"树中上层"对象集合的值（相同的方法用于递归运算符）。层次过滤器最好使用上层对象集合的所有值，否则，与不同对象组之间的过滤器一样，创建这样的树没有意义。最初，假设"树中上层"对象集合的所有值都是NULL。

信息

在当前平台实现中，层次组仅允许显示树（而不是有向图）。因此，层次过滤器只允许使用上层对象集合的值和接受下层（过滤后）对象值作为输入的属性（以确保无法通过不同方式到达同一树节点）

树中不同对象组的属性按列排列，即第一列显示每个对象组的第一个属性，第二列显示第二个属性，依此类推。树列的总数由树中最后一个对象组确定（上层组的所有"额外"属性都被简单地忽略）。

属性视图​

任何属性或动作都可以在表单上以以下视图之一显示：

  • 面板 (PANEL)：显示属性标题和该属性在表单对象当前值下的值的单独组件。
  • 工具栏 (TOOLBAR)：类似于面板，但此组件具有不同的默认位置（紧邻表格下方），并且如果工具栏所属的表格被隐藏，则工具栏也会被隐藏。
  • 表格列 (GRID)：表格中的单独列，显示表格中所有对象集合（行）的属性值。

对于每个对象组，您可以指定该组的属性将以哪种默认视图显示（默认情况下，此视图是表格列）。如果属性没有参数（即它没有显示组），则它以面板形式显示。动作默认始终以面板形式显示。

信息

在本节的其余部分，属性和动作的行为完全相同，因此我们将仅使用属性一词（行为对动作完全相同）。

如有必要，开发人员可以明确指定属性应使用的视图。

如果在任何时候对象组在表格中没有显示的属性，则表格会自动隐藏。

默认情况下，表单上每个属性的标题是属性本身的标题。如有必要，开发人员可以指定不同的标题，或者，如果需要更大的灵活性，可以使用属性作为标题。此标题属性可以接收所显示属性的上层对象作为输入。还值得注意的是，如果为属性定义了列内分组，则最好为创建的列设置不同的标题（以便以某种方式区分它们）：在这种情况下，建议使用接收所有（！）已定义列内分组对象作为输入的属性。

除了标题外，还可以为表单上的每个属性视图定义颜色（背景颜色和文本颜色），以及属性显示所需满足的条件。与标题一样，每个这些参数都是使用某个属性定义的。

过滤器组​

为了向用户提供选择要应用的过滤器的界面，可以将它们组合成过滤器组。对于每个这样的组，将在表单上创建一个特殊组件：用户可以使用它从组中选择一个过滤器作为当前活动过滤器。如果一个组中的多个过滤器应用于不同的对象组，则该组件将显示在最后一个对象组上。

开发人员可以为每个过滤器组指定一个名称，以便将来访问它（例如，在表单设计中）。

自定义过滤器/排序​

用户可以使用相应的界面更改现有排序或添加自己的排序，以及添加自己的过滤器：

  • 排序 - 通过双击列标题。
  • 过滤器 - 通过使用表格下方每个对象组的相应按钮。默认情况下，过滤器设置为表格中的活动属性，并根据输入值进行相等性过滤（对于所有类型，除了不区分大小写的字符串类型，其中过滤器设置为包含输入的字符串）。如有必要，开发人员可以通过使用相应的选项来明确指定默认过滤类型。

默认对象选择​

在交互式表单视图中，对象组过滤器可能会因各种用户操作而改变（例如，更改这些过滤器的上层对象，在过滤器组中选择过滤器等），之后当前对象可能不再满足新过滤器的条件。此外，当打开表单时，某些对象可能未被传递或传递为等于NULL。在这两种情况下，都需要将当前对象更改为某些当前默认对象。平台提供了几种选择新当前对象的选项：

  • 第一个 (FIRST) - 第一个对象集合（按照当前顺序）
  • 最后一个 (LAST) - 最后一个对象集合。
  • 前一个 (PREV) - 前一个对象集合（或尽可能接近它的对象）。
  • 未定义 (NULL) - NULL值集合。

如果未明确指定这些选项中的任何一个，平台将尝试确定对象组中的永久过滤器是否：a）对于上层对象的不同值互斥（如果有），和/或 b）过滤器选择了指定类的对象总数中很小百分比的对象。在这两种情况下，搜索前一个对象都没有意义，默认情况下选择第一个对象（FIRST）；在所有其他情况下，选择前一个对象（PREV）。

信息

值得注意的是，默认对象选择与对象搜索操作非常相似，其中搜索对象为：

  • 对于类型PREV
    • 打开表单时：传递的对象，或者如果没有，则表单对象类的最后使用的对象。
    • 在其他情况下：前一个当前对象值
  • 对于其他类型
    • 打开表单时 - 传递的对象
    • 在其他情况下 - 空对象集合

搜索方向由对象的默认类型确定（此处PREV等同于FIRST）。

对象运算符​

在向表单添加属性时，可以使用预定义的运算符集来实现最常见的对象操作场景，而不是使用特定属性（从而避免每次都需要在表单外部创建和命名这些属性）：

  • 对象值 (VALUE) - 对于内置类的表单对象，将添加一个带有一个参数的特殊属性，该属性显示当前对象值并允许用户更改它。对于自定义类，将添加一个显示数据库中对象ID的属性；当您尝试更改它时，它会显示一个包含该类对象列表的对话框。所选值将用作表单上对象的当前值。
  • 创建对象 (NEW) - 添加一个无参数的动作，该动作创建一个传递的表单对象类（或开发人员明确指定的类）的对象，然后自动使其成为当前对象。如果该类有后代，将向用户显示一个对话框，他可以在其中选择特定的子类。如果对创建对象的表单对象应用了任何过滤器，系统将尝试更改新创建对象的属性，使其满足这些过滤条件（通常，对于创建的对象，每个过滤器值类的默认值都被写入该过滤器）
  • 编辑对象 (EDIT) - 添加一个带有一个参数的动作，该动作调用System.formEdit动作（该动作依次为被编辑对象类打开默认的编辑表单）。
  • 创建和编辑对象 (NEWEDIT) - 添加一个无参数的动作，该动作创建一个表单对象类的对象，调用编辑对象动作（EDIT），如果输入未被取消，则将添加的对象设置为当前对象。
  • 删除对象 (DELETE) - 添加一个带有一个参数的动作，该动作删除当前对象。

您还可以为最后四个运算符指定选项（对所有其他动作忽略）：

  • 新会话 (NEWSESSION) - 在这种情况下，添加到表单的动作将在新会话中执行。在新会话中打开表单时，重要的是要记住当前会话（表单）中所做的更改将不可见。因此，仅当表单是从用户无法更改任何内容的表单打开，或者两个表单的属性和动作完全不相交时，才建议使用此机制。请注意，当在新会话中使用运算符创建新对象（NEW）时，不仅会创建对象，还会编辑它（NEWEDIT）（否则，会话会立即关闭，您的更改将丢失）。
  • 嵌套会话 (NESTEDSESSION) - 动作将在新的嵌套会话中执行。与新会话一样，NEW被替换为NEWEDIT。

选择/编辑表单​

对于每个表单，您可以指定它是查看/编辑给定类对象的默认表单。在这种情况下，当您调用使用对象操作运算符（创建/编辑对象）创建的动作时，将打开此表单。当在表单打开运算符中使用相应的表单选择选项时，也将打开相同的表单。

如果未为类定义列表/编辑表单，平台将自动创建一个。此表单将包含类的一个对象，以及与该类匹配并属于System.base属性组的所有属性。此外，创建、编辑和删除对象的动作将在新会话中自动添加到表单，以及对象值属性（如果没有与对象类对应的System.id属性组的属性添加到表单中，即对象的"ID"未添加到表单中）。

会话所有者​

由于表单默认在当前会话中打开，因此应用/取消对此会话的更改可能并不总是安全的：例如，在其他表单中所做的更改可能会意外应用。为避免此类情况，平台引入了会话所有者的概念——一个负责管理会话生命周期（例如，应用/取消更改）的表单。默认情况下，如果在表单打开时会话没有其他所有者，则认为该表单是会话所有者。

为了实现与会话所有者一起工作的机制，平台使用了一个名为System.sessionOwners的数值本地属性。相应地，打开表单时此属性增加1，关闭表单时减少1。因此，它显示"表单打开堆栈"的嵌套深度，如果会话没有所有者则为NULL，否则不为NULL。

如有必要，开发人员可以在打开表单时明确指定该表单是其使用的会话的所有者。

信息

会话所有权仅影响用于管理表单/会话生命周期的系统动作的显示/行为。使用其余动作时，建议开发人员自己考虑应用"错误"更改的风险（例如，使用上面提到的System.sessionOwners属性）。

表单/会话生命周期管理的系统动作​

以下系统动作会自动添加到任何表单（括号中指定了它们的名称）：

  • 刷新 (System.formRefresh) - 更新表单的当前状态，从数据库重新读取所有信息。
  • 保存 (System.formApply) - 将表单上所做的更改保存到数据库。
  • 取消 (System.formCancel) - 取消在表单上所做的所有更改。
  • 确定 (System.formOk) - 关闭当前表单，如果表单是会话所有者，则将更改应用到数据库。
  • 关闭 (System.formClose) - 关闭当前表单，不处理更改。
  • 放弃 (System.formDrop) - 关闭当前表单并返回NULL作为选定对象。

默认情况下，这些系统动作具有以下可见性条件：

动作条件
刷新始终
保存、取消如果表单是所有者并且可以在表单上调用更改当前会话的动作。如果平台确定取消更改肯定会导致表单对象的初始值发生变化（即选择其他对象），则可能不会显示取消。
确定、关闭如果表单是同步打开的
放弃如果表单是同步打开的，返回值并允许传递NULL值

如有必要，可以通过从表单设计中删除相应组件和/或使用打开表单运算符中的相应选项来显示/隐藏所有这些动作。

附加功能​

您可以指定将显示为表单图标的图像文件。

此外，如有必要，您可以为表单启用自动更新模式：System.formRefresh动作将以指定的间隔为表单执行。

语言​

所有上述选项以及定义表单结构都可以使用FORM语句完成。

打开表单​

要交互式视图显示表单，在交互式视图中使用相应的打开表单运算符。

示例​

date = DATA DATE (Order);
FORM showForm
    OBJECTS dateFrom = DATE, dateTo = DATE PANEL
    PROPERTIES VALUE(dateFrom), VALUE(dateTo)

    OBJECTS o = Order
    FILTERS date(o) >= dateFrom, date(o) <= dateTo
;

testShow ()  {
    SHOW showForm OBJECTS dateFrom = 2010_01_01, dateTo = 2010_12_31;

    NEWSESSION {
        NEW s = Sku {
            SHOW sku OBJECTS s = s FLOAT;
        }
    }
}
FORM selectSku
    OBJECTS s = Sku
    PROPERTIES(s) id
;

testDialog  {
    DIALOG selectSku OBJECTS s INPUT DO {
        MESSAGE 'Selected sku : ' + id(s);
    }
}

sku = DATA Sku (OrderDetail);
idSku (OrderDetail d) = id(sku(d));

changeSku (OrderDetail d)  {
    DIALOG selectSku OBJECTS s = sku(d) CHANGE;
