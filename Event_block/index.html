事件块

FORM语句的事件块 - 一组控制交互式表单视图中事件的结构。

语法​

EVENTS formEventDecl1, ..., formEventDeclN

其中每个formEventDecli具有以下语法：

ON eventType eventActionId(param1, ..., paramK) | { eventActionOperator }

说明​

事件块允许定义表单事件的处理程序，这些事件是由于某些用户操作而发生的。每个块可以有任意数量的以逗号分隔的事件处理程序。如果为一个事件定义了多个处理程序，则保证它们按照定义的顺序执行。

参数​

  • eventType

    表单事件类型。它使用以下关键字之一指定：

    • INIT
    • OK
    • OK BEFORE
    • OK AFTER
    • APPLY
    • APPLY BEFORE
    • APPLY AFTER
    • CANCEL
    • CLOSE
    • DROP
    • CHANGE objName – 指定当对象objName更改时执行操作。
    • QUERYOK
    • QUERYCANCEL
    • EXPAND componentSelector - 指定在展开componentSelector容器后执行操作。
    • COLLAPSE componentSelector - 指定在折叠componentSelector容器后执行操作。
    • TAB componentSelector - 指定在componentSelector选项卡变为活动状态后执行操作。
    • SCHEDULE PERIOD intPeriod [FIXED] - 创建一个调度程序，每隔intPeriod秒执行一次操作。FIXED表示下一个操作的时间段从当前操作开始计算。默认情况下，时间段从当前操作结束计算。

  • eventActionId

    操作ID，将作为事件处理程序。

  • param1, ..., paramK

    操作参数列表。每个参数使用表单上的对象名称指定。对象名称又使用简单ID指定。

  • actionOperator

    上下文相关操作操作符。您可以使用表单上已声明对象的名称作为参数。

示例​

showImpossibleMessage()  { MESSAGE '这不可能'; };

posted = DATA BOOLEAN (Invoice);

FORM invoice '发票' // 创建一个编辑发票的表单
    OBJECTS i = Invoice PANEL // 创建一个发票类的对象

//    ... 设置表单的其余行为

    EVENTS
        // 指定当用户点击OK时，应执行一个操作
        // 该操作将执行"提交"此发票的操作
        ON OK { posted(i) <- TRUE; },
 
        // 点击formDrop按钮时，显示消息表明这不可能，
        // 因为默认情况下该按钮只会在选择发票的表单中显示，
        // 而此表单基本上是发票编辑表单
        ON DROP showImpossibleMessage() 
;

CLASS Shift;
currentShift = DATA Shift();

CLASS Cashier;
currentCashier = DATA Cashier();

CLASS Receipt;
shift = DATA Shift (Receipt);
cashier = DATA Cashier (Receipt);

FORM POS '销售点' // 声明销售室中向客户销售产品的表单

    OBJECTS r = Receipt PANEL // 添加一个对象，用于存储当前收据

    //  ... 声明表单行为
;

createReceipt ()  {
    NEW r = Receipt {
        shift(r) <- currentShift();
        cashier(r) <- currentCashier();

        SEEK POS.r = r;
    }
}

// 通过表单扩展添加属性，以便可以将SEEK应用于已创建的
// 表单上的对象
EXTEND FORM POS 
    EVENTS
        // 打开表单时，执行创建新收据的操作，
        // 该操作填充班次、收银员和其他信息
        ON INIT createReceipt()
        // 每60秒应用一次
        ON SCHEDULE PERIOD 60 FIXED apply(); 
;
