生产环境部署

安装​

除了安装lsFusion外，这些安装程序/脚本还会安装OpenJDK、PostgreSQL和Tomcat。Tomcat嵌入在lsFusion客户端安装中，而OpenJDK和PostgreSQL则单独安装（特别是在单独的文件夹中）。

  • Windows
  • Linux

可执行exe文件： lsFusion 6.1（OpenJDK 21.0.8，PostgreSQL 17.6，Tomcat 9.0.110，IntelliJ IDEA Community Edition 2025.3.2）

  • x64

  • 旧版本
    • lsFusion 5.1 服务器和客户端
      • x64
    • lsFusion 4.1 服务器和客户端
      • x32
      • x64
    • lsFusion 3.1 服务器和客户端
      • x32
      • x64
    • lsFusion 2.4 服务器和客户端
      • x32
      • x64

后续，$INSTALL_DIR$指的是安装lsFusion时选择的文件夹（默认情况下为Program Files/lsFusion <版本>）。同时假设所有参数（端口、Web上下文名称）都保持默认值。

使用dnf/apt的Bash脚本（次要版本使用最新稳定版本）：

lsFusion 6 服务器和客户端（+ OpenJDK（default-jdk / java-openjdk），PostgreSQL 17，Tomcat 9.0.104）：

操作系统命令 / 脚本
RHEL 8+ / CentOS 8+ / Fedora 35+source <(curl -s https://download.lsfusion.org/dnf/install-lsfusion6)
Ubuntu 18+ / Debian 9+source <(curl -s https://download.lsfusion.org/apt/install-lsfusion6)

安装后​

端口​

安装完成后，以下服务将默认在本地计算机上安装并作为服务启动：

  • 数据库服务器（PostgreSQL）在端口5432上
  • 应用服务器（Server）在端口7652上
  • Web服务器（Client）在端口8080上

安装/更新应用程序​

要将开发的逻辑上传到已安装的应用服务器（Server），您必须：

将使用lsFusion语言开发的模块作为扩展名为lsf的文件放置在服务器类路径中的文件夹中（自动安装的默认值，见下文）。此外，如果有其他资源文件（例如报表文件、编译的Java文件、图像等）也必须放置在那里。这些文件可以放在类路径的子文件夹中，也可以放在jar文件（扩展名为jar的zip归档文件）中。复制完所有文件后，您需要重启服务器。

信息

通常将所有项目文件放在单个jar文件中很方便。要自动生成这样的文件，您可以使用Maven（使用assemble和noserver配置文件）或IDE中内置的构建工具。

默认情况下，服务器的类路径等于$APP_DIR$;$APP_DIR$/*;server.jar，即$APP_DIR$文件夹及其所有子文件夹，$APP_DIR$文件夹中的所有jar文件（但不包括其子文件夹），以及应用服务器本身的jar文件。

  • Windows
  • Linux

$APP_DIR$等于$INSTALL_DIR$/lib

$APP_DIR$等于/var/lib/lsfusion

应用服务器在自动创建的非特权用户lsfusion下安装和启动，因此该用户应该可以读取文件夹中的文件。

安装/更新客户端​

要让用户访问已安装的系统，您必须：

向用户发送链接http://<web服务器（Client）的Web地址>:8080。当用户打开此链接时，他们将默认被重定向到登录页面，必要时可以通过Java Web Start安装桌面客户端（需要安装Java（JDK），例如，通过这个需要注册的链接或这个无需注册的链接）。Web和桌面客户端会随着Web服务器的更新（Client）自动更新

信息

在Windows下，您还可以使用桌面客户端安装程序（具有正确操作系统版本和位数的lsfusion-desktop-*文件）。但是，与使用Java Web Start安装不同，以这种方式安装的桌面客户端不会自动更新。因此，您需要手动更新它，方法是从中央服务器下载桌面客户端新版本的文件（lsfusion-client-6.<新版本>.jar），并用它替换$INSTALL_DIR$/client.jar文件。

注意

以下给出的所有路径和命令适用于平台的主要版本6（对于其他版本，只需将6替换为所需的数字，例如lsfusion6-server→lsfusion11-server）

  • Windows
  • Linux

所有路径默认

路径根据Linux理念进行了更改（特别是使用符号链接）

更新​

单独安装的程序（OpenJDK，PostgreSQL）也单独更新（有关此过程的更多详细信息，请参阅这些程序的文档）

  • Windows
  • Linux

平台组件也相互单独更新。为此，您必须从中央服务器下载组件新版本的文件，并用它替换以下文件：

组件文件
应用服务器（Server）中央服务器上的文件：lsfusion-server-6.<新版本>.jar
要替换的文件：$INSTALL_DIR$/Server/server.jar
Web服务器（Client）中央服务器上的文件：lsfusion-server-5.<新版本>.jar
要替换的文件：$INSTALL_DIR$/Client/webapps/ROOT.war
要更新Tomcat，您需要下载Tomcat新版本的存档，并将其解压缩到$INSTALL_DIR$/Client文件夹中，不包含webapps目录和启动参数文件

平台组件也相互单独更新。为此，您必须运行以下命令：

应用服务器（Server）​

操作系统命令
RHEL 8+ / CentOS 8+ / Fedora 35+dnf update lsfusion6-server
Ubuntu 18+ / Debian 9+apt update lsfusion6-server

Web服务器（Client）​

操作系统命令
RHEL 8+ / CentOS 8+ / Fedora 35+dnf update lsfusion6-client
Ubuntu 18+ / Debian 9+apt update lsfusion6-client

不稳定版本​

将平台升级到SNAPSHOT版本：source <(curl -s https://download.lsfusion.org/apt/update-lsfusion6) <platform version>。

例如，source <(curl -s https://download.lsfusion.org/apt/update-lsfusion6) 6.2-SNAPSHOT。

重启​

对启动参数所做的任何更改，以及对lsFusion模块的更改，都需要重启服务器（更改lsFusion模块时只需重启应用服务器（Server））。可以通过以下方式完成：

  • Windows
  • Linux

应用服务器（Server）​

GUI
控制面板 > 管理 > 服务 > lsFusion 6 Server
命令行
# 停止服务器
$INSTALL_DIR/Server/bin/lsfusion6_server.exe //SS//lsfusion6_server
 
# 启动服务器
$INSTALL_DIR/Server/bin/lsfusion6_server.exe //ES//lsfusion6_server

Web服务器（Client）​

GUI
控制面板 > 管理 > 服务 > lsFusion 6 Client
命令行
# 停止服务器
$INSTALL_DIR/Client/bin/lsfusion6_client.exe //SS//lsfusion6_client
 
# 启动服务器
$INSTALL_DIR/Client/bin/lsfusion6_client.exe //ES//lsfusion6_client

应用服务器（Server）​

命令行
# 停止服务器
systemctl stop lsfusion6-server
 
# 启动服务器
systemctl start lsfusion6-server

Web服务器（Client）​

命令行
# 停止客户端
systemctl stop lsfusion6-client
 
# 启动客户端
systemctl start lsfusion6-client

日志​

平台日志写入以下文件夹：

  • Windows
  • Linux
组件文件夹
应用服务器（Server）$INSTALL_DIR$/Server/logs
Web服务器（Client）$INSTALL_DIR$/Client/logs
桌面客户端Users/<username>/.fusion/logs
组件文件夹
应用服务器（Server）/var/log/lsfusion6-server
Web服务器（Client）/var/log/lsfusion6-client
桌面客户端/home/<username>/.fusion/logs

主要日志（包括停止和启动服务器的过程）位于：

  • 应用服务器（Server）- stdout
  • Web服务器（Client）- catalina.out（因为Web服务器在Tomcat上运行）。
