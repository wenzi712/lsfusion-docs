物料管理

"物料管理"任务描述​

使用lsFusion平台创建的信息系统必须支持非常基础的供应链执行功能。

为了简单起见，让我们在系统中定义一种增加库存余额的文档类型——供应商收货单；以及一种相反操作的文档类型——向客户批发的发货单。

定义领域逻辑​

信息系统将由一组模块组成，每个模块实现逻辑上独立的功能块。每个模块可以使用其他模块的功能，这涉及到定义模块依赖的特殊语法结构。

根据我们的任务，让我们定义要实现的模块列表：仓库模块、商品模块、法人实体模块、收货单模块、发货单模块、当前余额模块。我们将单独定义将被执行的主模块，基本上是一个复合解决方案。模块的组成可以不同，由开发人员根据在其他地方重用功能的需要来确定。

定义仓库​

让我们创建一个模块，在其中定义仓库实例及其属性。

MODULE Stock;

定义仓库的概念及其属性：名称、地址。

CLASS Stock '仓库';

name '名称' = DATA STRING[100] (Stock) IN base;
address '地址' = DATA STRING[150] (Stock) IN base;

定义商品​

让我们创建一个模块，在其中定义商品的概念及其属性。

MODULE Item;

定义商品的概念及其属性：名称、条形码。

CLASS Item '商品';

name '名称' = DATA STRING[100](Item) IN base;
barcode '条形码' = DATA BPSTRING[13](Item) IN base;

设置商品的批发价格。

salePrice '批发价格' = DATA NUMERIC[17,2](Item) IN base;

定义法人实体​

让我们创建一个模块，在其中定义法人实体实例及其属性。在系统中，法人实体将充当供应商和客户。

MODULE LegalEntity;

定义法人实体的概念及其属性：名称、法定地址、税号。

CLASS LegalEntity '组织';

name '名称' = DATA STRING[100](LegalEntity) IN base;
address '地址' = DATA STRING[150](LegalEntity) IN base;
inn '税号' = DATA BPSTRING[9](LegalEntity) IN base;

定义法人实体税号的唯一性。

legalEntityINN = GROUP AGGR LegalEntity legalEntity BY inn(legalEntity);

legalEntityINN属性将组织和税号一对一绑定，并允许通过税号查找法人实体。该属性的表达式可以解释为：当按税号（inn属性）对法人实体进行分组时，每个组必须包含一个不重复的法人实体。

定义收货单​

让我们创建一个模块，在其中定义定义供应商收货单逻辑所需的所有实例和属性。

MODULE Receipt;

定义在Receipt模块中使用来自其他模块的功能。

REQUIRE Stock, Item, LegalEntity;

我们定义确定供应商收货单逻辑的概念。让我们假设系统中的所有文档（包括收货单和发货单）都由表头和项目明细组成。相应地，让我们定义收货单表头和收货单行的概念。

CLASS Receipt '收货单';
CLASS ReceiptDetail '收货单行';

每个收货单行都包含指向文档表头的链接，因此最终，表头和链接到此文档的行子集一起从用户角度定义了收货单。NONULL参数表示链接必须被定义。DELETE参数指定如果主Receipt对象被删除，所有链接到它的ReceiptDetail行也将被删除。默认情况下，当对象被删除时，所有指向它的链接都将被置空。这样，如果没有DELETE参数，系统将显示关于未定义链接的错误消息。

receipt '行文档' = DATA Receipt (ReceiptDetail) NONULL DELETE;

定义收货单中的行号。

index '行号' (ReceiptDetail d) =
        PARTITION SUM 1 IF d IS ReceiptDetail
        ORDER d BY receipt(d);
信息

在表达式中使用对象类的名称类似于使用系统为所有对象通过自动计数器创建的标识号（id）。在这种情况下，使用ORDER d结构有助于按其id的升序对收货单的行进行排序，即基本上按照它们的创建顺序。

这里，PARTITION运算符使用BY块，该块按特定属性对对象进行分组。表达式累计总和的计算在每个组内执行。在这种情况下，行号仅在此行的文档（receipt属性）内确定。

我们定义收货单表头的一组关键属性：编号、日期、供应商及其名称、接收产品的仓库及其名称。供应商和仓库的名称将来需要在表单上显示它们。

number '收货单编号' = DATA BPSTRING[10] (Receipt);
date '收货日期' = DATA DATE (Receipt);

supplier '供应商' = DATA LegalEntity (Receipt);
nameSupplier '供应商名称' (Receipt r) = name(supplier(r));

stock '仓库' = DATA Stock (Receipt);
nameStock '仓库名称' (Receipt r) = name(stock(r));

我们定义收货单行的一组关键属性：商品及其名称、数量、供应商价格、供应商金额（价格乘以数量）。

item '商品' = DATA Item (ReceiptDetail);
nameItem '商品名称' (ReceiptDetail d) = name(item(d));

quantity '数量' = DATA NUMERIC[16,4] (ReceiptDetail);
price '供应商价格' = DATA NUMERIC[17,2] (ReceiptDetail);
sum '供应商金额' (ReceiptDetail d) = quantity(d) * price(d);

定义发货单​

让我们创建一个模块，在其中定义批发发货单所需的所有实例和属性。

MODULE Shipment;

定义在Shipment模块中使用来自其他模块的功能。

REQUIRE Stock, Item, LegalEntity;

与收货单类似，我们定义发货单表头和行，以及行中指向表头的链接及其编号。

CLASS Shipment '发货单';
CLASS ShipmentDetail '发货单行';

shipment '行文档' = DATA Shipment (ShipmentDetail) NONULL DELETE;
index '行号' (ShipmentDetail d) =
        PARTITION SUM 1 IF d IS ShipmentDetail
        ORDER d BY shipment(d);

我们定义发货单的一组属性：编号、日期、客户及其名称、发货的仓库及其名称。

number '发货单编号' = DATA BPSTRING[10] (Shipment);
date '发货日期' = DATA DATE (Shipment);

customer '客户' = DATA LegalEntity (Shipment);
nameCustomer '客户名称' (Shipment s) = name(customer(s));

stock '仓库' = DATA Stock(Shipment);
nameStock '仓库名称' (Shipment s) = name(stock(s));

我们定义发货单的一组关键属性：商品及其名称、数量、销售价格、销售金额（价格乘以数量）。

item '商品' = DATA Item (ShipmentDetail);
nameItem '商品名称' (ShipmentDetail d) = name(item(d));

quantity '数量' = DATA NUMERIC[16,4](ShipmentDetail);
price '销售价格' = DATA NUMERIC[17,2](ShipmentDetail);
sum '销售金额' (ShipmentDetail d) = quantity(d) * price(d);

我们实现发货单中商品销售价格的自动填充，使用用户为商品定义的批发价格（salePrice属性）的值。发货单行的自动填充应在商品更改时工作（WHEN CHANGED语句）。

price(ShipmentDetail d) <- salePrice(item(d)) WHEN CHANGED(item(d));

定义当前商品余额​

当前商品余额定义为所有商品收货和所有发货之间的差额。

让我们创建一个单独的模块。

MODULE StockItem;

定义在StockItem模块中使用来自其他模块的功能。

REQUIRE Shipment, Receipt;

让我们定义仓库中当前商品余额的计算属性（数量方面）。

receivedQuantity '总收入' = GROUP SUM quantity(ReceiptDetail d) BY item(d), stock(receipt(d));
shippedQuantity '总支出' = GROUP SUM quantity(ShipmentDetail d) BY item(d), stock(shipment(d));
currentBalance '当前余额' (Item i, Stock s) = receivedQuantity (i, s) (-) shippedQuantity (i, s);

让我们禁止负商品余额。禁止将适用于导致负余额的任何用户操作。在这种情况下，用户将在屏幕上看到指定的消息。

CONSTRAINT currentBalance(Item i, Stock s) < 0 MESSAGE '商品余额不能为负';

定义视图逻辑​

为了能够使用创建的解决方案，让我们添加目录表单和当前余额表单，以及一组用于处理文档的配对表单：用于列出收货单的表单和用于编辑它们的表单，用于列出发货单的表单和用于编辑它们的表单。

首先，让我们创建目录表单。

在Stock模块中，我们添加一个表单，为用户提供创建和删除仓库以及更改其属性的功能。

FORM stocks '仓库'
	OBJECTS s = Stock
	PROPERTIES(s) name, address, NEW, DELETE
;

以类似的方式，我们将在Item模块中创建商品表单，在LegalEntity模块中创建法人实体表单。

CLASS Item '商品';

name '名称' = DATA STRING[100](Item) IN base;
barcode '条形码' = DATA BPSTRING[13](Item) IN base;
CLASS LegalEntity '组织';

name '名称' = DATA STRING[100](LegalEntity) IN base;
address '地址' = DATA STRING[150](LegalEntity) IN base;
inn '税号' = DATA BPSTRING[9](LegalEntity) IN base;

让我们创建收货单和发货单的编辑表单。这些表单将用于创建新文档或编辑现有文档。表单的布局将类似：两个垂直块，顶部包含正在创建/编辑的文档表头属性的面板，底部包含网格视图中的文档行及其属性。

在Receipt模块中，我们应该创建收货单编辑表单。对于我们正在构建的表单，我们指定它将用作创建/编辑收货单的默认表单（EDIT表单块）。

FORM receipt '收货单'
	OBJECTS r = Receipt PANEL
	PROPERTIES(r) number, date, nameSupplier, nameStock

	OBJECTS d = ReceiptDetail
	PROPERTIES(d) index, nameItem, quantity, price, sum READONLY, NEW, DELETE GRID
	FILTERS receipt(d) = r

	EDIT Receipt OBJECT r
;

当前收货单的行过滤是通过FILTERS receipt(d) == r表达式执行的。FILTERS构造在表单上显示相应类的对象，如果过滤器表达式返回的值不同于NULL。在这种情况下，如果行链接的文档表头（receipt属性）等于顶部块的当前对象，则收货单行将显示在表单上。换句话说，将只显示正在创建/编辑的文档的行。

此外，如果在表单上为该类的对象指定了过滤器，那么当用户按下NEW按钮时，新添加对象的属性将自动填充，使该对象满足过滤器条件。在这种情况下，当创建新的收货单行时，该行的receipt属性将自动填充为指向当前收货单表头的链接。

让我们在Shipment模块中创建发货单的编辑表单。对于我们正在创建的表单，我们指定它将用作创建/编辑发货单的默认表单（EDIT表单块）。

FORM shipment '发货单'
	OBJECTS s = Shipment PANEL
	PROPERTIES(s) number, date, nameCustomer, nameStock

	OBJECTS d = ShipmentDetail
	PROPERTIES(d) nameItem, quantity, price, sum READONLY, NEW, DELETE GRID
	FILTERS shipment(d) = s

	EDIT Shipment OBJECT s
;

在视觉上，供应商收货单和发货单表单将看起来几乎相同，由两个垂直块组成：一个带有文档表头表的块和一个带有文档行表的块。文档行应支持按文档进行视觉过滤，当在顶部块中导航时，其显示的子集将更改。

让我们创建收货单表单。在此表单上，我们将显示上面定义的文档表头及其行的所有属性。此外，我们将放置自动定义的按钮，用于使用上面创建的编辑表单创建/编辑收货单。除了用于创建/编辑收货单的按钮外，文档表头及其行的所有属性在表单上直接编辑时应为只读（READONLY运算符）。

FORM receipts '收货单列表'
	OBJECTS r = Receipt
	PROPERTIES(r) READONLY number, date, nameSupplier, nameStock
	PROPERTIES(r) NEWSESSION NEW, EDIT, DELETE

	OBJECTS d = ReceiptDetail
	PROPERTIES(d) READONLY index, nameItem, quantity, price, sum
	FILTERS receipt(d) = r
;

让我们以类似的方式创建发货单表单。

FORM shipments '发货单列表'
	OBJECTS s = Shipment
	PROPERTIES(s) READONLY number, date, nameCustomer, nameStock
	PROPERTIES(s) NEWSESSION NEW, EDIT, DELETE

	OBJECTS d = ShipmentDetail
	PROPERTIES(d) READONLY nameItem, quantity, price, sum
	FILTERS shipment(d) = s
;

接下来，在StockItem模块中，让我们创建一个显示当前余额的表单。表单应该是一个表，其行包含关于商品（其名称和条形码）、仓库名称以及该仓库中该商品的当前余额的信息。表单上行的数量将等于输入到系统中的商品数量乘以输入的仓库数量。为了只显示相关数据（即，只有那些商品和仓库，它们的交集的当前余额不为NULL），让我们向表单添加一个过滤器。

FORM currentBalanceItemStock '当前余额'
    OBJECTS si = (s = Stock, i = Item)
    PROPERTIES READONLY name(i), barcode(i), name(s), currentBalance(i, s)
    FILTERS currentBalance(i, s)
;

OBJECTS si = (s = Stock, i = Item)构造添加了一个名为si的对象组，它是Stock和Item类对象的笛卡尔积。

最后，让我们声明主模块并指定在其中将使用来自其他模块的哪些功能。

MODULE StockAccounting;

REQUIRE Stock, Item, LegalEntity, Receipt, Shipment, StockItem;

在StockAccounting模块中，我们组成系统菜单。目录应该添加到导航器的预定义masterData文件夹中，我们在目录之后立即显示。我们将当前余额表单放在主菜单（水平窗口root）中。当用户选择相应的root文件夹时，指向目录和文档表单的链接将显示在垂直toolbar上。

NAVIGATOR {
    NEW FOLDER masterData '目录' FIRST WINDOW toolbar {
        NEW items;
        NEW stocks;
        NEW legalEntities;
    }
    NEW FOLDER documents '文档' AFTER masterData WINDOW toolbar {
        NEW receipts;
        NEW shipments;
    }
    NEW currentBalanceItemStock AFTER documents;
}
