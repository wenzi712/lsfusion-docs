对象块

FORM语句的对象块 - 向表单结构添加对象组，并向交互式表单视图添加对象树。

对象块​

语法​

OBJECTS groupDeclaration1 [groupOptions1], ...., groupDeclarationN [groupOptionsN]

每个groupDeclaration是包含多个对象的对象组的声明：

[groupName =] (objectDeclaration1, ..., objectDeclarationK)

或者是包含单个对象的对象组：

objectDeclaration

每个声明对象的objectDeclaration具有以下语法：

[[name] [caption] =] classId objectOptions

对象选项objectOptions可以以任意顺序一个接一个列出。支持以下选项集：

ON CHANGE actionId(param1, ..., paramM)
ON CHANGE actionOperator 
EXTID objectExtID

在每个对象组声明之后，可以以任意顺序列出组选项groupOptions：

viewType
insertPosition
defaultObjectsType
PAGESIZE pageSize 
IN propertyGroup
EXTID extID
EXTKEY
SUBREPORT [subReportExpression]
BACKGROUND backgroundExpr
FOREGROUND foregroundExpr

描述​

单个OBJECTS块可以包含多个以逗号分隔的对象组声明。对象组可以只包含一个对象，也可以包含多个对象。对于单个对象，可以使用简化语法，无需指定对象组名称和使用括号。

参数​

  • groupName

    对象组的名称。简单ID。如果需要访问包含多个对象的对象组，则必须指定。如果对象组只包含一个对象，则对象组的名称将等于对象的名称，无需指定。

  • name

    对象名称。简单ID。如果对象类是内置类，则必须指定。如果对象类是自定义类，则无需指定名称。在这种情况下，它将等于类对象的名称。

  • classId

    对象类ID。

  • caption

    正在添加的对象的标题。字符串字面量。如果未指定标题，则类标题将成为对象标题。

对象选项​

  • ON CHANGE actionName(param1, ..., paramM)

    指定当对象的当前值更改时将调用的操作。

    • actionID

      操作ID。

    • param1, ..., paramM

      将用作操作参数的表单上对象名称的列表。这些对象的数量必须等于操作参数的数量。对象名称由简单ID定义。

  • ON CHANGE actionOperator

    创建当对象的当前值更改时将调用的操作。

    • actionOperator

      上下文相关的操作运算符。您可以使用表单上已声明对象的名称作为参数。

  • EXTID objectExtID

    指定将用于此表单对象的导出/导入的名称。仅用于结构化视图。

    • objectExtID

      字符串字面量。

对象组选项​

  • viewType

    此对象组的属性的默认视图。通过以下方式之一指定：

    • PANEL

      关键字，指定时选择面板视图类型

    • TOOLBAR

      关键字，指定时选择工具栏视图类型

    • GRID

      关键字，指定时选择表格列视图类型。默认使用。

    • PIVOT [pivotOptions]

      当指定PIVOT关键字时，选择数据透视表视图类型。此视图类型的选项pivotOptions可以以任意顺序一个接一个指定。

      • pivotType

        定义数据透视表初始显示模式的字符串字面量。可以等于以下值之一：

        • 'Table'（默认值）
        • 'Table Bar Chart'
        • 'Table Heatmap'
        • 'Table Row Heatmap'
        • 'Table Col Heatmap'
        • 'Bar Chart'
        • 'Stacked Bar Chart'
        • 'Line Chart'
        • 'Area Chart'
        • 'Scatter Chart'
        • 'Multiple Pie Chart'
        • 'Horizontal Bar Chart'
        • 'Horizontal Stacked Bar Chart'

      • calcType

        指定初始聚合函数。可以使用以下关键字之一设置：

        • SUM - 值的总和（默认值）
        • MAX - 值的最大值
        • MIN - 值的最小值

      • settingsType

        指定是否向用户显示数据透视表设置。可以通过以下关键字之一指定：

        • SETTINGS - 显示设置（默认值）
        • NOSETTINGS - 不显示设置

    • MAP [tileProvider]

      当指定MAP关键字时，选择地图视图类型。默认情况下，此视图使用OpenStreetMap地图。也可以使用Google或Yandex地图。为此，您需要在项目中包含Geo.lsf模块，然后获取Google或Yandex的API密钥并在Administration > Application > Settings > Navigation中指定。

      • tileProvider

        指定地图源的字符串字面量。可能的选项：

        • 'openStreetMap'（默认值）
        • 'google'
        • 'yandex'

    • CALENDAR

      关键字，指定时选择日历视图类型。

    • CUSTOM renderFunction [HEADER expr]

      当指定CUSTOM关键字时，选择自定义视图类型。

      • renderFunction

        指定负责显示数据的JavaScript函数名称的字符串字面量。此函数必须位于包含在项目资源中的.js文件中，并在客户端加载以供使用。它应返回一个包含三个函数的JavaScript对象：

        • render(element, controller)
        • update(element, controller, list, options)
        • clear(element)（可选）

        有关此机制的更详细描述，请参阅文章How-to: Custom Components (Objects)。

      • expr

        表达式，其值必须是JSON类的对象。用于传递不依赖于所描述对象组值的数据。

  • insertPosition

    指定对象组在对象组列表中的插入位置。最常与表单扩展机制一起使用。可以通过以下方式之一指定：

    • AFTER groupName

    • BEFORE groupName

      对象组将直接添加到表单结构中指定的对象组之前（关键字BEFORE）或之后（关键字AFTER）。如果要在其之前（之后）添加的组在树中，则它必须是该树中的第一个（最后一个）。

      • groupName

        对象组名称。简单ID。

    • FIRST

      关键字，表示对象组将添加到列表的开头。

    • LAST

      关键字，表示对象组将添加到列表的末尾。与默认添加不同，使用LAST插入的对象组将始终位于按声明顺序添加的所有对象组之后。

    • DEFAULT

      关键字，表示对象组按声明顺序添加。这是默认值。

  • defaultObjectsType

    指定在更改活动过滤器后，添加的对象组中的哪个对象集合将是当前的。通过以下关键字之一指定：

    • FIRST– 指定第一个对象集合（根据当前顺序）将是默认对象
    • LAST - 最后一个对象集合
    • PREV - 前一个（或尽可能接近的）对象集合
    • NULL - 无（重置）

    如果未指定此选项，平台将根据当前过滤器确定要使用的选项。

  • PAGESIZE pageSize

    指定表格中可读对象的数量。默认情况下，此数量根据用户界面中组件的大小动态确定，等于3 * <表格中可见行数>。值为0表示必须读取所有对象。

    • pageSize

      读取的对象数量。整数字面量。

  • IN propertyGroup

    指定对象组所属的属性和操作组。仅用于层次结构视图。

    • propertyGroup

      属性和操作组名称。复合ID。

  • EXTID extID

    指定将用于此对象组的导出/导入的名称。仅用于结构化视图。

    • extId

      字符串字面量。

  • EXTKEY

    当指定关键字EXTKEY时，此对象组的对象和属性的值在结构化视图中表示为键值对，其中键是对象（对象集）的值，值是属性值。默认情况下，它们表示为带有属性值列表的数组。

  • SUBREPORT [subReportExpression]

    指定在构建报表层次结构时，需要为此对象组生成单独的报表文件。

    • subReportExpression

      表达式，其值将用作所创建对象组的报表文件的名称。您可以使用表单上已声明对象的名称作为参数。假设当表单在打印视图中打开时，这些对象的值将被传递（如果未完成，它们将被视为等于NULL）。

  • BACKGROUND backgroundExpr

    指定属于此对象组的属性单元格的背景颜色。

    • backgroundExpr

      表达式，其值确定背景颜色。

  • FOREGROUND foregroundExpr

    指定属于此对象组的属性单元格的前景颜色。

    • foregroundExpr

      表达式，其值确定前景颜色。

示例​

CLASS Shipment;
// 声明发货表单
FORM shipments 'Deliveries'
    OBJECTS s = Shipment // 添加一个发货类的对象
                        PAGESIZE 100 // 指示表格应始终包含100行

    // ... 向表单添加属性和过滤器
;

// 声明一个将显示指定时间间隔内产品周转的表单
name = DATA STRING[100] (Item);
revenue = DATA NUMERIC[16,2] (Item, DATE, DATE);

FORM revenues 'Product turnovers'
    // 声明一个对象组，由2个带有适当标题的Date类对象组成，
    // 将始终显示为面板
    OBJECTS interval = (dateFrom 'Date (from)' = DATE, dateTo 'Date (to)' = DATE) PANEL, 
            i = Item // 添加产品列表
    // 向表单添加日期对象值的属性，用户可以通过这些属性选择日期
    PROPERTIES VALUE(dateFrom), VALUE(dateTo) 
    // 添加产品名称和日期间隔内产品周转的属性
    PROPERTIES name(i), revenue(i, dateFrom, dateTo) 
;
// 创建用于打印产品价格标签的表单
labelFile = DATA STRING[100] (Item);
printLabelFile (Item i)= OVERRIDE labelFile(i), 'MyModule_printLabel_i.jrxml' IF i IS Item;
FORM printLabel 'Price tag printing'
    OBJECTS i = Item  // 添加将打印价格标签的产品
    // 标记应使用名称存储在printLabelFile属性中的文件作为模板
    // （假设i值将在OBJECTS块中传递）
    REPORT printLabelFile(i)       
    // 例如，用户可以在那里输入myLabel1.jrxml，然后系统将使用名为myLabel1.jrxml的文件
    //  ... 打印所需的其他属性
;

对象树块​

语法​

TREE [name] groupDeclaration1 [parentBlock1], ...., groupDeclarationN [parentBlockN] [insertPosition]

每个groupDeclaration是对象组的声明，完全类似于上面描述的对象块中的声明。每个parentBlock可以以两种方式之一描述：

PARENT parentExpr
(PARENT parentExpr1, ..., parentExprK)

第一种选项用于指定该块的对象组由单个对象组成，第二种选项用于包含两个或更多对象的组。

描述​

对象树块允许您创建对象树。第一个指定的对象组将形成顶级对象列表，每个对象将有第二个指定对象组的对象的子列表，依此类推。

使用PARENT块创建层次结构对象组。为此，指定一个将为对象定义父元素的属性（如果对象组包含多个对象，则为多个对象）。

参数​

  • name

    正在创建的对象树的名称。简单ID。

  • parentExpr

    为包含单个对象的对象组定义层次结构的表达式。此表达式必须创建一个恰好有一个参数的属性，并返回作为输入传递的对象的父对象（如果传递的对象在顶级，则返回NULL）。

  • parentExpr1, ..., parentExprK

    为包含多个对象的对象组定义层次结构的表达式列表。这些表达式应该创建参数数量等于组中对象数量的属性。每个这些属性应该返回作为输入传递的对象集合的父对象之一（如果传递的对象集合在顶级，则返回NULL）。第一个属性应返回父对象集合的第一个对象，第二个属性应返回第二个对象，依此类推。

  • insertPosition

    指定树对象组在对象组列表中的插入位置。其语法完全类似于对象块中的相同选项。

示例​

CLASS SkuGroup;
name = DATA ISTRING[100] (SkuGroup);
active = DATA BOOLEAN (SkuGroup);
parent = DATA SkuGroup (SkuGroup) AUTOSET;

CLASS Sku;
name = DATA ISTRING[100] (Sku);
skuGroup = DATA SkuGroup (Sku);


FORM skus 'Sku'
    TREE groupTree g=SkuGroup PARENT parent(g)
    PROPERTIES READONLY name(g)
    FILTERS active(g)

    OBJECTS s = Sku
    PROPERTIES(s) name
    FILTERS skuGroup(s) == g
;

CLASS Group1;
name = DATA STRING[100] (Group1);

CLASS Group2;
name = DATA STRING[100] (Group2);

CLASS Group3;
name = DATA STRING[100] (Group3);

in = DATA BOOLEAN (Group1, Group2);
in = DATA BOOLEAN (Group2, Group3);

FORM groups
    TREE groups g1 = Group1, g2 = Group2, g3 = Group3
    PROPERTIES READONLY name(g1), name(g2), name(g3)
    FILTERS in(g1, g2), in(g2, g3)
;
