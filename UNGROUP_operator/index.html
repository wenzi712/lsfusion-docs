UNGROUP 运算符

危险

目前，UNGROUP 运算符尚未实现。

UNGROUP 运算符创建一个以扩展形式实现分发的属性。

语法​

UNGROUP 
propertyId [BY groupExpr1, ..., groupExprM] 
distributionType exPropertyId [BY exGroupExpr1, ..., exGroupExprM]
[ORDER [DESC] orderExpr1, ..., orderExprK]
[WHERE whereExpr]

其中 distributionType 可以通过以下几种方式描述：

PROPORTION [STRICT] ROUND(digits)
LIMIT [STRICT]

描述​

UNGROUP 运算符创建一个属性，该属性以"多对多"方式在同一组的对象集合之间分配某个值。

第一个 BY 块描述对象集合将被划分成的组。如果未指定 BY 块，则所有对象集合被视为属于同一组。

第二个 BY 块描述结果将按其分组以检查约束/计算比例的额外组。如果未指定第二个 BY 块，则所有对象集合被视为属于同一组。

ORDER 块定义进行分发的顺序。定义的顺序必须唯一确定。

WHERE 块定义对象集合将参与分发操作的条件。

参数​

  • propertyId

    被分发属性的ID。此属性的值必须是数字，且参数数量必须等于 BY 块中的组数。计算组/分区表达式的值时，将作为输入传递给此属性。

  • groupExpr1, ..., groupExprM

    组表达式列表。

  • distributionType

    分发类型。这些有以下类型：

    • PROPORTION

      指定使用比例分发的关键字。在这种情况下，特定组的被分发属性值按比例分发给属于该组的对象集合。比例由分发类型后指定的 expr 表达式定义。

      • STRICT

        指定此关键字时，被分发属性的值必须精确地（无余数）在属于该组的对象集合之间分配。如果分发后有余数（也可能为负数），则将其添加到根据 ORDER 块中定义的顺序的第一个对象集合中。

      • ROUND(digits)

        指定值将四舍五入到的小数位数。

        • digits

          指定小数位数的整数字面量。

    • LIMIT

      指定使用具有指定限制的分发的关键字。在这种情况下，被分发属性的值最初为第一个对象集合设置。如果此集合超过限制，则为第一个对象集合设置限制，并将被分发属性的剩余值分配给第二个对象集合。然后检查第二个对象集合是否超过限制，依此类推。限制由指定分发类型后指定的 expr 表达式定义。

      • STRICT

        指定此关键字时，被分发属性的值必须精确地（无余数）在属于该组的对象集合之间分配。如果分发后有余数，则将其添加到根据 ORDER 块中指定的顺序的最后一个对象集合中。

  • exPropertyId

    为 PROPORTION 类型分发定义比例，并为 LIMIT 类型分发定义限制的属性的ID。此属性的值必须是数字，且参数数量必须等于 BY 块中的组数。

  • exGroupExpr1, ..., exGroupExprM

    额外组表达式列表。

  • DESC

    关键字。指定对象集合的反向迭代顺序。

  • orderExpr1, ..., orderExprK

    确定计算聚合函数或分发过程中迭代对象集合顺序的表达式列表。要确定顺序，首先使用第一个表达式的值；然后，如果相等，则使用第二个表达式的值，依此类推。

  • whereExpr

    过滤表达式。只有过滤表达式值不为 NULL 的对象组才会参与分组。
