DESIGN 语句

DESIGN 语句用于更改表单设计。

语法​

语法由嵌套的设计语句块组成。以关键字 DESIGN 开头的外部块定义了将要更改其设计的表单：

DESIGN formName [caption] [CUSTOM] {
    designStatement1
    ...
    designStatementN
}

每个 designStatement 描述一个设计语句。设计语句有以下几种类型：

NEW name [insertPos] [{...}];
MOVE selector [insertPos] [{...}];  
selector [{...}];   
REMOVE selector;
propertyName = value;

前三个语句——创建（NEW）、移动（MOVE）和修改——可能会包含嵌套的设计语句块。设计语句删除（REMOVE）和更改属性值（=）是简单的单个语句。如果设计语句不包含嵌套语句块，则必须以分号结束。

每个 selector 可以是以下类型之一：

componentName
PROPERTY(formPropertyName)
FILTERGROUP(filterGroupName)
PARENT(selector)
GROUP([propertyGroupSelector][,groupObjectTreeSelector])
noGroupObjectTreeContainerType
groupObjectTreeContainerType(groupObjectTreeSelector)

反过来，groupObjectTreeSelector 可以是以下两种类型之一：

groupObjectSelector
TREE treeSelector

描述​

使用 DESIGN 语句，开发人员可以通过创建、移动和删除容器和组件，以及更改它们的某些属性来管理交互式表单视图的设计。默认情况下，每个表单都会创建一个默认设计以及相应的容器。如有必要，可以重新创建没有默认容器和先前配置设置的设计。这通过使用关键字 CUSTOM 来实现。

每个用大括号括起来的设计语句块允许修改特定组件及其后代。我们将此组件称为当前组件，如果我们知道该组件在我们的案例中应该是容器，则称为当前容器。在 DESIGN 关键字后的外部块中，main 容器是当前组件。以下是几种设计语句：

  • 创建语句（NEW）允许创建一个新容器，使其成为当前容器的后代。新创建的容器将是此语句中包含的设计语句块中的当前组件。
  • 移动语句（MOVE）允许将现有组件作为当前容器的直接后代。该组件首先从之前的父容器中移除。被移动的组件将成为此语句中包含的设计语句块中的当前组件。
  • 修改语句允许修改指定的组件，该组件必须是当前容器的后代（不一定是子组件）。指定的元素将是此语句中包含的设计语句块中的当前组件。
  • 删除语句（REMOVE）允许从组件层次结构中移除指定的组件。要移除的组件必须是当前容器的后代。
  • 更改属性值语句（=）允许更改当前组件指定属性的值。

此语句中描述的组件层次结构可以有任意数量的嵌套级别，并可以描述每个级别上的任意数量的组件及其属性。

要访问设计组件，您可以使用它们的名称或访问表单上的属性组件（PROPERTY）、父组件（PARENT）、属性组组件（GROUP）和其他基础组件/默认设计组件。

参数​

通用参数​

  • formName

    正在更改的表单的名称。复合 ID。

  • caption

    交互式视图模式下的新表单标题。字符串字面量。表单标题在导航器中不会更改。

  • name

    正在创建的容器的名称。简单 ID。

  • insertPos

    指定组件的插入位置。可以通过以下方式之一指定：

    • BEFORE selector

    • AFTER selector

      指定组件必须添加或移动到指定组件之前（关键字 BEFORE）或之后（关键字 AFTER）。指定的组件必须是当前容器的子组件。

    • FIRST

      关键字，指示组件应添加或移动到当前容器的子组件列表的开头。

    • LAST

      关键字，指示组件应添加或移动到当前容器的子组件列表的末尾。与默认添加不同，使用 LAST 插入的组件将始终位于按插入顺序添加的所有组件之后。

    • DEFAULT

      关键字，指示组件应按插入顺序添加或移动到当前容器的子组件列表中。这是默认值。

  • propertyName

    组件属性的名称。现有属性列表在下面的表格中提供。

  • value

    分配给相应容器属性的值。可接受的值类型在下面的表格中提供。

selector 参数​

  • componentName

    设计组件的名称。简单 ID。

  • formPropertyName

    表单上的属性/动作名称。

  • filterGroupName

    过滤器组的名称。简单 ID。

  • propertyGroupSelector

    属性组的名称。简单 ID。

  • groupObjectSelector

    表单上的对象组的名称。简单 ID。

  • treeSelector

    表单上的对象树的名称。简单 ID。

  • noGroupObjectTreeContainerType

    表单容器类型：

    • BOX – 通用表单容器
    • PANEL – 包含以 PANEL 视图显示且显示组未定义的属性的组件。
    • TOOLBARBOX – 通用工具栏容器，包含显示在面板中的属性组件，标记为放置在 TOOLBAR 上，且未定义对象组。
    • TOOLBARLEFT – 工具栏的左侧部分
    • TOOLBARRIGHT - 工具栏的右侧部分
    • TOOLBAR 包含以 TOOLBAR 视图显示且显示组未定义的属性的组件。

  • groupObjectTreeContainerType

    对象组/树容器的类型。

    • noGroupObjectTreeContainerType 表单的所有容器类型（语义相同）
    • GRID - 表格组件
    • TOOLBARSYSTEM - 系统工具栏（记录数量、组更改等）。
    • FILTERGROUPS - 包含过滤器组组件
    • FILTERS - 显示自定义过滤器的组件

示例​

DESIGN order { // 自定义表单设计，从默认设计开始
               // 标记所有层次结构的更改都将发生在最顶层容器上
    // 创建一个新容器，作为系统按钮之前的第一个容器，
    // 其中我们放置两个容器 - 头部和规格
    NEW orderPane FIRST { 
        fill = 1; // 指定容器应占据其可用的所有空间
        MOVE BOX(o) { // 将与对象 o 相关的所有内容移动到新容器中
            PANEL(o) { // 配置对象 o 面板中属性的显示方式
                horizontal = FALSE; // 使所有后代从上到下排列
                NEW headerRow1 { // 创建容器 - 第一行
                    horizontal = TRUE;
                    MOVE PROPERTY(date(o)) { // 移动订单日期属性
                        // 在表单设计中「覆盖」属性标题（替代标准标题）
                        caption = '编辑订单的日期'; 
                        // 为订单日期属性设置提示
                        tooltip = '在此处输入订单的制作日期'; 
                        background = #00FFFF; // 使背景为红色
                    }
                    MOVE PROPERTY(time(o)) { // 移动订单时间属性
                        foreground = #FF00FF; // 使颜色为绿色
                    }
                    MOVE PROPERTY(number(o)) { // 移动订单编号属性
                        // 设置用户应优选显示5个字符
                        charWidth = 5; 
                    }
                    MOVE PROPERTY(series(o)); // 移动订单系列属性
                }
                NEW headerRow2 {
                    horizontal = FALSE; // 后代 - 从上到下
                }
                MOVE PROPERTY(note(o));
            }

            size = (400, 300); //指定容器 o.box 应具有 400x300 像素的基本大小
        }
        // 创建一个容器，用于存储订单的各种规格
        NEW detailPane { 
            // 标记此容器应为选项卡面板，其中其后代是选项卡
            tabbed = TRUE;
            MOVE BOX(d) { // 将订单行容器添加为顶部面板中的选项卡之一
                caption = '行'; // 设置选项卡面板的标题
                // 使行号列永远不会有焦点
                PROPERTY(index(d)) { focusable = FALSE; } 
                GRID(d) {
                    // 确保默认情况下打开表单时焦点设置在行表格上
                    defaultComponent = TRUE; 
                }
            }
            MOVE BOX(s) { // 将商品总计容器添加为 detailPane 选项卡之一
                caption = '选择';
            }
        }
    }
}

// 将表单定义拆分为两个语句（第二个语句可以传输到另一个模块）
DESIGN order {
    // 从层次结构中删除包含打印和导出到 xls 按钮的容器，从而使它们不可见
    REMOVE TOOLBARLEFT; 
}
