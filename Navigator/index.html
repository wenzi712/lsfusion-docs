导航器

导航器是一个由导航器元素组成的层次结构（树）。导航器元素有三种类型：

  • 文件夹 - 将其他导航器元素组合成一个组。选择此元素会影响其子元素的显示（如果它们属于另一个窗口）。
  • 动作 – 执行指定的动作。只有不接受参数的动作才能添加到导航器中。
  • 表单 – 交互式视图以异步模式打开指定的表单。

当元素添加到导航器时，会为其定义一个父元素。导航器的根元素是System.root文件夹。

如果在特定的导航器文件夹中没有显示导航器元素，则该文件夹会自动隐藏。

与交互式表单视图一样，导航器显示在2D空间中：在用户设备的屏幕上。因此，它的设计可以/必须像所有其他图形视图一样定义。

语言​

要管理导航器，请使用NAVIGATOR语句。

示例​

FORM items;
FORM stocks;
FORM legalEntities;
FORM shipments;
hello()  { MESSAGE 'Hello world'; }
hi()  { MESSAGE 'Hi'; }

NAVIGATOR {
    // 创建新的导航器文件夹并使其所有后代显示在具有垂直工具栏的窗口中
    NEW FOLDER catalogs '目录' WINDOW toolbar { 
        // 在文件夹中为 items 表单创建表单元素，默认元素名称为表单名称
        NEW items; 
    }
    catalogs {  // 导航器元素编辑语句
        // 为 stocks 表单创建名为 stocksNavigator 的表单元素，并将最后一个元素添加到 catalogs 文件夹
        NEW FORM stocksNavigator '仓库' = stocks; 
        // 在 catalogs 文件夹中创建名为 legalEntities 的表单元素，紧跟在 items 元素之后
        NEW legalEntities AFTER items; 
        NEW shipments;
    }
    // 创建另一个文件夹，其元素也将显示在具有垂直工具栏的窗口中
    NEW FOLDER documents '文档' WINDOW toolbar { 
        // 文件夹本身将显示在根窗口中，当用户选择其中一个文件夹时，
        // 在具有垂直工具栏的窗口中将显示该特定文件夹的后代
        NEW ACTION hi;   // 创建动作元素
        NEW ACTION h=hello;   // 创建动作元素
        // 将 shipments 元素从 catalogs 文件夹移动到 document 文件夹的语句
        // 位于 hello 元素之前
        MOVE shipments BEFORE h; 
    }
}
