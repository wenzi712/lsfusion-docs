属性选项

当在=语句中声明属性时，可以在声明的末尾指定一组属性选项。

语法​

选项按照任意顺序一个接一个列出，以空格或换行符分隔：

propertyOption1 ... propertyOptionN

支持以下选项集（每个选项的语法在单独一行中表示）：

IN groupName
viewType
ON eventType { actionOperator }
CHANGEKEY key [SHOW | HIDE]
CHANGEMOUSE key [SHOW | HIDE]
STICKY | NOSTICKY
syncType
MATERIALIZED
TABLE tableName
INDEXED [LIKE | MATCH]
NONULL [DELETE] eventClause
AUTOSET
CHARWIDTH width [FLEX | NOFLEX]
PATTERN patternExpr
REGEXP rexpr [message] 
ECHO
DEFAULTCOMPARE [compare]
EVENTID eventId
LAZY [WEAK | STRONG]
imageSetting
annotationSetting

描述和参数​

  • IN groupName

    指定属性所属的属性和操作组。如果未指定该选项，则属性将默认属于System.private组。

    • groupName

      组名。复合ID。

  • MATERIALIZED

    将属性标记为物化的关键字。这些属性将存储在数据库的表字段中。

  • TABLE tableName

    指定属性将存储在其中的表。表的键数必须等于属性参数的数量，并且参数类必须与表键类匹配。如果未指定表，则属性将自动放置在系统中"最近"的现有表中。

    • tableName

      表名。复合ID。

  • INDEXED

    关键字。如果指定，则为此属性创建索引。类似于使用INDEX语句。

    • LIKE

      关键字。如果指定，则除了普通索引外还会创建GIN索引。

    • MATCH

      关键字。如果指定，则除了普通索引外还会创建GIN索引和带有to_tsvector的GIN索引。

  • NONULL [DELETE] eventClause

    添加确定性约束。如果由于某些更改导致某些对象违反了此约束，则将显示相应的消息，或者，如果指定了DELETE，则将删除此类对象。

    • DELETE

      关键字，如果指定，则当属性变为NULL时，作为属性参数的对象将被删除。

    • eventClause

      事件类型描述块。描述将通过该事件检查属性是否为NULL。

交互式视图块​

  • viewType

    指定将属性添加到表单时的属性视图类型。

    • GRID - 表列
    • TOOLBAR - 工具栏
    • PANEL - 面板

    这类似于在FORM语句的属性块中指定viewType选项。因此，如果此选项既未在属性选项中指定，也未在表单上的属性块中直接指定，则使用属性显示组在表单上的默认视图。

  • ON eventType { actionOperator }

    指定一个操作，该操作将是所有此属性交互视图的特定表单事件的默认处理程序。可以在FORM语句的属性块中覆盖。

    • eventType

      表单事件类型。由以下选项之一指定：

      • CHANGE - 当用户尝试更改属性值时发生。
      • CHANGEWYS - 当用户尝试使用特殊输入机制更改指定属性的值时发生。
      • GROUPCHANGE - 当用户尝试更改表中所有对象的属性值时（组编辑）发生。
      • EDIT - 当用户尝试编辑作为指定属性值的对象时发生。
      • CONTEXTMENU [caption] - 用户在表单上的属性上下文菜单中选择了指定项。如果需要，您还可以定义此菜单项的caption（字符串字面量）。如果未指定，则默认情况下它将与操作标题相同。

    • actionOperator

      上下文相关操作运算符。定义在事件上执行的操作的运算符。您可以使用属性本身的参数作为运算符参数。

  • imageSetting

    属性的图标设置。此选项允许您手动配置图标。它可以采用以下形式之一：

    • IMAGE [imageLiteral]

      属性的手动图标指定。如果未提供imageLiteral，则启用自动分配模式。

      • imageLiteral

        值定义图标的字符串字面量。

    • NOIMAGE

      指示属性不应有图标的关键字。

  • annotationSetting

属性注释。以@@开头。支持以下注释：

  • @@deprecated

  • @@deprecated(since)

  • @@deprecated(since, message)

    将属性标记为已弃用，不建议使用。 插件将此类属性显示为删除线。

    • since 指示属性被视为已弃用的平台版本的字符串字面量。

    • message 提供为什么将属性标记为已弃用的解释的字符串字面量。

DESIGN语句默认值块​

  • CHARWIDTH width [FLEX | NOFLEX]

    指定应该对用户可见的属性值的字符数。为默认设计设置值，可以在DESIGN语句中覆盖。

    • width

      字符数。整数字面量。

    • FLEX

      关键字。如果指定，则属性值的扩展系数自动设置为其基本大小。

    • NOFLEX

      关键字。如果指定，则属性值的扩展系数自动设置为零。

  • PATTERN patternExpr

    指定属性值的格式化模式。定义模式的语法类似于DecimalFormat或SimpleDateFormat，取决于值类型。

    • patternExpr

      表达式，其值定义格式化模式。

  • REGEXP rexpr [message]

    指定编辑后属性值应对应的正则表达式。为默认设计设置值，可以在DESIGN语句中覆盖。

    • rexpr

      描述正则表达式的字符串字面量。规则类似于Java中接受的规则。

    • message

      描述如果用户输入与正则表达式不匹配的值将显示给用户的消息的字符串字面量。如果未指定，将显示默认消息。

  • ECHO

    导致显示星号*字符而不是属性值的关键字。例如，用于密码。可以在DESIGN语句中覆盖。

  • CHANGEKEY key [SHOW | HIDE]

    指定触发编辑此属性的组合键。为默认设计设置值，可以在DESIGN语句中覆盖。

    • key

      字符串字面量，描述组合键。语法：

      keyStroke [;(modeKey=modeValue;)*]

      • keyStroke 组合键的字符串表示。定义原则类似于Java类方法KeyStroke.getKeyStroke(String)中参数的指定方式。(http://docs.oracle.com/javase/7/docs/api/javax/swing/KeyStroke.html#getKeyStroke(java.lang.String))。

      • (modeKey=modeValue;)* 指定keyStroke执行条件的选项。支持以下选项：

        • priority = priorityValue 优先级，整数值。如果多个属性满足CHANGEKEY条件，则执行优先级较高的属性。 如果未设置优先级，则等于该属性在表单中的序号。此外，在任何情况下，如果对象组匹配，则会将1000添加到优先级值中。

        • preview = previewValue 所有事件都会检查执行两次：首先是isPreview = true，然后是isPreview = false。支持的previewValue值：

          • AUTO, ONLY -> isPreview
          • NO -> !isPreview
          • ALL -> true

        • dialog = dialogValue 检查CHANGEKEY是否应在对话框窗口中执行。支持的dialogValue值：

          • AUTO, ALL -> true
          • ONLY -> isDialog
          • NO -> !isDialog

        • window = windowValue 检查CHANGEKEY是否应在模态窗口中执行。支持的windowValue值：

          • AUTO, ALL -> true
          • ONLY -> isWindow
          • NO -> !isWindow

        • group = groupValue 检查对象组是否匹配。支持的groupValue值：

          • AUTO, ALL -> true
          • ONLY -> equalGroup
          • NO -> !equalGroup

        • editing = editingValue 检查CHANGEKEY是否应在属性编辑模式下执行。支持的editingValue值：

          • AUTO -> !(isEditing() && getEditElement().isOrHasChild(Element.as(event.getEventTarget())))
          • ALL -> true
          • ONLY -> isEditing
          • NO -> !isEditing

        • showing = showingValue 检查属性当前是否在表单上可见。支持的showingValue值：

          • AUTO, ONLY -> isShowing
          • ALL -> true
          • NO -> !isShowing

        • panel = panelValue 检查属性是否位于面板中。支持的panelValue值：

          • AUTO -> !isMouse || !isPanel
          • ALL -> true
          • ONLY -> isPanel
          • NO -> !isPanel

        • cell = cellValue 检查属性是否位于表格单元格中。支持的cellValue值：

          • AUTO -> !isMouse || isCell
          • ALL -> true
          • ONLY -> isCell
          • NO -> !isCell

        除priority外，所有选项的默认值均为AUTO。

    • SHOW

      关键字。指定后，组合键将显示在属性标题中。默认使用。

    • HIDE

      关键字。指定后，组合键将不会显示在属性标题中。

  • CHANGEMOUSE key [SHOW | HIDE]

    指定触发开始编辑属性的鼠标组合键。为设计设置默认值，可以在DESIGN指令中覆盖。

    • key

    字符串字面量，描述鼠标组合键。语法：

    keyStroke [;(modeKey=modeValue;)*]

    • keyStroke 鼠标组合键的字符串表示。目前，唯一支持的值是DBLCLK — 双击。

    • (modeKey=modeValue;)* 语法与CHANGEKEY的语法相同。

    • SHOW

      指示鼠标组合键应显示在属性标题中的关键字。这是默认行为。

    • HIDE

      指示鼠标组合键不应显示在属性标题中的关键字。

  • STICKY | NOSTICKY

    关键字。STICKY表示表格中的属性将固定在左侧，并在向右滚动时保持可见。NOSTICKY取消此固定。默认情况下，STICKY或NOSTICKY是通过启发式方法确定的。

  • syncType

    定义属性是同步执行还是异步执行：

    • WAIT — 同步。

    • NOWAIT — 异步。这是默认行为。

  • DEFAULTCOMPARE [compare]

    指定属性的默认过滤器类型。

    • compare

      默认过滤器类型。字符串字面量。可以是以下值之一：=, >, <, >=, <=, !=, CONTAINS, LIKE。对于所有数据类型，默认值为=，但不区分大小写的字符串类型除外，对于这些类型，默认值为CONTAINS。如果启用了System.defaultCompareForStringContains，则无论区分大小写如何，所有字符串数据的默认值均为CONTAINS。可以在DESIGN语句中覆盖。

  • EVENTID eventId

    为属性指定特殊输入模式。

    • eventId

      字符串字面量。现在仅支持SCANNER值。启用特殊的keydown处理模式，以检测GS（组分隔符）输入。

  • LAZY [WEAK | STRONG]

    指定属性的缓存级别。 WEAK表示在应用程序服务器上缓存属性值（如果是为所有固定参数读取）。 STRONG表示缓存不会在任何属性更改时完全清除，而是会触发事件，并且特定值将被清除。 默认值为WEAK。

示例​

// 通过上下文无关的DATA属性操作符定义的属性
cost 'Cost' (i) = DATA NUMERIC[12,3] (Item);

// 通过表达式定义的属性
weightedSum 'Weighted amount' (a, b) = 2*a + 3*b;

// 此属性的标题将为'diff'，参数为(a, b)
diff = a - b;

// 通过DATA操作符和附加属性选项定义的属性
teamName 'Team name' = DATA BPSTRING[30](Team) IN baseGroup TABLE team;
