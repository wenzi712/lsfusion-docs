通过HTTP协议交互

示例 1​

任务​

我们有一组与国家关联的城市。

CLASS Country '国家';
id '代码' = DATA STRING[20] (Country) IN id;
name '名称' = DATA ISTRING[100] (Country) IN id;

country (STRING[20] id) = GROUP AGGR Country c BY id(c);

CLASS City '城市';
name '名称' = DATA ISTRING[100] (City) IN id;

country '国家' = DATA Country (City);
nameCountry '国家' (City c) = name(country(c));

FORM cities '城市列表'
    OBJECTS c = City
    PROPERTIES(c) name, nameCountry, NEW, DELETE
;

NAVIGATOR {
    NEW cities;
}

我们需要向特定URL发送一个HTTP请求，以JSON格式添加城市。

解决方案​

postCity '发送' (City c)  {
    EXPORT JSON FROM countryId = id(country(c)), name = name(c);

    LOCAL result = FILE();
    EXTERNAL HTTP 'http://localhost:7651/exec?action=Location.createCity' PARAMS exportFile() TO result;

    LOCAL code = STRING[10]();
    LOCAL message = STRING[100]();
    IMPORT JSON FROM result() TO() code, message;
    IF NOT code() == '0' THEN {
        MESSAGE '错误: ' + message();
    }
}

EXTEND FORM cities
    PROPERTIES(c) postCity
;

EXPORT操作符将创建一个FILE格式的JSON，然后将其写入exportFile属性。以下是生成文件的示例：

{"countryId":"123","name":"旧金山"}

然后我们调用EXTERNAL操作符，它向指定的URL发送请求，并将生成的文件内容作为Body传递。在这种情况下，由于FROM块中的属性类型为JSON，因此将使用application/json作为内容类型。URL中编码了<namespace>.<property name>。在这种情况下，被调用的createCity操作的命名空间是Location。所有参数都以ID p的形式依次传递。服务器的响应将被写入result属性。假设响应以JSON格式接收，并且具有以下类型之一：

{"code":"0","message":"OK"}

{"code":"1","message":"无效的国家代码"}

响应由IMPORT操作符处理，该操作符将相应的参数分别解析到code和message属性中。如果发生任何错误，用户将看到相应的错误消息。

示例 2​

任务​

与示例 1类似。

我们需要处理传入的HTTP请求，并根据请求中提供的参数在数据库中创建新城市。

解决方案​

createCity (FILE f)  {

    LOCAL cy = STRING[20] ();
    LOCAL ne = STRING[100] ();

    IMPORT JSON FROM f AS FILE TO() cy = countryId, ne = name;

    IF NOT country(cy()) THEN {
        EXPORT JSON FROM code = '1', message = '无效的国家代码';
        RETURN;
    }

    NEW c = City {
        name(c) <- ne();
        country(c) <- country(cy());

        APPLY;
    }

    EXPORT JSON FROM code = '0', message = 'OK';
}

由于该属性名为createCity，并且位于命名空间为Location的模块中，因此处理请求的URL如下所示：

http://localhost:7651/exec?action=Location.createCity

HTTP请求的Body将作为FILE类型的参数传递。从countryId和name参数读取的值分别写入本地属性cy和ne。

如果没有具有相应代码的国家，则生成与前面示例类似的JSON文件，并调用RETURN操作符中止执行。默认情况下，响应消息值也存储在exportFile属性中。

如果所有操作都成功完成，则生成相应的"OK消息"作为响应。

示例 3​

任务​

我们有图书订单的逻辑。

CLASS Book '图书';
id '代码' = DATA STRING[10] (Book) IN id;
name '名称' = DATA ISTRING[100] (Book) IN id;

book (STRING[10] id) = GROUP AGGR Book b BY id(b);

CLASS Order '订单';
date '日期' = DATA DATE (Order);
number '编号' = DATA STRING[10] (Order);

CLASS OrderDetail '订单明细';
order '订单' = DATA Order (OrderDetail) NONULL DELETE;

book '图书' = DATA Book (OrderDetail) NONULL;
nameBook '图书' (OrderDetail d) = name(book(d));

quantity '数量' = DATA INTEGER (OrderDetail);
price '单价' = DATA NUMERIC[14,2] (OrderDetail);

FORM order '订单'
    OBJECTS o = Order PANEL
    PROPERTIES(o) date, number

    OBJECTS d = OrderDetail
    PROPERTIES(d) nameBook, quantity, price, NEW, DELETE
    FILTERS order(d) == o

    EDIT Order OBJECT o
;

FORM orders '订单列表'
    OBJECTS i = Order
    PROPERTIES(i) READONLY date, number
    PROPERTIES(i) NEWSESSION NEW, EDIT, DELETE
;

NAVIGATOR {
    NEW orders;
}

我们需要向特定URL发送一个HTTP请求，以JSON格式创建新订单。

解决方案​

FORM exportOrder
    OBJECTS order = Order PANEL
    PROPERTIES dt = date(order), nm = number(order)

    OBJECTS detail = OrderDetail
    PROPERTIES id = id(book(detail)), qn = quantity(detail), pr = price(detail)
    FILTERS order(detail) == order
;

exportOrder '发送' (Order o)  {
    EXPORT exportOrder OBJECTS order = o JSON;

    LOCAL result = FILE();
    EXTERNAL HTTP 'http://localhost:7651/exec?action=Location.importOrder' PARAMS exportFile() TO result;
}

EXTEND FORM orders
    PROPERTIES(i) exportOrder;
;

要创建带有嵌套标签的JSON，我们需要创建一个表单，其中包含通过FILTERS操作符块链接的相应对象。根据对象之间的依赖关系，系统将生成具有相应结构的JSON文件。在考虑的示例中，输出的JSON结构将如下所示：

{
   "dt":"20.08.18",
   "nm":"1",
   "detail":[
      {
         "pr":5.99,
         "id":"b1",
         "qn":3
      },
      {
         "pr":6.99,
         "id":"b2",
         "qn":2
      }
   ]
}

我们没有为order创建自定义标签，因为对象值作为参数传递给EXPORT操作符。 在此示例中，忽略了HTTP请求的响应。

示例 4​

任务​

与示例 3类似。

我们需要处理传入的HTTP请求，并根据请求中提供的参数在数据库中创建新订单。

解决方案​

date = DATA LOCAL DATE();
number = DATA LOCAL STRING[10]();

id = DATA LOCAL STRING[10] (INTEGER);
quantity = DATA LOCAL INTEGER (INTEGER);
price = DATA LOCAL NUMERIC[14,2] (INTEGER);
FORM importOrder
    PROPERTIES dt = date(), nm = number()

    OBJECTS detail = INTEGER
    PROPERTIES id = id(detail), qn = quantity(detail), pr = price(detail)
;

importOrder (FILE f)  {
    IMPORT importOrder JSON FROM f;

    NEW o = Order {
        date(o) <- date();
        number(o) <- number();
        FOR id(INTEGER detail) DO NEW d = OrderDetail {
            order(d) <- o;
            book(d) <- book(id(detail));
            quantity(d) <- quantity(detail);
            price(d) <- price(detail);
        }

        APPLY;
    }
}

要导入JSON格式的相应文件，我们需要创建一个结构类似的表单，除了将INTEGER类型用作对象类。在导入过程中，标签值将被放置在具有相应名称的属性中。date和number属性没有参数，因为它们在JSON中的值是在最顶层提供的。

示例 5​

任务​

与示例 4类似。

我们需要向特定URL发送HTTP请求，以JSON格式创建订单，如前一个示例所示，但所有内容必须包装在order标签中。

解决方案​

GROUP order;
FORM exportOrderNew
    OBJECTS o = Order
    PROPERTIES IN order dt = date(o), nm = number(o)

    OBJECTS detail = OrderDetail IN order
    PROPERTIES id = id(book(detail)), qn = quantity(detail), pr = price(detail)
    FILTERS order(detail) == o
;

exportOrderNew '发送（新格式）' (Order o)  {
    EXPORT exportOrderNew OBJECTS o = o JSON;

    LOCAL result = FILE();
    EXTERNAL HTTP 'http://localhost:7651/exec?action=Location.importOrderNew' PARAMS exportFile() TO result;
}

EXTEND FORM orders
    PROPERTIES(i) exportOrderNew;
;

与前一个示例不同，这里我们使用GROUP操作符创建一个名为order的属性组。在声明表单时，我们将采购订单的所有属性以及detail对象放入此属性组中。结果JSON将如下所示：

{
   "order":{
      "dt":"20.08.18",
      "nm":"1",
      "detail":[
         {
            "pr":5.99,
            "id":"b1",
            "qn":3
         },
         {
            "pr":6.99,
            "id":"b2",
            "qn":2
         }
      ]
   }
}

解决方案​

FORM importOrderNew
    PROPERTIES IN order dt = date(), nm = number()

    OBJECTS detail = INTEGER IN order

;

importOrderNew (FILE f)  {
    IMPORT importOrderNew JSON FROM f;

    NEW o = Order {
        date(o) <- date();
        number(o) <- number();
        FOR id(INTEGER detail) DO NEW d = OrderDetail {

与导出过程一样，我们将所有属性和detail对象放在order组中，以正确接收新版本的JSON。

示例 7​

任务​

与示例 3类似。

我们需要使用HTTP GET请求返回给定日期的订单编号列表，其中提供了此日期。

解决方案​

FORM exportOrders
    OBJECTS date = DATE PANEL

    OBJECTS order = Order
    PROPERTIES nm = number(order)
    FILTERS date(order) = date
;

getOrdersByDate (DATE d) {
    EXPORT exportOrders OBJECTS date = d JSON;
}

HTTP请求应发送到的URL将如下所示：http://localhost:7651/exec?action=Location.getOrdersByDate&p=12.11.2018。

响应JSON将如下所示：

{
    "order": [
        {
            "nm": "42"
        },
        {
            "nm": "65"
        }
    ]
}

示例 8​

任务​

与示例 3类似。

每个订单都有一个附加的文件列表。

CLASS Attachment '附件';
order = DATA Order (Attachment) NONULL DELETE;
name '名称' = DATA STRING (Attachment);
file = DATA FILE (Attachment);

我们需要实现HTTP GET请求，该请求将为传递的订单内部标识符返回其参数和文件列表。 此外，我们还需要实现一个单独的请求来检索特定文件的内容。

解决方案​

FORM orderAttachments
    OBJECTS o = Order
    PROPERTIES(o) number, date

    OBJECTS attachments = Attachment
    PROPERTIES id = VALUE(attachments), name(attachments)
;

getOrderAttachments (LONG orderId) {
    FOR LONG(Order o AS Order) = orderId DO {
        EXPORT orderAttachments OBJECTS o = o JSON;
    }
}

要生成HTTP请求，请使用以下URL：http://localhost:7651/exec?action=getOrderAttachments&p=32178。 其中_p_参数包含订单内部标识符。

响应将返回例如以下JSON：

{
    "date": "20.10.2021",
    "number": "12",
    "attachments": [
        {
            "name": "文件 1",
            "id": 32180
        },
        {
            "name": "文件 2",
            "id": 32183
        }
    ]
}

_id_属性将包含内部文件标识符。然后可以通过使用以下URL的查询读取这些文件的内容： http://localhost:7651/exec?action=getOrderAttachment&p=32180。_getOrderAttachment_操作声明如下：

getOrderAttachment (LONG id) {
    FOR LONG(Attachment a AS Attachment) = id DO
        exportFile() <- file(a); 
}

查询将返回一个文件，其_Content-Type_对应于_file_属性的扩展名。
