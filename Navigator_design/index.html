导航器设计

导航器设计决定了导航器在用户设备屏幕上的显示方式（我们将导航器显示的屏幕空间称为桌面）。

导航器设计由多个窗口组成，这些窗口是桌面组件，每个窗口显示特定的导航器元素。

每个窗口必须有一个标题，该标题将在用户界面中显示窗口时使用。

窗口中的导航器元素布局​

对于每个导航器元素，您可以指定其子元素应显示在哪个窗口中。必要时，元素本身也可以显示在那里。因此，每个窗口中显示的子树（导航器元素）集合是唯一确定的。图形表示如下：

每个窗口占据桌面的预定义部分。图形表示如下：

整个桌面的大小为100 x 100像素。创建窗口时，必须指定窗口的左上角坐标、宽度和高度，以像素表示。理想情况下，窗口应"覆盖"整个桌面区域。如果没有覆盖，那么空闲区域将分配给其中一个窗口（不保证是哪个窗口）。允许两个窗口具有完全相同的坐标和大小。在这种情况下，它们将显示在同一位置，但可以使用选项卡在它们之间切换。

选定文件夹​

在任何时刻，每个窗口中都可以有一个当前用户选定的导航器文件夹。因此，如果元素项属于其父文件夹窗口之外的窗口，则该元素仅当其父文件夹在其窗口中被选定时才会在其窗口中显示。预定义的System.root文件夹始终被视为已选定。如果某个窗口在某个时刻不显示任何导航器元素，则该窗口会自动隐藏。

系统窗口​

有几个预定义的系统窗口是客户端应用程序工作所必需的：

  • System.forms：用户表单在其中打开的窗口。
  • System.log：向用户显示消息的窗口。如果此窗口不可见，消息将以系统对话框表单的形式显示给用户。

此外，为了方便开发，还会自动创建额外的窗口：

  • System.root：水平工具栏，建议在其中显示导航器元素System.root的子元素。导航器文件夹System.root默认显示在这里。
  • System.toolbar：垂直工具栏，建议在其中显示一些显示在System.root窗口中的导航器元素的后代。
  • System.system：带有系统按钮的工具栏，提供对主要界面功能的便捷访问。
  • System.logo：用于显示徽标的窗口。

语言​

要管理窗口，请使用WINDOW语句。

示例​

// 在System模块中创建系统窗口
WINDOW logo HORIZONTAL POSITION(0, 0, 10, 6) VALIGN(CENTER) HALIGN(START) HIDETITLE HIDESCROLLBARS CLASS logoWindowClass();
WINDOW root HORIZONTAL POSITION(10, 0, 70, 6) VALIGN(CENTER) HALIGN(CENTER) HIDETITLE HIDESCROLLBARS CLASS rootWindowClass();
WINDOW system HORIZONTAL POSITION(80, 0, 20, 6) VALIGN(CENTER) HALIGN(END) HIDETITLE HIDESCROLLBARS CLASS systemWindowClass();

WINDOW toolbar VERTICAL POSITION(0, 6, 20, 94) HIDETITLE CLASS toolbarWindowClass();

// 桌面底部的水平工具栏，其中所有按钮将居中且文本对齐向上
// 在这个工具栏中，例如，可以放置用于快速打开的表单
WINDOW hotforms HORIZONTAL BOTTOM VALIGN(CENTER) TEXTVALIGN(START);
