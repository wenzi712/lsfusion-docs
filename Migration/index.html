迁移

基于lsFusion的信息系统使用关系数据库管理系统进行永久数据存储。需要记住的是，在对系统逻辑进行某些更改后，平台无法确定数据应如何迁移。在这些情况下，开发人员必须通过创建特殊的迁移文件migration.script来明确定义迁移方法，该文件应位于应用服务器的CLASSPATH中。

迁移文件由描述数据库结构指定版本中所做更改的块组成。服务器启动时，将应用迁移文件中所有版本高于数据库中存储版本的更改。更改按照版本号从低到高的顺序应用。如果数据库结构更改成功，所有应用块的最大版本将作为当前版本写入数据库。每个块的描述语法如下：

V<版本号> {
    change1
    ...
    changeN 
}

版本号是一组由点分隔的一个或多个数字。比较两个版本的数字时，首先比较版本的第一个数字，如果相等则比较第二个数字，依此类推。如果一个版本包含的数字少于另一个版本，则在比较过程中向数字较少的版本添加零。例如，版本号1.3等同于1.3.0.0，版本1.2高于版本1.1.3。在迁移文件中，版本号用大写字母V表示：V1.0、V2.0.11。

迁移文件允许您处理系统元素的规范名称更改，这些更改在重命名和/或转移到另一个命名空间时发生。更改类型如下：

PROPERTY oldNS.oldName[class1,...,classN] -> newNS.newName[class1,...,classN]
STORED PROPERTY oldNS.oldName[class1,...,classN] -> newNS.newName[class1,...,classN]
FORM PROPERTY oldNS.oldFormName.oldName(object1,...,objectN) -> newNS.newFormName.newName(object1,...,objectN)  
CLASS oldNS.oldName -> newNS.newName
OBJECT oldNS.oldClassName.oldName -> newNS.newClassName.newName
TABLE oldNS.oldName -> newNS.newName
NAVIGATOR oldNS.oldName -> newNS.newName

更改属性或动作的名称​

当重命名属性/动作和/或移动到另一个命名空间时，属性/动作的规范名称会更改。在迁移文件中添加指定旧规范名称和新规范名称的PROPERTY更改将允许您保留安全策略设置以及Reflection.properties表中的设置。如果属性是主要的，则在更改此属性的规范名称时必须添加STORED PROPERTY更改以保留数据。然后，在服务器启动时，数据库表中与此属性对应的字段将被重命名。否则，旧字段将被重命名为名称为<old ID>_deleted的字段（例如，在删除属性时），并创建一个具有空值的新字段。除此之外，STORED PROPERTY类型等同于PROPERTY类型。

信息

在STORED PROPERTY和PROPERTY更改的右侧不必指定签名，因为此处的签名会自动从左侧获取。

更改表单上的属性/动作名称​

使用迁移文件更改表单上的属性名称时，您可以保留此属性/动作在表单上表设置的信息。为此，使用FORM PROPERTY更改类型。旧名称和新名称是表单命名空间的名称、表单的名称和表单上属性的名称，用点分隔。此外，使用此类型的更改，您可以在更改表单的规范名称时保留表设置的信息。为此，需要在迁移文件中为表单上所有规范名称已更改的表单的属性/动作添加FORM PROPERTY更改。

更改自定义类的名称​

当重命名自定义类和/或移动到另一个命名空间时，此类的规范名称会更改。在这种情况下，必须在迁移文件中反映这些更改，以保留此类的对象以及与这些对象关联的所有数据。为此，在迁移文件中添加CLASS更改，指定旧类名和新类名。这也会自动重命名此类的静态对象（如果存在）。

信息

值得注意的是，更改类的规范名称可能会导致数据属性的规范名称发生更改。目前这些更改不会自动跟踪，也必须添加到迁移文件中。

更改静态对象的名称​

重命名静态对象时，使用OBJECT更改，这允许保留与对象关联的数据。旧名称和新名称是类命名空间的名称、类的名称和对象的名称，用点分隔。

更改表的名称​

当重命名表和/或移动到另一个命名空间时，此表的规范名称会更改。在这种情况下，创建具有新名称的表后，系统会在单独的请求中自动移动旧表中的所有记录。但是，如果您在迁移文件中添加TABLE更改，指定旧规范表名和新规范表名，则会执行查询来重命名旧表，这将显著更快。

更改导航器元素的名称​

当重命名导航器元素和/或移动到另一个命名空间时，此元素的规范名称会更改。为了保留与此元素关联的安全策略设置，请在迁移文件中添加NAVIGATOR更改，指定导航器元素的旧规范名称和新规范名称。

更改命名空间的名称​

由于命名空间的名称用于系统元素的规范名称，更改它会导致包含在其中的系统元素的规范名称发生更改。因此，如果命名空间名称发生更改，则必须将所有上述元素的信息放入迁移文件中。当将系统元素移动到不同的命名空间时，也必须执行相同的操作。

示例​

migration.script

V0.3.1 {
    STORED PROPERTY Item.gender[Item.Article] -> Item.dataGender[Item.Article] // DATA属性更改 
    PROPERTY System.SIDProperty[Reflection.Property] -> Reflection.dbNameProperty[Reflection.Property] // 并行转移到另一个命名空间并更改属性名称
    FORM PROPERTY Item.itemForm.name(i) -> Item.itemForm.itemName(i)
}
 
V0.4 {
    FORM PROPERTY Document.documentForm.name(i) -> Document.itemForm.itemName(i)
    FORM PROPERTY Item.itemForm.itemName(i) -> Item.itemForm.iname // 为表单上的属性添加显式名称：iname = itemName(i)
    CLASS Date.DateInterval -> Date.Interval
    OBJECT Geo.Direction.North -> Geo.Direction.north
    TABLE User.oldTable -> User.newTable
}
