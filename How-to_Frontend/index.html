前端集成

组织 React 应用程序与基于 lsFusion 的应用程序交互的最简单方法是通过 JSON API 进行HTTP协议通信。要导出满足指定条件的对象列表，最方便的是使用位于 eval/action URL 的接口。您可以在 BODY 中向其传递 lsFusion 程序代码，然后它将执行这些代码。如果需要从请求中返回数据，请使用EXPORT 操作符。默认情况下，它以 JSON 格式返回数据，然后可以使用 JavaScript 轻松处理。

信息

在这些示例中，我们将进行无需授权的请求。要使服务器开始接受无需授权的请求，您需要在 管理 > 设置 表单的 设置 选项卡上将 enableAPI 选项设置为值 2。

更安全的方法是为每个请求创建一个单独的操作，并添加 @@api 标记，然后将 enableAPI 设置为值 0。在这种情况下，服务器将只接受针对这些特定操作的请求，并且仅接受来自授权用户的请求。在这些操作内部，您可以在开始处理之前使用 currentUser() 属性检查用户是否有权限。

示例 1​

为了清晰起见，我们实现了比分表示例中的简单表单（只读模式）。

我们将读取平面数据集实现为一个带有一个参数的函数——查询文本，在其左侧添加 EXPORT FROM：

const url = "https://demo.lsfusion.org/hockeystats/eval/action";
function select(script) {
  const params = {
    method: "post",
    body: "EXPORT FROM " + script
  }
  return fetch(url, params).then(response => response.json());
}

示例 2​

让我们考虑一个稍微复杂一点的情况，当您需要根据用户选择的数据进行参数化查询时。对于后端逻辑，我们将使用物料流程管理示例。

假设我们需要构建一个表单，其中需要显示发货单，并能够按日期和仓库进行过滤。当用户选择特定文档时，应显示其行项目。

为了实现向后端发送请求以接收带过滤的发货单，我们声明一个函数：

const url = "https://demo.lsfusion.org/mm/eval/action";
function select(script, data) {
  var formData = new FormData();

  formData.append("script", "EXPORT FROM " + script);
  for (var name in data) {
    formData.append(name, data[name]);
  }
  const params = {
    method: "post",
    headers: {
      "Content-type": "multipart/form-data"
    },
    body: formData
  };
  return fetch(url, params).then(response => response.json());
}

第一个 select 函数使用内容类型 multipart/form-data 发出 POST 请求，将请求文本作为其第一个参数传递给服务器，并将请求的值作为其他参数传递。

例如，形式如下的函数调用

select("id = Shipment s, number(s) WHERE date(s) = $1", { date: new Date().toISOString().substr(0, 10) })

将返回今天日期的所有发货单。需要注意的是，日期参数的名称在后端不使用，可以是任意内容。只有参数的顺序很重要。

为了方便，我们还声明了一个函数，该函数将通过仅添加值不为空的参数来生成请求：

function selectWhere(script, wheres) {
  var exprs = [], params = {};
  for (var i = 0; i < wheres.length; i++) {
    if (wheres[i].value != null) {
      exprs.push(wheres[i].expr + "=$" + (i + 1));
      params = { ...params, ...{ ["p" + i]: wheres[i].value } };
    }
  }
  return select(script + (exprs.length > 0 ? " WHERE " : "") + exprs.join(" AND "), params);
}

上面相应的函数调用可以替换为：

selectWhere("id = Shipment s, number(s)", [{ expr: "date(s)", value : new Date().toISOString().substr(0, 10) }])

使用上述请求，我们实现所需的逻辑。为此，我们定义了两个组件：

  • Shipments，它将使用 List 组件（来自上述的 Material-UI）显示文档列表。它还将包含一个 Filters 组件，用户将使用该组件设置过滤参数。
  • Details，它将显示所选文档的行项目。如果需要，它可以嵌入到 Shipments 组件中。

示例 3​

我们实现一个简单的商品输入 CRUD 界面。

由于这需要更改数据库中的信息，我们声明一个函数，该函数将在平台上执行任意代码（带有错误处理）：

function handleErrors(response) {
  if (!response.ok) {
    response.text().then(text => console.log(text));
    throw Error(response);
  }
  return response;
}

function evaluate(script, data) {
  var formData = new FormData();


  formData.append("script", script);
  for (var name in data) {
    formData.append(name, data[name]);
  }
  const params = {
    method: "post",
    headers: {
      "Content-type": "multipart/form-data"
    },
    body: formData
  };
  return fetch(url, params).then(handleErrors);
}

例如，要在数据库中创建一个项目，您需要执行以下代码：

evaluate("NEW s = Item { name(s) <- $1; barcode(s) <- $2; salePrice(s) <- $3; APPLY; }", { name : "My item", barcode : "4341", salePrice : 2.34 } )

与上面的示例一样，参数的名称并不重要：它们的顺序才是重要的。APPLY 操作符将更改保存到数据库。没有它，数据将不会被保存，并将在请求完成时被丢弃。

要更改产品的属性，您可以使用以下代码（其中 id 是产品的内部 ID）：

evaluate("FOR Item s = $0 DO { name(s) <- $1; barcode(s) <- $2; salePrice(s) <- $3; APPLY; }", { id : 32494, name : "My item", barcode : "4341", salePrice : 2.34 })

要删除给定的产品：

evaluate("DELETE s WHERE s AS Item = $1; APPLY; ", { id : 32494 })

示例 4​

在这个示例中，我们实现了查看和编辑带行项目的文档列表的功能。

我们需要将特定文档的行项目集传递给服务器。为了通过一个请求执行所有更改，而不是必须为每一行发出单独的请求，我们将它们作为 JSON 格式文件中的参数传递。为此，我们修改 evaluate 函数如下：

function isObject(obj) {
  return obj === Object(obj);
}
function evaluate(script, data) {
  var formData = new FormData();


  formData.append("script", script);
  console.log(data);
  for (var name in data) {
    if (isObject(data[name]))
      formData.append(
        name,
        new Blob([JSON.stringify(data[name])], { type: "application/json" })
      );
    else formData.append(name, data[name]);
  }
  const params = {
    method: "post",
    headers: {
      "Content-type": "multipart/form-data"
    },
    body: formData
  };
  return fetch(url, params).then(handleErrors);
}

如果对象作为 data 对象中的字段传递，现在会将其转换为 JSON 并作为 application/json 类型的文件传递。

为了接受此文件作为最终参数（在此示例中为 $5）并通过删除旧行并创建新行来更新行项目，我们将在平台上使用以下代码：

IMPORT JSON FROM $5 AS FILE FIELDS LONG item, NUMERIC[16,3] quantity, NUMERIC[16,3] price, NUMERIC[16,3] sum DO 
    NEW d = ShipmentDetail { 
        shipment(d) <- s; 
        item(d) <- GROUP MAX i AS Item IF LONG(i) = item; 
        quantity(d) <- quantity; 
        price(d) <- price; 
        sum(d) <- sum; 
    }

为了解析 JSON，我们使用特殊的 IMPORT 操作符。它假设它将接收一个平面 JSON，作为包含数值字段 item、quantity、price 和 sum 的对象数组。将为每个对象创建一个 ShipmentDetail。然后将其链接到 Shipment 类的相应对象 s。之后，将写入内部代码等于传递值的 SKU，然后写入其他属性。更复杂的 JSON 解析可以在数据导入和通过 HTTP 协议交互示例中找到。

保存更改可能会违反约束。在这种情况下，更改将不会保存到数据库（但是，它们将保留在更改会话中）。在这种情况下，值 TRUE 将被写入 canceled 属性，约束消息将被写入 applyMessage 属性。为了处理这种情况，我们将在平台上使用以下代码：

APPLY; 
IF canceled() THEN 
    EXPORT FROM message = applyMessage();

如果更改失败，消息将以 JSON 形式在响应的 BODY 中返回，带有单个字段 applyMessage。

传递给 evaluate 的最终代码（例如，用于创建文档）将如下所示：

NEW s = Shipment {
    number(s) <- $1; date(s) <- $2; customer(s) <- $3; stock(s) <- $4;
    IMPORT JSON FROM $5 AS FILE FIELDS LONG item, NUMERIC[16,3] quantity, NUMERIC[16,3] price, NUMERIC[16,3] sum DO 
        NEW d = ShipmentDetail { 
            shipment(d) <- s; 
            item(d) <- GROUP MAX i AS Item IF LONG(i) = item; 
            quantity(d) <- quantity; 
            price(d) <- price; 
            sum(d) <- sum; 
        }
    APPLY; 
    IF canceled() THEN 
        EXPORT FROM message = applyMessage();
}

相应地，对于编辑（首先，删除现有行）：

FOR Shipment s = $0 DO {
    number(s) <- $1; date(s) <- $2; customer(s) <- $3; stock(s) <- $4;
    DELETE ShipmentDetail d WHERE shipment(d) = s;
    IMPORT JSON FROM $5 AS FILE FIELDS LONG item, NUMERIC[16,3] quantity, NUMERIC[16,3] price, NUMERIC[16,3] sum DO 
        NEW d = ShipmentDetail { 
            shipment(d) <- s; 
            item(d) <- GROUP MAX i AS Item IF LONG(i) = item; 
            quantity(d) <- quantity; 
            price(d) <- price; 
            sum(d) <- sum; 
        }
    APPLY; 
    IF canceled() THEN 
        EXPORT FROM message = applyMessage();
}

为了处理约束违反错误（因为在此实现中，两种情况下的响应状态都是 200），我们假设如果答案的 BODY 为空，则没有错误。如果有任何文本，则这是包含约束文本的 JSON。例如，我们可以使用以下代码：

evaluate("...").then(response =>
    response.text().then(text => {
        if (text === "") {
            // ok
        } else {
            console.log(JSON.parse(text).applyMessage);
        }
      })
    );

返回的 JSON 还包含其他实用字段。

对象表可以分别从 game.list 和 team.list 读取。当前对象存储在 game.value 和 team.value 中。我们使用这些数据形成两个表：游戏表和团队表。为了标记当前行，我们使用 equals 函数，该函数比较例如 game.value 和 game.list[<row>].value 的值。这是必要的，因为如果在对象组中声明了多个对象，value 可能包含更复杂的对象。

在表单加载之前，辅助函数 formCreated 返回 false，表不显示。

if (!formCreated(this.state)) return <div>Loading</div>;

numberOfPendingRequests 函数还允许我们在表单上显示尚未收到响应的服务器请求数量。

<div>
{numberOfPendingRequests(this.state) > 0
    ? "Loading, number of requests : " +
    numberOfPendingRequests(this.state)
    : ""}
</div>

使用 change 函数更改表单上的当前对象及其属性值。

change(updateState => this.setState(updateState), changes);

它接受一个将更改当前状态的回调和一个存储更改列表的对象。表单的当前状态（App 对象）在每次调用 change 后不断更新，以新值为准，同时考虑用户之前所做的更改。

例如，要更改当前选择的游戏，我们可以使用以下调用：

change(updateState => this.setState(updateState), { game : { value : 6063 } });

这里 6063 是所选对象的 value。

要更改属性的值，我们可以使用以下代码：

change(updateState => this.setState(updateState), { game : { value : 6063, hostGoals : 3 } });

此调用将当前对象更改为 ID 为 6063 的游戏，并将主队进球的值更改为传递的值（3）。如果不传递 value 标签，则将对先前选择的游戏进行进球更改。

相同的方案用于更改不是数据的属性，而是对象属性的简单组合：

change(updateState => this.setState(updateState), { game : { value : 6063, hostTeamName: "Montreal Canadiens" } });

系统本身将找到具有给定名称的团队，并将主队的值更改为它找到的团队。

要从列表中选择团队，您可以使用 react-select 组件。可以使用第一个示例中声明的 select 函数来组织读取多个团队。
