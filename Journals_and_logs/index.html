日志与记录

日志​

  • 错误日志（图1）。

该日志包含操作过程中发生的所有错误。错误分为以下几类（Object class列）：

  • 服务器上发生的错误——错误显示为白色背景，并包含在Exception on server类中；
  • 服务器上发生并被客户端应用程序接收的错误——错误显示为粉色背景，并包含在Exception on server (from client)类中；
  • 客户端应用程序中发生的错误——错误显示为黄色背景，并分为两个类：Exception on client和Exception on web client；
  • 连接错误——错误显示为蓝色背景，并分为两个类：
    • Temporary connection loss - 与服务器的连接被中断，但随后恢复；
    • Permanent connection loss - 与服务器的连接被中断且无法恢复；

图1 错误日志。

Exception trace部分显示错误的java堆栈；Exception LSF trace显示lsfusion堆栈。

  • 连接日志（图2）。

图2 连接日志。

日志存储有关连接到系统的用户的信息，从哪台计算机连接，该计算机的特性，以及连接/断开连接的日期和时间信息。在表单上，您可以显示当前正在使用数据库的用户：Active connections（活动连接）。

Form（表单）部分显示用户进入了哪些表单以及次数。Session（会话）部分允许您跟踪某些表单的更改何时应用。

  • 启动日志存储应用服务器启动（重启）的日期和时间信息。您还可以看到安装服务器的计算机名称和应用程序版本（如果在构建期间填写）。

  • 更改日志包含有关在连接日志的Session部分中反映的更改的更详细信息（图3）。

图3 更改日志。

Change（更改）列显示值发生变化的属性（列）列表，以及更改的数量（行）。只有当前表单中的更改才会被记录：在其他表上同时更改的依赖属性不会出现在此列表中。

表单允许您通过勾选Only user changes（仅用户更改）来过滤用户进行的更改（排除系统更改）。

  • 客户端应用程序日志包含有关在给定时间段内与应用服务器一起工作时的连接质量的信息（图4）。

图4 客户端应用程序日志。

除了系统内存指标外，您还可以在表单上部分析平均响应时间（ping）（以毫秒为单位）以及客户端计算机上java应用程序可用和使用的内存。要分析的时间段通过在Date and time（日期和时间）部分输入Date from（开始日期）和Date to（结束日期）来设置。除了日期外，您还可以在此处为相同的指标（ping和内存）设置阈值——这允许您获取客户端PC超过阈值的总时间（以秒为单位）。

底部的Data（数据）选项卡提供响应时间以及可用和使用内存变化的时间顺序。此信息仅对桌面客户端显示。

这些日志中应该存储多少信息在Admin form > Settings > Logging tab（管理表单>设置>日志选项卡）中指定（图5）。

图5 设置日志保留天数。

如果您需要跟踪任何属性（列）的各个值的更改，则开发了用户日志记录机制以允许您这样做。例如，假设您需要记录员工目录中员工姓氏的更改。为此：

  • 转到Surname（姓氏）列中的任何条目，右键单击以调出Configure property policy（配置属性策略）菜单（图6）；

图6 选择要记录更改的属性。

  • 在Security policy（安全策略）表单中，勾选Logged by user（由用户记录）并点击OK（确定）（图7）；

图7 设置用户日志记录。

  • 一旦您重新启动应用服务器，右键单击Surname（姓氏）属性将调出一个额外的Show Change History（显示更改历史记录）菜单项。如果当前记录的姓氏已被某人更改，则这将反映在属性更改历史记录中（图8）。

图8. 属性更改历史记录。

这些日志的保留时间设置为与更改日志相同的保留时间。

日志文件​

平台为每个平台组件支持以下日志集：

组件文件夹日志文件
应用服务器（Server）$FUSION_DIR$/logs，其中$FUSION_DIR$是应用服务器启动文件夹
  • stdout - 标准输出日志（输出到标准输出流，即OS控制台、IDE等）。包括start和explain日志。
  • stderr - 一般错误日志
  • start - 停止和启动过程的日志
  • remote、invocation - 与访问应用服务器相关的过程日志
  • sql、sqlhand、sqlconnection、sqlconflict、sqladjust - 与访问数据库服务器相关的过程日志
  • explain、explaincompile - 显示查询计划的日志（分别为数据库服务器和应用服务器）
  • lru - 内存管理过程日志（主要是LRU缓存）
  • allocatedbytes - 内存分配过程日志
  • assert - 各种检查是否满足指定条件的日志（或者更确切地说，不满足的情况）
  • mail - 邮件日志
  • jasperReports - JasperReports日志
  • jdbc - jdbc驱动日志
  • exinfo - 附加信息日志（未包含在上述内容中）
Web服务器（Client）$CATALINA_BASE$/logs，其中$CATALINA_BASE$是Tomcat安装文件夹
  • catalina.out - 一般输出日志
  • gwtlog、gwtlog-err - GWT日志
  • invocation - 与访问Web服务器相关的过程日志
桌面客户端$USER_DIR$/.fusion/logs，其中$USER_DIR$是用户文件夹
  • stdout - 标准输出日志（输出到标准输出流，即OS控制台、IDE等）。
  • stderr - 一般错误日志
  • remote、invocation - 与访问应用服务器相关的过程日志
  • jasperReports - JasperReports日志
信息

在Linux下自动安装时，这些文件夹的符号链接（以及lsFusion启动参数的符号链接）会自动创建到其他文件夹，其位置更符合Linux理念。
