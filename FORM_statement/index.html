FORM 语句

FORM 语句创建一个表单。

语法​

FORM name [caption] formOptions
    formBlock1
    ...
    formBlockN
;

指定表单名称和标题后，按任意顺序指定表单选项 formOptions

imageSetting
LOCALASYNC

在表单选项之后，按任意顺序描述表单的块 formBlock1 ... formBlockN：

OBJECTS ... 
TREE ...
PROPERTIES ...
FILTERS ...
[EXTEND] FILTERGROUP ...
USERFILTERS ...
ORDERS ...
EVENTS ...
REPORT propertyExpression
FORMEXTID extID
EDIT className OBJECT objectName
LIST className OBJECT objectName

描述​

FORM 语句声明一个新表单并将其添加到当前模块中。此外，该语句用于描述表单结构，以及其静态和部分交互（除表单设计外）视图。在语句开始时，指定名称和标题，然后是表单选项和包含任意数量的描述表单结构的块的声明。它们可以以任何顺序使用，只要每个块在它使用其元素的块之后声明。每个块可以使用任意次数。

参数​

  • name

    表单名称。简单 ID。名称在当前命名空间中必须是唯一的。

  • caption

    表单标题。字符串字面量。如果未定义标题，则表单的名称将作为其标题。

表单选项 (formOptions)​

  • imageSetting

    表单的图标设置。此选项允许您手动配置图标。它可以有以下形式之一：

    • IMAGE [imageLiteral]

      表单的手动图标指定。如果未提供 imageLiteral，则启用自动分配模式。

      • imageLiteral

        值定义图标的字符串字面量。

    • NOIMAGE

      指示表单不应有图标的关键字。

  • LOCALASYNC

    指示本地事件处理将在更改显示在表单上之后执行的关键字。

表单块 (formBlock1 ... formBlockN)​

  • OBJECTS ...

    向表单添加对象。对象块语法。

  • TREE ...

    向表单添加对象树。对象树块的语法。

  • PROPERTIES ...

    向表单添加属性和操作。属性和操作块的语法。

  • FILTERS ...

    向表单添加固定过滤器。固定过滤器块的语法。

  • [EXTEND] FILTERGROUP ...

    向表单添加过滤器组或扩展现有过滤器组。过滤器组块的语法。

  • USERFILTERS ...

    向表单添加用户定义过滤器。用户过滤器块的语法。

  • ORDER ...

    向表单添加排序选项。排序块的语法。

  • PIVOT ...

    定义数据透视表视图类型的初始设置。数据透视块的语法。

  • EVENTS ...

    定义在特定事件上执行的操作。事件块的语法。

  • FORMEXTID extID

    指定用于导出/导入此表单的名称。仅在结构化视图中使用。

    • extId

      字符串字面量。

  • REPORT propertyExpression

    指定其值将用作空组的报表文件名称的属性。您可以使用表单上已声明对象的名称作为参数。假设这些对象的值将在表单以打印视图打开时传递（如果没有发生，它们将被视为等于 NULL）。

    • propertyExpression

      表达式。

  • EDIT сlassName OBJECT objectName

    将当前表单设置为指定类的所有对象的编辑表单。

    • className

      自定义类的名称。编辑此类的对象时，将打开创建的表单。使用复合 ID定义。

    • objectName

      用于编辑的表单对象的名称。使用简单 ID定义。

  • LIST сlassName OBJECT objectName

    将当前表单设置为指定类的对象的列表表单。

    • className

      将使用创建的表单列出其对象的自定义类的名称。使用复合 ID 定义。

    • objectName

      其当前值将用作所选对象的表单对象的名称。使用简单 ID 定义。

示例​

CLASS Document;

// declaring the Documents form
FORM documents 'Documents'
    // Adding one object of the Document class. The object will be available by this name 
    // in the DESIGN, SHOW, EXPORT, DIALOG, etc. operators.
    OBJECTS d = Document 


    // ... adding properties and filters to the form

    // marking that this form should be used when it is necessary to select a document, 
    // while the d object should be used as a return value
    LIST Document OBJECT d 
;

CLASS Item;

// declaring the Product form
FORM item 'Product'
    // adding an object of the Item class and marking that it should be displayed
    // in the panel (i.e., only one value is visible)
    OBJECTS i = Item PANEL 

    // ... adding properties and filters to the form

    // marking that this form should be used when it is necessary to add or edit a product
    EDIT Item OBJECT i 
;

// declaring a form with a list of Products
FORM items 'Products'
    OBJECTS i = Item

    // ... adding properties and filters to the form

    // adding buttons that will create and edit the product using the item form
    PROPERTIES(i) NEWSESSION NEW, EDIT 
;

CLASS Invoice;
CLASS InvoiceDetail;

// declaring the invoice print form
FORM printInvoice
    OBJECTS i = Invoice // adding an object of the invoice class for which printing will be executed

    // ... adding properties and filters to the form
;

// splitting the form definition into two statements (the second statement can be transferred to another module)
EXTEND FORM printInvoice
    // adding invoice lines, each of which will be used in the report as a detail
    OBJECTS d = InvoiceDetail 

    // ... adding properties and filters to the form
;
// declaring an action that will open the invoice print form
print (Invoice invoice)  { PRINT printInvoice OBJECTS i = invoice; }
