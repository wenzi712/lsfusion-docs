表单结构

表单结构是由对象、属性、操作及其之间的关系组成的复合体。

对象​

创建表单时，首先必须定义它将显示哪些对象。对于每个表单对象，您需要指定其类。

可以将对象组合成对象组。在这种情况下，表格将显示这些对象的"笛卡尔积"（即，对于两个对象 - 所有对，三个对象 - 三元组等）。

根据对象组添加到表单的顺序，形成一个有序的对象组列表。因此，对于某组对象组，编号最大的组称为该组的最后组（即最新的）。一组对象的最后组的确定方式类似：首先构建这些对象所包含的对象组集合，然后为获得的对象组集合确定最后组。

当前对象​

表单上的每个对象在任何时候都有一个当前值。它会因交互视图中的相应用户操作或静态视图中读取数据时"虚拟地"改变。

属性和操作​

定义对象后，可以将属性和操作添加到表单中，并将这些对象作为参数传递给它们。

信息

添加操作仅与交互视图相关。在静态视图中，添加的操作将被忽略。

信息

属性和操作在表单显示上下文中的行为完全相同，因此，在本节的其余部分，我们将仅使用术语属性（操作的行为完全类似）。

显示组​

每个属性恰好显示在表单的一个对象组中（该组称为此属性的显示组）。默认情况下，显示组是此属性接收作为输入的对象集合的最后组。如有必要，开发人员可以明确指定显示组（在结构化视图中使用时有某些约束）。

列内分组​

默认情况下，属性在其显示组中恰好显示一次。在这种情况下，不在此属性显示组中的对象（这些对象称为上层对象）的值用作它们的当前值。但是，也可以多次显示一个属性，以便所有对象集合都用作某些上层对象的值（而不是它们的当前值）。通过这种属性显示，形成一种"矩阵" - 上层对象 x 显示组的对象。因此，要创建这样的矩阵，在将属性添加到表单时，必须指定哪些上层对象（具体来说，对象组）必须用于创建列（这些对象组称为列内分组）。

在确定显示组时，列内分组的属性被忽略。

属性组​

表单上的属性可以组合成组，这些组反过来用于交互（默认设计）和层次结构表单视图。默认情况下，属性全局包含在一个组中（即，这种包含为所有表单的属性一次定义），但是，这种包含可以为特定表单重新定义。

默认设置​

表单上的属性在各种视图中有大量显示设置，其中大多数不仅可以直接为表单上的属性设置，还可以为属性本身设置（在创建它时）。这些设置将是默认设置，即，如果未为表单上的特定属性明确设置设置，则使用属性本身的设置。此外，这些默认设置在使用组合、选择和先前值操作符时会"继承"（即，如果属性f(a)的默认宽度为10，则属性g(a) = f(a) IF h(a)默认也将具有宽度10）。

过滤器​

对于每个表单，开发人员可以创建过滤器，这些过滤器将限制表单上可用于查看/选择的对象集合列表。

要定义过滤器，必须指定将用作过滤条件的属性。过滤器将应用于该属性作为输入的对象集合的最后组的表（即，类似于属性显示组的定义）。在这种情况下，仅显示属性值不为NULL的对象集合（行）。

排序​

默认情况下，在所有对象组视图中，对象集合以某种非确定性顺序显示，这由平台的特定实现确定（最常见的是，内部标识符用于确定顺序）。如有必要，开发人员可以通过指定表单上用作排序的属性列表来明确定义此顺序。同时，对于此列表中的每个属性，可以指定排序是升序还是降序（默认情况下使用升序选项）。

NULL值始终被视为最小值。

语言​

要创建新表单并定义其结构，使用FORM语句。

示例​

CLASS Document;

// declaring the Documents form
FORM documents 'Documents'
    // Adding one object of the Document class. The object will be available by this name
    // in the DESIGN, SHOW, EXPORT, DIALOG, etc. operators.
    OBJECTS d = Document 


    // ... adding properties and filters to the form

    // marking that this form should be used when it is necessary to select a document, 
    // while the d object should be used as a return value
    LIST Document OBJECT d 
;

CLASS Item;

// declaring the Product form
FORM item 'Product'
    // adding an object of the Item class and marking that it should be displayed
    // in the panel (i.e., only one value is visible)
    OBJECTS i = Item PANEL 

    // ... adding properties and filters to the form

    // marking that this form should be used when it is necessary to add or edit a product
    EDIT Item OBJECT i 
;

// declaring a form with a list of Products
FORM items 'Products'
    OBJECTS i = Item

    // ... adding properties and filters to the form

    // adding buttons that will create and edit the product using the item form
    PROPERTIES(i) NEWSESSION NEW, EDIT 
;

CLASS Invoice;
CLASS InvoiceDetail;

// declaring the invoice print form
FORM printInvoice
    OBJECTS i = Invoice // adding an object of the invoice class for which printing will be executed

    // ... adding properties and filters to the form
;

// splitting the form definition into two statements (the second statement can be transferred to another module)
EXTEND FORM printInvoice
    // adding invoice lines, each of which will be used in the report as a detail
    OBJECTS d = InvoiceDetail 

    // ... adding properties and filters to the form
;
// declaring an action that will open the invoice print form
print (Invoice invoice)  { PRINT printInvoice OBJECTS i = invoice; }
