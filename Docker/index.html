Docker 安装

信息

要使用 Docker 容器，您需要安装 Docker 和 Docker Compose。

使用 Docker Compose 启动 lsFusion 平台​

  • 从中央服务器下载 compose.yaml 文件到您选择的文件夹（我们将其称为 $FUSION_DIR$）。该文件包含运行三个容器的设置：

    • PostgreSQL
    • 应用服务器
    • Web 客户端

  • compose.yaml 设置（可选）：

    • 如果您想更改启动设置（例如，使用不同的容器版本或自定义环境变量），请根据 Docker 文档 编辑 compose.yaml 文件。
    • 应用服务器启动参数也可以通过容器的环境变量设置 - 在 environment 属性中。例如，要将服务器区域设置更改为俄语并设置自定义 Xmx 值，请编写： 
      environment:
        - USER_SETLANGUAGE=ru
        - USER_SETCOUNTRY=RU 
        - JAVA_OPTS=-Xmx10g
      Spring 在环境变量中搜索启动参数时，会自动将它们转换为大写并将点替换为下划线。在上面的例子中，环境变量的值将被替换到相应的参数中：user.setLanguage 和 user.setCountry。
    • 可用的 lsFusion 容器镜像：
      • 服务器
      • 客户端

  • 启动容器：

    导航到 $FUSION_DIR$ 文件夹并运行命令：

    docker-compose up -d

    启动完成后，Web 客户端将在 http://localhost:8080/ 可用。

  • 使用项目文件：

    • 首次运行后，$FUSION_DIR$ 文件夹中将创建子文件夹：

      • docker-client-conf - 客户端配置。
      • docker-db - 数据库数据。
      • docker-server - 服务器文件。

      这些目录被绑定挂载到相应的容器中。

    • 在 docker-server 文件夹中放置 lsFusion 语言模块（.lsf 文件或包含它们的文件夹），以及其他资源（报告、Java 文件、图像、CSS、JS 等）。服务器日志和 settings.properties 文件也位于同一文件夹中。

创建和运行项目的 Docker 镜像​

如果您的项目继承了 lsFusion 平台的 Maven 模块 logics，您可以使用内置工具创建 Docker 镜像并生成 compose.yaml 文件。

创建 Docker 镜像​

  • 构建镜像：

    构建 Docker 镜像与 Maven 阶段相关联，并由 docker 配置文件激活。

    • 在 install 阶段，镜像被构建并加载到本地注册表。
    • 在 deploy 阶段，镜像被上传到公共注册表（例如 Docker Hub）。

    要构建镜像，请在项目文件夹中运行命令：

    mvn install -P assemble,docker

    如果您想基于包含 lsFusion 服务器的 uber-jar 构建镜像，请添加 embed-server 配置文件：

    mvn install -P assemble,embed-server,docker

  • 将镜像上传到公共注册表：

    要组装镜像并上传到 Docker Hub，请执行：

    mvn deploy -P assemble,docker

    或者（对于包含服务器的 uber-jar）：

    mvn deploy -P assemble,embed-server,docker

  • 配置镜像名称：

    默认镜像名称为：local/<artifactId>:<version>（artifactId、version 是项目模块的 pom.xml 文件中相应标签的值）。您可以通过 pom.xml 中的 Maven 属性覆盖部分名称或整个名称：

    <properties>.
        <docker.image.namespace>foo</docker.image.namespace>
        <docker.image.repository>bar</docker.image.repository>
        <docker.image.tag>1.0</docker.image.tag>
        <!-- 或者 -->
        <docker.image.fullName>foo/bar:1.0</docker.image.fullName>
    </properties>

生成和使用 compose.yaml​

  • 自动生成：

    • 当您使用带有 docker 配置文件的上述命令之一构建项目时，Maven 会自动生成 compose.yaml 文件。
    • 该文件生成时会替换 lsFusion 平台版本和项目 Docker 镜像的名称。
    • 该文件保存在 target 文件夹或 Maven 的 docker.compose.outputDirectory 属性指定的路径中。此外，文件内容在构建后会打印到控制台。

  • 运行生成的 compose.yaml：

    启动和配置类似于 启动 lsFusion 平台 中描述的步骤，但有一些特定点：

    • Docker Compose 项目名称默认为 artifactId 标签的值。要生成不同的项目名称，请覆盖 Maven 属性 docker.compose.projectName。
