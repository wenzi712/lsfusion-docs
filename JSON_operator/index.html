JSON 操作符

JSON操作符是一个属性，用于从指定的属性或在一般情况下从表单创建JSON。

语法​

JSON FROM [columnId1 =] propertyExpr1, ..., [columnIdN = ] propertyExprN 
  [WHERE whereExpr] [ORDER orderExpr1 [DESC], ..., orderExprL [DESC]]
  [TOP topExpr] [OFFSET offsetExpr]
JSON ( formName [OBJECTS objName1 = expr1, ..., objNameK = exprK]
  [TOP (topExpr | (topGroupId1 = topPropertyExpr1, ..., topGroupIdT = topPropertyExprT))]
  [OFFSET (offsetExpr | (offsetGroupId1 = offsetPropertyExpr1, ..., offsetGroupIdF = offsetPropertyExprF))] )

描述​

JSON操作符是一个属性，用于从指定的属性或表单创建JSON。

当在OBJECTS块中导出表单时，可以添加额外的过滤器来检查表单上的对象与传递的值是否相等。这些对象不会参与构建对象组层次结构。

参数​

  • formName

    要从中导出数据的表单名称。复合ID。

  • objName1 ... objNameK

    为其指定了过滤（固定）值的表单对象名称。简单ID。

  • expr1 ... exprK

    表达式，其值确定表单对象的过滤（固定）值。

  • propertyExpr1, ..., propertyExprN

    表达式列表，数据从其值中导出

  • columnId1, ..., columnIdN

    结果JSON中的列ID列表，相应属性的数据将被导出到其中。每个列表元素是简单ID或字符串字面量。如果未指定ID，则默认等于expr<列号>。

  • whereExpr

    表达式，其值是导出的条件。如果未指定，则等于所有导出属性的析取（即至少一个属性必须非NULL）。

  • orderExpr1, ..., orderExprL

    表达式列表，导出的数据按此排序。只能使用列表propertyExpr1, ..., propertyExprN中存在的属性

  • DESC

    关键字。指定降序排序。默认使用升序排序。

  • TOP topExpr

  • TOP (topExpr | (topGroupId1 = topPropertyExpr1, ..., topGroupIdT = topPropertyExprT))

    仅导出前n条记录，其中n是表达式topExpr的值或组对象topGroupIdT的topPropertyExprT的值。

  • OFFSET offsetExpr

  • OFFSET (offsetExpr | (offsetGroupId1 = offsetPropertyExpr1, ..., offsetGroupIdF = offsetPropertyExprF))

    仅导出偏移量为m的记录，其中m是表达式offsetExpr的值或组对象offsetGroupIdF的offsetPropertyExprF的值。

示例​

FORM testF 
      OBJECTS j = INTEGER
      PROPERTIES ab='34'
      OBJECTS i = INTEGER
      PROPERTIES name = 'Name ' + (i AS INTEGER)
;

run() {
	MESSAGE JSON (testF OBJECTS j=4 FILTERS mod(i,2) = 0);
}
MESSAGE JSON FROM code = '1', message = 'OK';
