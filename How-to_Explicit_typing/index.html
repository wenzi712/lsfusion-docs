显式类型指定

在某些情况下，您可能希望为不同的系统属性或操作使用相同的名称。

例如，让我们创建两个名为sum的属性：一个用于计算订单行的总和，第二个用于计算整个订单的总和：

sum = DATA NUMERIC[10,2] (OrderDetail);
sum = GROUP SUM sum(OrderDetail od) BY order(od);

相应地，第一个属性接收一个OrderDetail类的参数作为输入，而第二个属性接收一个Order类的参数。

但是，如果我们创建一个带有一个参数的约束，然后尝试在不显式指定该参数的类的情况下引用sum属性，将会发生错误：

所有此类引用都需要显式指定类：

CONSTRAINT sum(Order o) < 0 MESSAGE '订单金额必须为正';
