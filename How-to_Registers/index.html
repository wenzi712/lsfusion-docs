账册

累计账册​

假设我们需要实现SKU库存余额计算的逻辑。

REQUIRE Utils;

CLASS SKU 'SKU';
CLASS Stock '仓库';

从理论上讲，我们可以创建一个属性，明确设置所有操作，将所有入库相加并减去所有出库。这种方法的弱点是，每当添加新操作时，都需要将其添加到计算余额的公式中。此外，构建一个包含所有可能影响特定SKU和仓库余额的操作列表的表单将会很困难。模块化也会被破坏，因为声明余额属性的模块将依赖于包含影响它的操作的所有模块。

为了给系统提供高效的扩展性，最好使用账册来实现这种功能。为此，我们引入一个抽象类SKULedger。该类的一个实例将反映一个SKU在一个仓库中余额的单次变化（正或负）。为其设置抽象属性，这些属性需要在实现该类时定义。

信息

所有账册可以有任意数量和类型的维度。在本例中，维度是SKU和仓库。

CLASS ABSTRACT SKULedger '产品余额变更账册';

posted '已完成' = ABSTRACT BOOLEAN (SKULedger);
dateTime '日期/时间' = ABSTRACT DATETIME (SKULedger);

sku 'SKU' = ABSTRACT SKU (SKULedger);
stock '仓库' = ABSTRACT Stock (SKULedger);

quantity '数量' = ABSTRACT NUMERIC[14,2] (SKULedger);

balance '余额' = GROUP SUM quantity(SKULedger l) IF posted(l) BY stock(l), sku(l);

balance '截至日期/时间的余额' = GROUP SUM quantity(SKULedger l) IF posted(l) AND dateTime(l) <= DATETIME dt BY stock(l), sku(l), dateTime(l);

当前余额和特定时间段的余额仅从SKULedger类的属性计算，而不引用特定操作。这段代码可以并且必须在单独的模块中声明。包含特定操作的模块将使用并扩展此类。

例如，让我们看一个操作：仓库入库。

CLASS Receipt '仓库入库';
posted '已完成' = DATA BOOLEAN (Receipt);
dateTime '日期/时间' = DATA DATETIME (Receipt);

stock '仓库' = DATA Stock (Receipt);

CLASS ReceiptDetail '仓库入库行';
receipt '入库' = DATA Receipt (ReceiptDetail) NONULL DELETE;

sku 'SKU' = DATA SKU (ReceiptDetail);

quantity '数量' = DATA NUMERIC[14,2] (ReceiptDetail);
price '价格' = DATA NUMERIC[14,2] (ReceiptDetail);

要将其"过账"到账册中，我们需要使用ReceiptDetail类扩展类SKULedger用于仓库入库。我们还需要扩展账册的属性。

EXTEND CLASS ReceiptDetail : SKULedger;

// 必须指定[SKULedger]，因为在同一示例中ReceiptDetail还继承了PriceLedger，平台需要知道需要实现哪个属性
posted[SKULedger](ReceiptDetail d) += posted(receipt(d));
dateTime[SKULedger](ReceiptDetail d) += dateTime(receipt(d));

stock[SKULedger](ReceiptDetail d) += stock(receipt(d));

sku[SKULedger](ReceiptDetail d) += sku(d);
quantity[SKULedger](ReceiptDetail d) += quantity(d);

让我们看看更复杂的情况，当我们有一个记录从一个仓库到另一个仓库转移的文档。

CLASS Transfer '仓库间移动';
posted '已完成' = DATA BOOLEAN (Transfer);
dateTime '日期/时间' = DATA DATETIME (Transfer);

fromStock '仓库（来源）' = DATA Stock (Transfer);
toStock '仓库（目标）' = DATA Stock (Transfer);

CLASS TransferDetail '仓库出货行';
transfer '移动' = DATA Transfer (TransferDetail) NONULL DELETE;

sku 'SKU' = DATA SKU (TransferDetail);

quantity '数量' = DATA NUMERIC[14,2] (TransferDetail);
price '价格' = DATA NUMERIC[14,2] (TransferDetail);

在这种情况下，文档中的数据必须"过账"到账册两次。类似于仓库入库，我们将行作为带有负值的出库操作过账到账册。

EXTEND CLASS TransferDetail : SKULedger;

posted(TransferDetail d) += posted(transfer(d));
dateTime(TransferDetail d) += dateTime(transfer(d));

stock(TransferDetail d) += fromStock(transfer(d));

sku(TransferDetail d) += sku(d);
quantity(TransferDetail d) += -quantity(d);

要将其过账到SKU被转移到的仓库的账册中，我们使用对象聚合。转移文档中的行将生成一个对象，该对象又将被"过账"到账册。

CLASS TransferSKULedger '入库仓库（账册）' : SKULedger;
transferSKULedger = AGGR TransferSKULedger WHERE posted(TransferDetail transferDetail);

posted(TransferSKULedger d) += d IS TransferSKULedger;
dateTime(TransferSKULedger d) += dateTime(transfer(transferDetail(d)));

stock(TransferSKULedger d) += toStock(transfer(transferDetail(d)));

sku(TransferSKULedger d) += sku(transferDetail(d));
quantity(TransferSKULedger d) += quantity(transferDetail(d));

账册对象只有在转移文档被过账后才会创建。因此，在这种情况下，posted属性将始终等于TRUE。

应该注意的是，具有一个仓库的文档也可以使用聚合过账到账册。聚合方案更灵活，但需要在系统中创建额外的对象，这从性能角度可能会更差。

信息账册​

信息账册技术使得可以灵活实现某个指标随时间变化的逻辑。与库存账册不同，它计算的不是指标的总和，而是在特定时间段内的最新值。

为了实现这种技术，我们引入一个抽象类PriceLedger。它的实例反映一个SKU和一个仓库在特定时间的单次价格变化。

CLASS ABSTRACT PriceLedger '入库价格变更账册';

posted '已完成' = ABSTRACT BOOLEAN (PriceLedger);
dateTime '日期/时间' = ABSTRACT DATETIME (PriceLedger);

sku 'SKU' = ABSTRACT SKU (PriceLedger);
stock '仓库' = ABSTRACT Stock (PriceLedger);

price '价格' = ABSTRACT NUMERIC[14,2] (PriceLedger);

price '价格' (Stock stock, SKU sku, DATETIME dateTime) =
    GROUP LAST price(PriceLedger l)
          ORDER dateTime(l), l
          WHERE posted(l) AND dateTime(l) <= dateTime
          BY stock(l), sku(l);

price '价格' (Stock stock, SKU sku) =
    GROUP LAST price(PriceLedger l)
          ORDER dateTime(l), l
          WHERE posted(l)
          BY stock(l), sku(l);

price '价格' (SKU sku, DATETIME dateTime) =
    GROUP LAST price(PriceLedger l)
          ORDER dateTime(l), l
          WHERE posted(l) AND dateTime(l) <= dateTime
          BY sku(l);

结果，我们获得了按SKU和仓库在指定日期/时间的价格、最新价格以及该SKU在所有仓库中的最新价格的属性。

文档过账到信息账册的方式与过账到库存账册的方式相同。

EXTEND CLASS ReceiptDetail : PriceLedger;

// 必须指定[PriceLedger]，因为在同一示例中ReceiptDetail还继承了SKULedger，平台需要知道需要实现哪个属性
posted[PriceLedger](ReceiptDetail d) += posted(receipt(d));
dateTime[PriceLedger](ReceiptDetail d) += dateTime(receipt(d));

stock[PriceLedger](ReceiptDetail d) += stock(receipt(d));

sku[PriceLedger](ReceiptDetail d) += sku(d);
price[PriceLedger](ReceiptDetail d) += price(d);

在这种情况下，需要明确指定抽象属性的签名，因为可能有多个具有相同名称和命名空间的属性（对于类SKULedger，属性的命名方式完全相同）。
