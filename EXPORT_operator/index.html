EXPORT 操作符

EXPORT操作符：创建一个动作，用于将指定属性导出到文件，或者更一般地说，在结构化视图中打开表单。

Syntax​

EXPORT [exportFormat] FROM [columnId1 =] propertyExpr1, ..., [columnIdN = ] propertyExprN 
  [WHERE whereExpr] [ORDER orderExpr1 [DESC], ..., orderExprL [DESC]]
  [TOP topExpr] [OFFSET offsetExpr] [TO propertyId]
EXPORT formName [OBJECTS objName1 = expr1, ..., objNameK = exprK] [exportFormat] 
  [TOP (topExpr | (topGroupId1 = topPropertyExpr1, ..., topGroupIdT = topPropertyExprT))]
  [OFFSET (offsetExpr | (offsetGroupId1 = offsetPropertyExpr1, ..., offsetGroupIdF = offsetPropertyExprF))]
  [TO (propertyId | (groupId1 = propertyId1, ..., groupIdM = propertyIdM))]

exportFormat can be specified by one of the following options:

JSON [CHARSET charsetStr]
XML [HEADER | NOHEADER] [ATTR] [CHARSET charsetStr]
CSV [separator] [HEADER | NOHEADER] [ESCAPE | NOESCAPE] [CHARSET charsetStr]
XLS [SHEET sheetProperty] [HEADER | NOHEADER]
XLSX [SHEET sheetProperty] [HEADER | NOHEADER]
DBF [CHARSET charsetStr]
TABLE

描述​

EXPORT操作符创建一个动作，用于将指定属性或表单的数据导出到文件。支持以下文件格式：XML、JSON、CSV、XLS、XLSX、DBF、TABLE。

如果未指定导出文件格式，则默认为JSON。

如果数据导出到的属性属于FILE类，则生成文件的扩展名取决于格式，如下所示：

格式扩展名
JSONjson
XMLxml
CSVcsv
XLSxls
XLSXxlsx
DBFdbf
TABLEtable

在OBJECTS块中导出表单时，可以添加额外的过滤器来检查表单上的对象是否与传递的值相等。这些对象不会参与构建对象组层次结构。

参数​

导出源​

  • formName

    要导出数据的表单名称。复合ID。

  • objName1 ... objNameK

    为其指定了过滤（固定）值的表单对象名称。简单ID。

  • expr1 ... exprK

    表达式，其值决定表单对象的过滤（固定）值。

  • propertyExpr1, ..., propertyExprN

    表达式列表，数据从其值导出。每个属性映射到结果文件的表格列。

  • columnId1, ..., columnIdN

    结果文件中的列ID列表，相应属性的数据将导出到这些列中。列表中的每个元素是简单ID或字符串字面量。如果未指定ID，则默认等于expr<列号>。

  • whereExpr

    一个表达式，其值是导出的条件。如果未指定，则默认为所有导出属性的析取（即至少有一个属性必须非NULL）。

  • orderExpr1, ..., orderExprL

    表达式列表，导出的数据按此排序。只能使用列表propertyExpr1, ..., propertyExprN中存在的属性

  • DESC

    关键字。指定降序排序。默认使用升序排序。

  • TOP topExpr

  • TOP (topExpr | (topGroupId1 = topPropertyExpr1, ..., topGroupIdT = topPropertyExprT))

    仅导出前n条记录，其中n是表达式topExpr的值，或对于组对象topGroupIdT，是topPropertyExprT的值。

  • OFFSET offsetExpr

  • OFFSET (offsetExpr | (offsetGroupId1 = offsetPropertyExpr1, ..., offsetGroupIdF = offsetPropertyExprF))

    仅导出偏移量为m的记录，其中m是表达式offsetExpr的值，或对于组对象offsetGroupIdF，是offsetPropertyExprF的值。

导出格式​

  • ATTR

    一个关键字，指定值应导出到父标签的属性中。如果未指定，值将导出到子标签中。仅适用于导出到XML。

  • separator

    CSV文件中的分隔符。字符串字面量。如果未指定，则默认分隔符为;。

  • HEADER | NOHEADER

    关键字，指定CSV、XLS或XLSX文件中是否存在标题行（HEADER表示存在，NOHEADER表示不存在）。默认值为NOHEADER。

    使用NOHEADER选项时，如果列名是预定义名称之一（A、B、...、Z、AA、...、AE），则它会导出到具有相应数字的列，后续列按顺序导出到该列之后的列。

    对于XML有其他含义：使用HEADER时，结果文件的第一行包含类似<?xml version="1.0" encoding="UTF-8"?>的字符串。使用NOHEADER导出时不包含此字符串。默认值为HEADER。

  • ESCAPE | NOESCAPE

    关键字，指定CSV文件中是否对特殊字符（\r、\n、"（双引号）和指定的分隔符（separator）进行转义（ESCAPE表示转义，NOESCAPE表示不转义）。只有在保证指定的分隔符不会出现在数据中的情况下，使用NOESCAPE才有意义。默认值为ESCAPE。

  • CHARSET charsetStr

    一个选项，指定用于导出的编码。

    • charsetStr

      定义编码的字符串字面量。JSON、XML、CSV的默认值为UTF-8，DBF的默认值为CP1251。

  • sheetProperty

    属性ID，其值用作导出文件中的工作表名称。该属性不能有参数。用于XLS和XLSX导出格式。

导出目标​

  • propertyId

    生成的文件将写入的属性ID。此属性不能有参数，其值必须是文件类（FILE、RAWFILE、JSONFILE等）。如果未指定此属性，则默认使用System.exportFile属性。

  • groupId1, ..., groupIdM

    要导出数据的表单中的对象组名称。简单ID。仅用于将表单导出为平面格式。

  • propertyId1 , ..., propertyIdM

    为指定对象组生成的文件将写入的属性ID。这些属性不能有参数，其值必须是文件类（FILE、RAWFILE、JSONFILE等）。仅用于将表单导出为平面格式。对于对象的空组，使用名称root。

示例​

CLASS Store;

name = DATA STRING[20] (Sku);
weight = DATA NUMERIC[10,2] (Sku);

in = DATA BOOLEAN (Store, Sku);

exportSkus (Store store)  {
    // 将指定仓库的所有Sku（其中in (Store, Sku)已指定）上传到DBF
    EXPORT DBF CHARSET 'CP866' FROM id(Sku s), name(s), weight(s) WHERE in(store, s); 
    // 上传到CSV，没有标题行且不转义特殊字符
    EXPORT CSV NOHEADER NOESCAPE FROM id(Sku s), name(s), weight(s) WHERE in(store, s); 
    // 上传JSON，按name[Sku]属性降序排序
    EXPORT FROM id(Sku s), name(s), weight(s) WHERE in(store, s) ORDER name(s) DESC; 
    // 上传JSON {"ff":"HI"}，因为默认获取名称值，并且平台
    // 将对象{"value":"HI"}获取为"HI"
    EXPORT FROM ff='HI'; 
    // 上传JSON "HI"，因为默认获取名称值，并且平台
    // 自动将对象{"value": "HI"}转换为"HI"
    EXPORT FROM 'HI'; 
}
FORM exportSku
    OBJECTS st = Store

    OBJECTS s = Sku
    PROPERTIES(s) id, name, weight
    FILTERS in(st, s)
;

exportSku (Store store)  {
    // 将指定仓库的所有Sku（其中in (Store, Sku)已指定）上传到DBF
    EXPORT exportSku OBJECTS st = store DBF CHARSET 'CP866';
    EXPORT exportSku XML;
    EXPORT exportSku OBJECTS st = store CSV ',';
}
