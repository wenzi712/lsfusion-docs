打印视图 (PRINT)

该操作符创建一个打开表单并以打印视图显示的动作。

格式​

在该操作符中，您可以指定表单数据在显示给用户之前将被转换的格式：DOC、DOCX、XLS、XLSX、PDF、HTML、RTF。

默认情况下，如果打印视图指定了格式，当客户端接收到指定格式的文件时，它会尝试使用标准的操作系统方式打开此文件（即使用与指定格式关联的程序）。然而，如有必要，除了格式之外，您还可以指定一个属性，该属性将包含生成的报表文件。在这种情况下，不会向客户端发送任何数据，所有数据处理将完全在服务器上进行。

交互式打印视图​

除了以指定格式导出外，打印视图还允许以以下交互式模式向用户显示信息：

  • PREVIEW - 表单将在客户端的新预览窗口中显示，以便用户可以决定将此表单导出到哪种格式或发送到打印机。如果未定义其他模式/格式，则默认使用此模式。
  • NOPREVIEW - 表单将自动发送到打印机。在此模式和前一模式中，您可以指定一个属性，用于确定将用于打印表单的打印机（如果不指定，则使用默认打印机）。
  • MESSAGE - 表单将作为消息显示。在这种情况下，假设表单由单个对象组组成。相应地，显示的消息由标题（包含所有具有空显示组的属性）和表格（其中行是该唯一对象组的对象集合，列是未在标题中显示的属性，即那些存在显示组且等于所打开表单的唯一对象组的属性）组成。在此模式下工作的操作符本质上是消息显示操作符的泛化。值得注意的是，此操作符模式在约束中使用（当向用户显示违反约束的对象时）。

与交互式视图类似，打印视图的交互模式使开发人员能够设置流程管理和表单位置的选项（它们的行为与交互式视图中表单打开操作符的相应选项相同）。

信息

当前实现：在MESSAGE模式下，表单始终显示为窗口（这种情况下不支持"异步窗口"选项）；在异步PREVIEW模式下，表单始终显示为标签页；在同步PREVIEW模式下 - 显示为窗口。NOPREVIEW模式始终是异步的，其中表单位置没有意义（因为不会直接向用户显示表单）

语言​

要打印视图打开表单，请使用PRINT操作符。

示例​

FORM printOrder
    OBJECTS o = Order
    PROPERTIES(o) currency, customer

    OBJECTS d = OrderDetail
    PROPERTIES(d) idSku, price
    FILTERS order(d) == o
;

print (Order o)  {
    PRINT printOrder OBJECTS o = o; // 打印

    LOCAL file = FILE ();
    PRINT printOrder OBJECTS o = o DOCX TO file;
    open(file());

    //v 2.0-2.1 语法
    LOCAL sheetName = STRING[255]();
    sheetName() <- 'encrypted';
    PRINT printOrder OBJECTS o = o XLS SHEET sheetName PASSWORD 'pass';

    //v 2.2 语法
    //PRINT printOrder OBJECTS o = o XLS SHEET 'encrypted' PASSWORD 'pass';
}
