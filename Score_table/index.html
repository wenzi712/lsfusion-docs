积分表

"积分表"任务描述​

这个信息系统使用lsFusion平台创建，应包含曲棍球锦标赛的记分功能。

锦标赛是指球队之间的一组比赛（每场比赛有两支球队参加），比赛结果会为球队带来积分。

在每场比赛中，比赛结果可以是其中一支球队的常规时间胜利（获胜球队获得3分）、加时赛胜利（获胜球队获得2分，失利球队获得1分）或点球决胜胜利（获胜球队获得2分，失利球队获得1分）。

球队在积分表中的排名由总积分决定。如果积分相同，则考虑额外参数：常规时间胜利次数、加时赛胜利次数、点球决胜胜利次数、进球与失球之差、进球数。这些额外参数按指定顺序依次应用，直到球队结果获得唯一排名。

定义领域逻辑​

模块声明​

我们声明一个模块，在其中实现所需功能。我们为模块分配一个任意名称（例如HockeyStats）。

MODULE HockeyStats;

我们定义在HockeyStats模块中使用其他模块的功能。特别是，我们需要系统模块System，其中声明了示例中使用的一些系统元素。

REQUIRE System, Utils;

球队定义​

我们引入球队的概念，为此我们使用相应的语句CLASS创建一个单独的类。

CLASS Team 'Team';

我们为创建的类分配一个名称（例如Team），该名称将在构建表达式时使用，以及在自定义表单上显示的标题（例如'Team'）。

为了在处理稍后创建的表单时轻松识别所有球队，我们为球队创建一个名称。换句话说，我们创建一个name属性，可以为Team类的对象定义。

name 'Team name' = DATA STRING[30] (Team) IN base;

因此，球队名称是一个数据（用户输入）字符串类型属性。使用IN选项，创建的属性被添加到预定义的base属性组中。属于base组的对象属性将在Team类对象选择对话框中自动显示。

信息

可选地，您可以创建用于编辑和查看球队的表单。

FORM team 'Team'
    OBJECTS t = Team PANEL
    PROPERTIES(t) name

    EDIT Team OBJECT t
;

FORM teams 'Teams'
    OBJECTS t = Team
    PROPERTIES(t) READONLY name
    PROPERTIES(t) NEWSESSION NEW, EDIT, DELETE

    LIST Team OBJECT t
;

NAVIGATOR {
    NEW teams;
}

比赛定义​

我们引入比赛的概念及其属性：日期、参赛者（主队和客队）及其名称。

CLASS Game 'Game';

date 'Date' = DATA DATE (Game);
hostTeam = DATA Team (Game);
guestTeam = DATA Team (Game);
hostTeamName 'Hosts' (Game game) = name(hostTeam(game));
guestTeamName 'Guests' (Game game) = name(guestTeam(game));

hostTeam和guestTeam属性是比赛的数据对象属性，其值分别是对主队和客队的引用（即对特定Team类对象的引用）。为比赛主队和客队名称创建的属性（hostTeamName和guestTeamName）用于后续在表单上使用。如果将hostTeam和guestTeam属性添加到表单中，用户将看到数据库中对象的内部ID。

我们引入约束，即比赛参赛者必须是两支不同的球队。

CONSTRAINT hostTeam(Game team) = guestTeam(team) CHECKED BY hostTeam, guestTeam MESSAGE 'Host and guest teams must be different';

这个表达式的运行机制如下：当比赛的主队或客队发生变化时，系统检查这些球队相等的条件hostTeam(team) == guestTeam(team)，如果满足，系统将阻止对数据库应用更改，并向用户显示指定的消息'Host and guest teams must be different'。换句话说，在CONSTRAINT运算符之后指定的表达式的结果必须为NULL。在所有其他情况下，限制将被视为违反。此外，由于CHECKED BY块，创建的约束将在选择比赛的主队或客队时过滤球队（即从选择球队时出现的对话框中的球队列表中排除已设置为对手的球队）。

我们定义每支球队在比赛中进球的数量。

hostGoals 'H goals' = DATA INTEGER (Game);
guestGoals 'G goals' = DATA INTEGER (Game);

定义的属性使用INTEGER类型，因为每支球队进球的数量是整数。

我们引入约束，即比赛不能以平局结束。系统应禁止用户为比赛中的两支球队设置相同数量的进球，并发出带有指定文本的消息。

CONSTRAINT hostGoals(Game game) = guestGoals(game) MESSAGE 'The game cannot end in a draw';

确定比赛获胜者​

我们确定比赛的获胜者 - 进球数多于对手的球队。

winner(Game game) = IF hostGoals(game) > guestGoals(game)
                    THEN hostTeam(game)
                    ELSE guestTeam(game);

这里我们使用IF... THEN... ELSE运算符，它检查主队在这场比赛中进球数多于客队的条件。如果满足，获胜者是主队；如果不满足，获胜者是客队。

通过类似的原则，参加比赛并进球数少于对手的球队将被视为失败者。

looser(Game game) = IF hostGoals(game) > guestGoals(game)
                    THEN guestTeam(game)
                    ELSE hostTeam(game);

确定比赛结果​

我们引入可能的比赛结果概念，预定义了一组值：常规时间胜利、加时赛胜利、点球决胜胜利。

CLASS GameResult 'G/R' {
    win 'W',
    winOT 'LO',
    winSO 'LB'
}

为此，我们创建一个GameResult类，并在其中添加三个静态对象，这些对象使用大括号{ }中指定的表达式进行指定。在这种情况下，值win、winOT、winSO和W、OW、SW将分别存储在系统属性staticName和staticCaption中。

我们创建resultName属性，它将返回比赛结果的标题（W、OW或SW）。为此，我们使用系统属性staticCaption，该属性支持系统中的所有对象，并使用IF运算符约束其签名，表明对象必须是Game类的。此属性被添加到base属性组中，以便在选择GameResult类对象的自动对话框中显示。

resultName 'Name' (GameResult game) = staticCaption(game) IF game IS GameResult IN base;

我们确定特定比赛的结果。如果其中一支球队以2个或更多进球获胜，则比赛结果被视为常规时间胜利。如果不是，且仅当不是时，比赛结果（给定比分的胜利类型）将由用户设置。但是，用户不能将常规时间胜利设置为比赛结果。

userResult = DATA GameResult (Game);
result (Game game) = OVERRIDE userResult(game),
    (GameResult.win IF ((hostGoals(game) (-) guestGoals(game)) > 1 OR (guestGoals(game) (-) hostGoals(game)) > 1));
resultName 'G/R' (Game game) = resultName(result(game));

CONSTRAINT ((hostGoals(Game game) (-) guestGoals(game)) > 1 OR (hostGoals(game) (-) guestGoals(game)) < -1) AND userResult(game)
    MESSAGE 'The result of the game is determined automatically';

为了确定比赛结果，使用了OVERRIDE运算符，该运算符返回按表达式指定顺序的第一个非NULL值。在这种情况下，计算result属性将返回GameResult.win静态类的对象（如果比赛中的进球差大于1）或userResult数据属性的值。

为了始终确定比赛的结果，我们创建一个约束，确保如果结果不是根据比赛比分计算的，则用户设置userResult属性的值。

CONSTRAINT ((hostGoals(Game game) (-) guestGoals(game)) < 2 AND (hostGoals(game) (-) guestGoals(game)) > -2) AND NOT userResult(game)
    MESSAGE 'Specify the result of the game';

NOT userResult(game)表达式的结果仅在userResult(game)未定义（即如果它是NULL）时为真。因此，如果比分差为1且用户未指定胜利类型，则约束将被违反。

创建积分表​

积分表是锦标赛中球队的排名 - 按排名排序的球队列表。

我们定义确定球队在积分板上位置的指标：

  • 球队在主场和客场进行的比赛数量及其总数

    hostGamesPlayed = GROUP SUM 1 BY hostTeam(Game game);
    guestGamesPlayed = GROUP SUM 1 BY guestTeam(Game game);
    gamesPlayed 'G' (Team team) = hostGamesPlayed(team) (+) guestGamesPlayed(team);

这里，使用(+)而不是算术+，以在至少一个项的值为NULL时获得正确的结果。在这种情况下使用(+)相当于用0替换可能的NULL。如果其中一项是NULL，那么使用算术+也将导致NULL值。

要确定球队在主场和客场进行的比赛数量，使用GROUP SUM运算符，该运算符允许获取按一个或多个属性分组的特定类对象的给定表达式计算结果的总和（类似于Excel中的小计）。这里指定数字1进行求和，所有比赛按客队和主队分组（BY块）。结果，例如，hostGamesPlayed属性为球队（因为hostTeam属性计算的结果是Team类对象）确定作为主队进行的比赛数量（即对于主队等于已定义的所有比赛，数字1的总和）。通过此计算，系统分析输入到系统中的所有比赛。

  • 在常规时间、加时赛和额外时间（点球）中获胜的比赛数量

    gamesWonBy(Team team, GameResult type) = OVERRIDE [GROUP SUM 1 BY winner(Game game), result(game)](team, type),
                                                      0 IF team IS Team AND type IS GameResult MATERIALIZED;

    gamesWon 'W' (Team team) = gamesWonBy(team, GameResult.win);
    gamesWonOT 'WO' (Team team) = gamesWonBy(team, GameResult.winOT);
    gamesWonSO 'WB' (Team team) = gamesWonBy(team, GameResult.winSO);

由于确定球队每种类型胜利数量的逻辑几乎相同，我们创建并使用中间属性gamesWonByResult，它为一对对象（参数）定义。此属性计算球队（第一个参数）某种给定类型胜利的数量（第二个参数）。gamesWonBy属性的值通过OVERRIDE运算符计算，该运算符接受括号[...]中指定的表达式和0作为输入。如果表达式值为NULL，则整个属性的结果将是值0。使用方括号[...]指定嵌套表达式，使用GROUP SUM构造。在括号中使用给定表达式与使用具有类似表达式的先前定义的属性相同。因此，构造[...]允许您简单地减少代码行数。这里，GROUP SUM返回所有按比赛获胜者和比赛结果分组的比赛的数字1的总和。

gamesWonByResult属性的总结果将是给定球队的特定类型胜利的数量，如果球队没有这种类型的胜利，则为零（即，如果对于给定球队和胜利类型，[GROUP SUM 1 BY winner(Game game), result(game)]为NULL）。

  • 在常规时间、加时赛和额外时间进行的比赛数量（通过与上述胜利数量属性类似的方式确定）

    gamesLostBy(Team team, GameResult type) = OVERRIDE [GROUP SUM 1 BY looser(Game game), result(game)](team, type),
                                                       0 IF team IS Team AND type IS GameResult MATERIALIZED;

    gamesLost 'L' (Team team) = gamesLostBy(team, GameResult.win);
    gamesLostOT 'LO' (Team team) = gamesLostBy(team, GameResult.winOT);
    gamesLostSO 'LB' (Team team) = gamesLostBy(team, GameResult.winSO);

我们计算球队在锦标赛中获得的积分数量。计算方法是每支球队每种类型胜利的数量乘以授予的积分数量的总和。

points 'Points' (Team team) = gamesWon(team) * 3 + (gamesWonSO(team) + gamesWonOT(team)) * 2 + gamesLostOT(team) + gamesLostSO(team);

为了用作排名球队的额外指标，我们计算球队的总进球数和失球数。

hostGoalsScored = GROUP SUM hostGoals(Game game) BY hostTeam(game);
guestGoalsScored = GROUP SUM guestGoals(Game game) BY guestTeam(game);
goalsScored 'Scored goals' (Team team) = OVERRIDE hostGoalsScored(team) (+) guestGoalsScored(team), 0 IF team IS Team;

hostGoalsConceded = GROUP SUM guestGoals(Game game) BY hostTeam(game);
guestGoalsConceded = GROUP SUM hostGoals(Game game) BY guestTeam(game);
goalsConceded 'Conceded goals' (Team team) = OVERRIDE hostGoalsConceded(team) (+) guestGoalsConceded(team), 0 IF team IS Team;

我们确定球队在积分板上的位置。

place 'Rank' (Team team) = PARTITION SUM 1 ORDER DESC points(team), gamesWon(team), gamesWonOT(team), gamesWonSO(team),
                                               (OVERRIDE goalsScored(team) (-) goalsConceded(team), 0), goalsScored(team);

place属性"球队在积分表中的位置"是使用PARTITION SUM构造确定的，该构造对按ORDER运算符指定的顺序的某个类的所有对象的累计总数计算指定表达式的计算结果之和。重要的是要记住，参与确定顺序的所有属性的值不得为NULL。为此，倒数第二个表达式使用OVERRIDE运算符，以便在NULL时使用数字0。

因此，为每个命令确定place属性的逻辑如下：

  • 所有球队按照某些参数（得分点数、常规时间获胜场数以及ORDER DESC运算符后指定的其他属性）的值降序排列（排名）
  • 为每个球队计算指定SUM表达式的值之和（在这种情况下为数字1）。总和是为排名列表中该球队之前的所有球队（包括该球队）计算的。即，第一支球队为1，第二支球队为1+1，第三支球队为1+1+1，依此类推。

定义视图逻辑​

我们添加一个界面，允许您使用开发的系统，将数据输入系统并从中获取必要的信息。正在创建的表单将由两个垂直排列的块组成，在上部块中，用户将能够添加、修改和删除带有所有属性的游戏，而在下部块中将有一个积分表，显示游戏结果并允许添加或删除球队以及更改其名称。

我们声明带有名称和标题的表单。我们向表单添加Game类对象块，包含系统中定义的所有属性。我们还在表单上放置一个按钮，用于添加新游戏和删除它。

FORM scoreTable 'Score table'
    OBJECTS game = Game
    PROPERTIES(game) date, hostTeamName, hostGoals, guestGoals, guestTeamName, resultName, NEW, DELETE
;

FORM语句创建一个空表单，具有某些默认功能。使用OBJECTS game=Game表达式，将game对象添加到表单：一个表视图块，包含系统中输入的Game类的所有实例。使用带有后续列出的属性子集的表达式PROPERTIES(game)，将指定的属性添加到表单，并将"game"块的对象作为参数传递给它们。除了先前创建的属性外，操作NEW和DELETE也被放置在表单上，这些操作将在视觉上显示为按钮，并允许添加和删除Game类的对象。

显示在表单上的原始类型数据属性（date、hostGoals、guestGoals）将在视觉上显示为可由用户填充和更改的单元格。返回另一个对象属性的计算属性（hostTeamName、guestTeamName、resultName）将显示为单元格。当单击这些单元格时，将显示带有其对象列表和基本组属性的对话框（例如，当单击单元格hostTeamName"Guests"时，将出现带有球队列表的对话框）。在对话框中，您可以选择其中一个对象，从而更改原始表单对象的属性值（例如，更改游戏主队）。

我们通过向表单添加积分表块来扩展表单。积分表将显示为球队列表（Team类的对象）及其统计指标，使用ORDERS运算符按等级排序。

EXTEND FORM scoreTable
    OBJECTS team = Team
    PROPERTIES(team) place, name, gamesPlayed, gamesWon, gamesWonOT, gamesWonSO,
                     gamesLostSO, gamesLostOT, gamesLost, goalsScored, goalsConceded, points, NEW, DELETE
    ORDERS place(team)
;
信息

上面的表单可以使用单个代码块定义，而不使用EXTEND FORM语句。

FORM scoreTable 'Score table'
    OBJECTS game = Game
    PROPERTIES(game) date, hostTeamName, hostGoals, guestGoals, guestTeamName, resultName, NEW, DELETE

    OBJECTS team = Team
    PROPERTIES(team) place, name, gamesPlayed, gamesWon, gamesWonOT, gamesWonSO,
                     gamesLostSO, gamesLostOT, gamesLost, goalsScored, goalsConceded, points, NEW, DELETE
    ORDERS place(team)
;

我们将创建的表单放置在程序的主菜单上。

NAVIGATOR {
    NEW scoreTable;
}

MODULE HockeyStats;

REQUIRE System, Utils;

CLASS Team 'Team';

name 'Team name' = DATA STRING[30] (Team) IN base;

FORM team 'Team'
    OBJECTS t = Team PANEL
    PROPERTIES(t) name

    EDIT Team OBJECT t
;

FORM teams 'Teams'
    OBJECTS t = Team
    PROPERTIES(t) READONLY name
    PROPERTIES(t) NEWSESSION NEW, EDIT, DELETE

    LIST Team OBJECT t
;

    NEW teams;
}

CLASS Game 'Game';

date 'Date' = DATA DATE (Game);
hostTeam = DATA Team (Game);
guestTeam = DATA Team (Game);
hostTeamName 'Hosts' (Game game) = name(hostTeam(game));
guestTeamName 'Guests' (Game game) = name(guestTeam(game));

looser(Game game) = IF hostGoals(game) > guestGoals(game)
                    THEN guestTeam(game)
                    ELSE hostTeam(game);
