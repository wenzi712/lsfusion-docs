表单设计

表单设计 定义了表单在交互视图中的显示方式。

与任何GUI一样，表单设计是一个层次结构，其节点是组件。组件又可以是：

  • 容器：包含其他组件的组件。
  • 基础组件：表单结构和表单交互视图中描述的元素的图形视图。

每个组件在表单内必须有自己唯一的名称。

容器​

任何容器的所有子组件组成一个有序列表。需要确定表单上每个容器的子组件应如何放置。为此，可以为容器指定水平、选项卡式或行选项。

如果某个容器没有子组件，或者子组件不可见，则该容器会自动隐藏。反之，如果组件不是任何容器的子组件，则它不会在表单上显示。

基础组件​

在定义表单设计时，开发人员可以使用以下基础组件，这些组件根据表单结构自动创建：

对象组 / 树：

  • 表格/树（GRID）：由行和列组成的组件，其中行对应于相应对象组的对象集合，列对应于属性和动作。
  • 系统工具栏（TOOLBARSYSTEM）：由按钮组成的面板，用户可以使用这些按钮对表格中的行执行各种系统动作。如果表格变为不可见，则自动隐藏。
  • 用户筛选器（FILTERS）：用户可以使用此组件为表格创建和应用自己的筛选器。

筛选器组

  • 筛选器组（FILTERGROUP）：用户可以使用此组件激活表单筛选器组中所需的筛选器。

属性 / 动作

  • 属性面板（PROPERTY）：显示属性标题和当前值的组件。标题可以在值单元格的左侧或顶部。如果属性在表格中显示，则不显示。

尺寸和组件布局​

开发人员可以控制平台如何在其内部组件之间分配可用的容器大小，以及这些组件如何相对于彼此定位。

首先，对于每个组件，您可以指定该组件无论容器的放置算法如何都会获得的基础大小。

为了确定组件的最终大小及其在容器内的位置，使用以下算法：

对于每个容器，其中一个方向被视为动态的，另一个方向被视为静态的。动态方向由容器的名称确定，例如，对于垂直容器，动态方向将是垂直的；对于水平分割器，它将是水平的。对于分栏或可滚动容器，以及选项卡面板，动态方向被视为垂直的。此外，根据方向：

  • 动态：所有组件按照它们添加到容器的顺序一个接一个地放置。此外，可以为每个组件指定扩展系数（flex）。在这种情况下，容器中剩余的空间（即减去所有内部组件的基本尺寸）按照它们的扩展系数比例在所有组件之间分配。

  • 静态：可以为每个组件指定对齐方式（alignment）。其值可以是在开始处（START）、居中（CENTER）、在结束处（END）或拉伸（STRETCH）。在前三种情况下，组件获得其基础大小作为最终大小，并按照指定的对齐类型（即开始处、中心或结束处）进行定位。如果使用Stretch作为对齐类型，则组件的最终大小将是顶部容器的大小（但不小于基础大小），并且它将恰好位于中心。

信息

例如，在垂直顶部容器的情况下，如果组件设置为在开始处对齐，则它将位于容器的最左侧；如果设置为Stretch，则组件将占据从左到右边界的所有空间。

信息

此组件布局算法是CSS弹性盒布局的特例（并在Web客户端中使用它实现）。例如，CSS flex-grow 属性对应于扩展系数，flex-basis 对应于基础大小。

列容器将其静态（水平）方向分成N个相同的部分（其中N是列数）：每个部分然后有自己的组件，就好像这个部分是一个单独的垂直容器一样。

对于基础组件，您可以指定自动大小选项（autoSize）：在这种情况下，基础大小将自动更改，以恰好包含此基础组件的所有内容（例如，对于表格：所有记录加上标题）。

表格中的属性布局（或者更确切地说，显示其值的列）的完成方式与表格是水平容器且表格的列是此容器的内部组件的情况相同。

信息

由于当前实现在Web客户端中使用CSS表格布局固定机制在表格内放置列，并且其功能受到显著限制，因此表格中显示的属性的扩展系数可以等于0或基础大小。

标题和值单元格在属性面板内的放置方式与面板是容器（如果标题在左侧则为水平，在顶部则为垂直）的情况相同，其中单元格的扩展系数为1，标题为0，两者的对齐方式都设置为STRETCH

属性值单元格大小​

在显示属性时，您可以显式指定整个容器的尺寸（如果属性在面板中，则包括标题）以及值单元格本身的尺寸。为此，将前缀value添加到大小名称，例如valueWidth）。此外，值得注意的是，这些尺寸（值单元格的尺寸，而不是整个容器的尺寸）也用于确定属性在表格中显示时的列宽。

对于属性值单元格，也可以不使用像素，而是使用示例值（widthValue）指定基础宽度。在这种情况下，平台会考虑用户的字体/掩码/区域设置来计算并以像素为单位设置宽度，以便用户可以恰好看到指定的示例值（不多也不少）。假设示例值必须是字符串或者与属性值的类相等的类。

此外，单元格宽度可以以字符（charWidth）为单位指定，这相当于给出一个由给定数量的零组成的字符串示例值。

在后两种情况下（即以示例值或字符指定宽度时），如果属性值类在输入期间暗示右侧存在按钮（例如，DATE类），则此按钮的宽度（21像素）将添加到属性值单元格的宽度中。

默认尺寸和布局​

默认情况下，组件的扩展系数和对齐方式确定如下：

组件扩展系数对齐方式
表格 / 树1STRETCH
可滚动容器、分割器和选项卡面板内的组件1STRETCH
水平容器内的属性面板或表格中的属性。属性值是动态长度的内置类（即字符串和数字）的对象值单元格的宽度START
垂直容器内的属性面板。属性值是动态长度的内置类（即字符串和数字）的对象0STRETCH
所有其他0START

默认情况下，容器的基础大小（除选项卡面板外）对于动态方向等于其所有子组件的基础大小之和，对于静态方向等于最大值。选项卡面板的基础高度是其当前选项卡的基础高度和选项卡标题栏的高度之和，基础宽度与当前选项卡的基础宽度相同。

表格/树的基础宽度默认为130像素，高度为70像素。属性面板的基础大小的确定方式与面板是容器（如果标题在左侧则为水平，在顶部则为垂直）的情况相同，该容器由标题和值单元格组成。其余基础组件（以及属性面板中的标题）的基础大小以包含其中所有文本的方式确定。

属性值单元格尺寸​

默认情况下，使用以下公式确定属性值单元格的宽度：

属性值类宽度单位宽度/值
字符串以字符为单位IF length <= 12 result = length ELSE IF length = INFINITE result = 15 ELSE result = 12 + (length - 12) ^ 0.7
数字以字符为单位IF length <= 6 result = length ELSE IF this = DOUBLE result = 10 ELSE result = MIN(6 + (length - 6) ^ 0.7, 10)
COLOR以像素为单位40
文件和文件链接以像素为单位18
BOOLEAN以像素为单位25
DATE示例值1991_11_21
DATETIME示例值1991_11_21_10:55:55
TIME示例值10:55:55
用户类以字符为单位7

属性值单元格的默认高度等于所用字体的高度，除了其值属于TEXT类的属性（在这种情况下，高度是字体高度的四倍）。

窗口大小​

如果表单以窗口模式打开，则它没有顶部容器，因此需要确定此窗口的初始大小。此大小的确定方式类似于默认基础大小，唯一的区别是对于表格/树，默认大小不是作为常量确定的（默认值为130，70），而是以包含其全部内容的方式确定的（类似于自动调整大小机制），但宽度不小于130，高度不小于140。

默认设计​

可以基于表单的结构为每个表单创建自动设计。开发人员可以修改自动设计或从头创建设计。

自动设计的生成如下：

  • BOX：包含此表单的所有组件。垂直容器。扩展系数：1，对齐方式：STRETCH。
    • PANEL：包含在PANEL 视图中显示且显示组未定义的属性的组件（该属性没有参数）。内部结构和布局类似于对象组容器的内部结构和布局。
      • GROUP...
    • OBJECTS：包含为此表单上的对象组/树创建的所有组件。垂直容器。扩展系数：1，对齐方式：STRETCH。
      • BOX(<对象组/树>)：包含此对象组的所有组件。垂直容器。扩展系数：1，对齐方式：STRETCH。
        • GRID (<对象组 / 树>)：表格的基础组件。
        • TOOLBARBOX(<对象组 / 树>)：包含工具栏的所有组件（负责工具栏内的布局）。水平容器。对齐方式：STRETCH
          • TOOLBARLEFT(<对象组 / 树>)：工具栏的左侧。水平容器。对齐方式：CENTER。
            • TOOLBARSYSTEM(<对象组 / 树>)：系统工具栏的基础组件。对齐方式：CENTER。
          • TOOLBARRIGHT(<对象组 / 树>)：工具栏的右侧。水平容器。扩展系数：1，对齐方式：CENTER。
            • FILTERGROUPS(<筛选器组>)：包含为对应于对象组的筛选器组创建的所有组件。水平容器。对齐方式：CENTER。
              • FILTERGROUP：筛选器组的基础组件。对齐方式：CENTER。
            • TOOLBAR(<对象组 / 树>)：包含在TOOLBAR 视图中显示且显示组等于指定组的属性的组件。水平容器。对齐方式：CENTER。
              • PROPERTY(<属性>)：属性面板的基础组件。
        • FILTERS(<对象组 / 树>)：用户筛选器的基础组件。对齐方式：STRETCH。
        • PANEL(<对象组 / 树>)：包含在PANEL 视图中显示的属性的组件。垂直容器。对齐方式：STRETCH。如果多个属性属于需要创建单独容器的组，则为它们创建相应的容器层次结构，并将这些属性的组件放置在其中：
          • GROUP(<属性组>, <对象组 / 树>)：包含属于指定对象组和属性组的属性的组件（或不属于任何属性组：在这种情况下，未指定属性组，例如GROUP(,a))）。列容器。
            • PROPERTY(<属性>)：属性面板的基础组件。
    • TOOLBARBOX：包含在TOOLBAR 视图中显示且没有显示组的属性组件（例如，属性没有参数）。内部结构和布局类似于对象组的相应内部结构和布局（除了FILTERGROUPS，在没有对象组的情况下没有意义，因此在此容器中不存在）。
      • TOOLBARLEFT, TOOLBARRIGHT, TOOLBAR...

默认设计示例​


FORM myForm 'myForm'
    OBJECTS myObject = myClass
    PROPERTIES(myObject) myProperty1, myProperty2 PANEL
    FILTERGROUP myFilter
        FILTER 'myFilter' myProperty1(myObject)
;

默认设计中的容器和组件层次结构将如下所示：

语言​

要设置表单的设计，请使用DESIGN语句。

示例​

DESIGN order { // 自定义表单设计，从默认设计开始
    // 标记所有对层次结构的更改都将发生在最顶层容器
    // 在系统按钮之前创建一个新容器作为第一个容器，其中我们
    // 放置两个容器 - 标题和规格
    NEW orderPane FIRST { 
        fill = 1; // 指定容器应占据所有可用空间
        MOVE BOX(o) { // 将与对象o相关的所有内容移动到新容器
            PANEL(o) { // 配置对象o面板中属性的显示方式
                horizontal = FALSE; // 使所有后代从上到下排列
                NEW headerRow1 { // 创建容器 - 第一行
                    horizontal = TRUE;
                    MOVE PROPERTY(date(o)) { // 移动订单日期属性
                        // 在表单设计中"覆盖"属性标题（而不是标准标题）
                        caption = 'Date of the edited order'; 
                        // 为订单日期属性设置提示
                        tooltip = 'Input here the date the order was made'; 
                        background = #00FFFF; // 将背景设置为红色
                    }
                    MOVE PROPERTY(time(o)) { // 移动订单时间属性
                        foreground = #FF00FF; // 将颜色设置为绿色
                    }
                    MOVE PROPERTY(number(o)) { // 移动订单编号属性
                        // 设置用户最好显示5个字符
                        charWidth = 5; 
                    }
                    MOVE PROPERTY(series(o)); // 移动订单系列属性
                }
                NEW headerRow2 {
                    horizontal = FALSE; // 后代 - 从上到下
                }
                MOVE PROPERTY(note(o));
            }

            size = (400, 300); // 指定容器o.box应具有400x300像素的基础大小
        }
        // 创建一个容器来存储订单的各种规格
        NEW detailPane { 
            // 标记此容器应为选项卡面板，其中其后代为选项卡
            tabbed = TRUE;
            MOVE BOX(d) { // 将包含订单行的容器添加为顶部面板中的一个选项卡
                caption = 'Lines'; // 设置选项卡面板的标题
                // 使行号列永远不会获得焦点
                PROPERTY(index(d)) { focusable = FALSE; } 
                GRID(d) {
                    // 确保在打开表单时默认将焦点设置到行表格
                    defaultComponent = TRUE; 
                }
            }
            MOVE BOX(s) { // 将包含sku总计的容器添加为detailPane选项卡之一
                caption = 'Selection';
            }
        }
    }
}

// 将表单定义拆分为两个语句（第二个语句可以转移到另一个模块）
DESIGN order {
    // 从层次结构中移除带有打印和导出到xls按钮的容器，
    // 从而使它们不可见
    REMOVE TOOLBARLEFT; 
}
