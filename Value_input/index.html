值输入

此类别包含负责输入值的操作符：

  • 值输入 (INPUT)
  • 值请求 (REQUEST)

除了上述操作符外，还可以通过交互式视图中的消息显示 (ASK) 和 表单打开 (DIALOG) 操作符的对话框表单进行数据输入。

取消和输入结果​

在值输入操作符中，输入操作可能会被取消（对于每个操作符，这种情况以其自己的方式确定）。如果发生这种情况，TRUE值将被写入System.requestCanceled属性（否则将为NULL）。

对于所有值输入操作符，平台允许指定一个操作（我们称之为主操作），该操作仅在输入操作成功时（即未被取消时）执行。输入结果（如果有）作为参数传递给此操作。您可以为所有值输入操作符指定一个替代操作，该操作将在操作被取消时执行。

初始值和自动更改​

值输入通常假定存在某个初始（先前）值，输入从该值开始（即初始值被分配给当前值），用户随后可以更改该值。与大多数其他值一样，初始值被定义为属性。

此外，在许多情况下（特别是在属性更改事件处理程序中），有时不仅需要输入值，还需要将该值写入某个属性（通常是调用更改事件处理程序的属性。这是确保所见即所得（WYSYWIG）所必需的）。在大多数情况下，被更改的属性等于初始值属性。为了更方便地实现这种场景，您可以在值输入操作符中指定一个特殊选项（CHANGE）。此选项将自动将属性更改添加到主操作的末尾。在这种情况下，被更改的属性将是初始值属性，而更改的值将是输入结果（不过，如有必要，开发人员可以明确指定被更改的属性）。值得注意的是，上述功能只不过是语法糖，但它可以使代码更加简洁和可读。

信息

在当前平台实现中，初始值和自动更改仅支持原始输入操作符（INPUT）和对话框表单打开（DIALOG）。
