安全策略

用户角色定制​

系统使用用户（User）的概念。用户是一个包含在系统中工作的人员信息的对象。

用户可用的功能由其角色（Role）或角色列表决定。角色决定哪些表单、字段和按钮可用。

用户卡片（管理 > 安全 > 用户）显示分配给用户的角色（图1）。

图1. 分配员工角色

设置系统元素访问权限​

每个角色的安全策略在管理 > 安全 > 安全策略中配置。在这里，使用添加、删除和复制按钮确定角色的组成。复制角色时，所有权限也会被复制。

图2. 按角色配置安全策略

对于每个角色，需要指定名称（Name）和代码（Code），还可以设置以下参数：

  • 显示详细属性描述 - 如果将鼠标悬停在属性、按钮、字段等上，将在弹出窗口中显示描述。

图3. 详细属性描述

  • 单一表单实例 - 在导航器中选择已在桌面上打开的表单时，用户将转到该表单，不会打开额外的窗口。如果需要打开多个包含此表单的窗口，在调用表单时必须按住Ctrl键。此限制也可以在用户选项卡上为特定用户设置。
  • 客户端启动时的表单 - 可以为角色定义用户登录程序后立即打开的表单。为此，在导航器选项卡中，为所需表单的客户端启动序号字段指定顺序号（见图2）。
  • 客户端启动时最大化 - 登录时，程序将以全屏模式打开（导航菜单将隐藏）
  • 禁用角色 - 为该角色配置的访问权限将不适用。

导航器选项卡​

导航器选项卡显示系统中存在的所有表单。它们以树状或表格形式显示。在树状视图中，表单列表反映了现有模块的结构，最便于控制对它们的访问。

可以在访问列中为每个表单设置三种状态之一：

允许 - 用户可以访问表单

禁止 - 用户不能访问表单

默认 - 具有给定角色的用户默认可以使用该表单。如果用户有多个角色，则只要这些角色中有允许或没有禁止，表单就可用。

图4. 配置表单访问权限示例

为树分支设置的状态会自动应用于子分支。如有必要，可以为子分支设置不同的访问状态，在这种情况下，父元素的状态将以灰色背景突出显示，以便用户可以直观地确定此分支结构中存在具有不同访问状态的元素。根据图4所示的设置，用户在主数据菜单中只能使用模板表单。

图5. 设置表单访问权限的结果

属性选项卡​

属性选项卡显示系统中可用的所有表单元素：字段、按钮、复选框等。与表单本身一样，属性也以树状和表格形式呈现。

角色对属性操作可以有三种访问状态之一：

查看 - 定义属性对用户是否可见

修改 - 定义用户是否可以更改属性的值

跟踪 - 如果属性值是对象，定义用户是否可以访问该对象以更改它

属性的访问状态设置方式与表单相同。但是，考虑到应用程序中存在大量不同的属性，在属性选项卡上，可以方便地定义对对应树上部分支的属性组的访问级别，或者从属性角度查看角色权限的状态。

图6. 管理属性的访问权限

要设置对表单上特定属性的访问权限，使用上下文菜单非常方便。为此，右键单击要配置的按钮或字段，然后在上下文菜单中选择设置属性策略。将打开安全策略窗口，您可以在其中为每个角色或多个角色设置属性的访问级别，然后单击确定。通过上下文菜单进行并保存的所有更改也会显示在安全策略表单的属性选项卡上。

图7. 配置单个属性访问权限的选项

用户选项卡​

此选项卡显示所有被分配高亮显示角色作为主要或次要角色的用户。

图8. 用户选项卡

在安全策略表单上进行的所有更改在用户下次登录后生效。

平台默认预安装了4个角色：

  • Administrator（管理员）
  • Readonly（只读）
  • Default（默认）
  • Self register（自注册）
信息

请注意，为表单或属性设置的默认访问权限会根据设置的附加角色应用允许或限制。设置的默认访问权限不会应用默认角色设置中指定的值。要让用户应用默认角色中配置的访问权限，必须为用户分配该角色。

信息

Note that when set for a form or property, Default access applies permission or restriction, depending on the additional roles set. The set Default access does not apply the value specified in the Default role settings. To have the accesses configured in the Default role applied to a user, the user must be assigned that role.
