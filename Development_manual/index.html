开发环境手动安装

信息

要安装应用服务器、IDE和客户端，计算机上必须首先安装Java 8或更高版本。

信息

为了应用服务器正常工作，它必须能够访问至少9.6版本的PostgreSQL数据库管理服务器。PostgreSQL服务器必须接受使用md5或trust方法的密码认证连接。您可以通过编辑pg_hba.conf文件来配置授权，如PostgreSQL 文档中所述。

安装IDE插件​

信息

要安装插件，计算机上必须安装Intellij IDEA 2023.2或更高版本。

通过IDE安装应用服务器​

  • 在创建新的lsFusion项目时，点击lsFusion library对面的Download按钮：IDEA会自动从中央服务器下载最新（非测试版）版本的lsFusion服务器的JAR文件，并将该文件安装为此项目的依赖项（或者更确切地说，作为其唯一模块：File > Project Structure > Modules > project name > Dependencies tab）。此外，如果需要，您可以下载服务器的其他版本（不同于最新版本）或选择本地磁盘上先前下载的服务器JAR文件。
信息

请注意，IDEA会在其设置中记住下载/指定的应用服务器文件，并在创建其他lsFusion项目时自动设置它。

安装桌面客户端​

  • 服务器启动后，在启动日志的最后几行中，会有一行包含JNLP文件链接（例如，https://download.lsfusion.org/java/lsfusion-client-6.1.jnlp），运行该文件将使用Java Web Start技术自动安装客户端。

安装Web客户端​

  • 安装Apache Tomcat 9版本。
  • 如果使用Java 11以上版本，请将--add-opens=java.base/java.util=ALL-UNNAMED添加到Apache Tomcat启动参数中。
  • 下载lsFusion平台的Web客户端。
  • 将war文件放置在Apache Tomcat的webapps文件夹中，例如，命名为lsfusion.war。
  • Web客户端将在http://localhost:8080/lsfusion地址可用。

在现有Java项目中安装​

通过IDE安装应用服务器​

  • 从中央服务器下载所需版本的lsfusion-server-<version>.jar文件（例如，lsfusion-server-6.1.jar）到所需项目模块的文件夹（我们将此文件夹称为$FUSION_DIR$）。
  • 如果数据库服务器位于另一台计算机上，并且如果数据库服务器已启用授权（例如，对于Postgres，使用md5方法且postgres密码不为空），请设置数据库服务器连接参数（例如，通过在项目文件夹中创建启动设置文件）
  • 将下载的文件添加为所需项目模块的依赖项（File > Project Structure > Modules > module name > Dependencies tab > +）
  • 创建启动配置（创建新的lsFusion项目时，这部分和上面两个步骤会自动完成）。如果平台作为库加载，您可以使用lsfusion.xml配置文件中ID为logicsInstance的Spring bean及其start()和stop()方法（分别负责启动和停止应用服务器）来代替创建配置。
信息

对于现有的Maven项目，服务器的安装和加载可以（并且应该）以不同的方式进行（见下文）。

通过Maven安装应用服务器（仅适用于Maven项目）​

  • 在pom.xml中注册为父级lsfusion.platform.build:logics或依赖项lsfusion.platform:server（目前这些构件不在中央仓库中，因此必须额外指定到lsFusion仓库的路径）。例如：

    <repositories>
        <repository>
            <id>lsfusion</id>
            <name>lsFusion Public Repository</name>
            <url>https://repo.lsfusion.org</url>
        </repository>
    </repositories>

    <parent>
        <groupId>lsfusion.platform.build</groupId>
        <artifactId>logics</artifactId>
        <version>6.1</version>
    </parent>

    第一个选项（使用父级）的优点是：

    • Maven将自动配置uber-jar装配（即包含所有项目文件的单个文件）。您可以使用Maven配置文件assemble启动此装配 - 当此配置文件被激活时，在package阶段会额外生成一个带有assembly后缀的JAR文件，其中不仅包含项目文件，还包含所有项目依赖项的文件。生成的jar文件将只不包含应用服务器文件。这在应用服务器与应用本身分开安装（例如，使用自动安装）且不希望在生成的uber-jar中包含应用服务器的情况下很有用。除了assemble，平台还支持Maven配置文件embed-server，当它被激活（与assemble一起）时，在生成带有assembly后缀的jar文件时也会包含应用服务器文件（示例命令：mvn package -P assemble,embed-server）。

    • Maven和IDE将自动确定源代码和资源目录（例如，默认为src/main/lsfusion）

    • Maven将自动配置应用服务器方面的编织。然而，这很少需要 - 只有当项目中有应用程序RMI服务器（即继承自lsfusion.interop.server.RmiServerInterface并通过RMI远程访问的对象），或使用了应用服务器的各种系统注解（例如，用于缓存执行结果的lsfusion.server.base.caches.IdentityLazy）时才需要。

    在第二种情况下，所有上述内容必须由开发人员直接手动配置。

    对于其他不是使用创建新lsFusion项目操作创建的项目，对于Maven项目，您必须手动创建设置文件和启动配置（或者，如果平台需要作为库加载，则使用特殊Spring bean）

信息

如果处理大型项目，并且项目中积极使用元代码（例如ERP），建议在IDEA启动参数（idea.exe.vmoptions或idea64.exe.vmoptions）中将Xmx增加到至少2GB。
